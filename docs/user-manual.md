# Proma 使用手册

Proma 是一个本地优先的 AI 桌面工作台，把 Chat、Agent、模型配置、工作区、Skills、MCP、联网搜索和本地文件能力放在同一个客户端里。它适合两类使用方式：轻量问题直接用 Chat，复杂任务交给 Agent 在工作区里推进。

## 快速上手

首次使用建议按这个顺序配置：

1. 打开 Proma，完成环境检查。Agent 模式依赖本机 Shell、Node.js / Bun、Git 等基础环境。
2. 进入 **设置 > 模型配置**，添加至少一个模型渠道。
3. 如果只使用 Chat，配置 OpenAI、Anthropic、Google 或 OpenAI 兼容渠道即可。
4. 如果要使用 Agent，优先配置 Anthropic 或 Anthropic 兼容渠道，并在 **Agent 供应商** 区域启用。
5. 进入 **设置 > Agent 配置**，选择或创建工作区，确认当前工作区的 Skills 和 MCP。
6. 如需联网检索，进入 **设置 > Chat 工具 > 联网搜索**，打开开关并用测试关键字验证。

## Chat 和 Agent 怎么区分

### Chat 模式

Chat 是轻量对话模式，适合直接回答、分析、总结和改写。它不会主动修改你的本地项目文件，主要通过模型对话和 Chat 工具完成一次性任务。

适合 Chat 的场景：

- 问答、解释、翻译、润色、头脑风暴。
- 粘贴内容或上传附件后做总结、对比、结构化提炼。
- 需要快速联网搜索一个问题。
- 同时对比不同模型的回答。
- 使用 Chat 工具调用一个明确的 API 或服务。

### Agent 模式

Agent 是任务执行模式，适合让 AI 在工作区中读取文件、调用命令、使用 Skills、配置 MCP，并产出可落地的结果。Agent 会保留会话上下文，并根据权限策略请求确认。

适合 Agent 的场景：

- 修改、创建、整理本地文件。
- 阅读项目代码、分析问题、执行测试、提交代码。
- 需要多步骤调研、规划、执行和复盘。
- 需要使用工作区 Skills 或 MCP 服务。
- 需要让 AI 沉淀可复用流程。

简单判断：**只需要回答时用 Chat，需要行动、改文件、跑命令或交付结果时用 Agent。**

## 模型配置

模型配置位于 **设置 > 模型配置**。

每个模型渠道通常需要填写：

- 渠道名称：方便你在 Chat 或 Agent 里识别。
- Provider：模型供应商或协议类型。
- Base URL：模型服务地址。
- API Key：模型服务密钥，会通过 Electron safeStorage 加密保存。
- 模型列表：可用模型名称，可单独启用或关闭。

### Chat 可用渠道

Chat 模式支持多种 Provider 和兼容协议，包括 Anthropic、OpenAI、Google、DeepSeek、Kimi、MiniMax、豆包、通义千问、智谱 AI 和自定义 OpenAI 兼容端点。

### Agent 可用渠道

Agent 模式底层使用 Claude Agent SDK，因此要求渠道提供 Anthropic 或 Anthropic 兼容协议。添加渠道后，需要在 **模型配置 > Agent 供应商** 中启用该渠道，Agent 模式才会显示和使用。

如果 Agent 调用模型时出现 `HTTP 200 但响应为空或格式异常` 一类错误，通常不是 Proma 本地配置问题，而是模型网关返回了 SDK 无法解析的内容。可以先尝试：

- 切换到另一个 Anthropic 兼容模型。
- 检查 Base URL 是否走了错误代理或网关。
- 降低单次输入内容量。
- 避免让工具一次性返回过大的网页正文或文件内容。

## 工作区

Agent 的所有长期能力都围绕工作区组织。不同工作区可以有不同的 Skills、MCP、工作区文件和会话。

典型用法：

- 一个代码仓库对应一个工作区。
- 一个产品项目或业务领域对应一个工作区。
- 临时研究任务可以建一个独立工作区，避免污染常用上下文。

工作区数据默认存放在 `~/.proma/agent-workspaces/` 下。

## Skills

Skill 是 Agent 的可复用工作流。它不是单纯的 API 工具，而是一组“遇到某类任务时应该怎么做”的说明、脚本和参考文件。

### Skill 适合做什么

- 固化反复出现的工作流程。
- 记录项目、团队或岗位的交付规范。
- 给 Agent 提供专门领域的方法论和辅助脚本。
- 把复杂任务拆成稳定步骤，减少每次重复解释。

### Skill 和 Chat 工具的区别

| 类型 | 适合解决 | 使用位置 |
| --- | --- | --- |
| Skill | 流程、规范、方法、长期复用能力 | Agent 模式 |
| Chat 工具 | 调 API、查服务、执行一次明确查询 | Chat 模式 |
| MCP | 给 Agent 接入外部系统和数据源 | Agent 模式 |

如果你想“以后遇到这类任务都按这个流程做”，适合做 Skill。  
如果你想“帮我调用这个 HTTP API 查一下结果”，更适合做 Chat 工具。

## 内置 Skills

Proma 会预置一组默认 Skills，新工作区会自动获得这些能力。旧工作区会根据 Skill 版本自动升级或补齐缺失 Skill。

### 文档和文件类

| Skill | 能力 |
| --- | --- |
| `docx` | 创建、读取、编辑和整理 Word 文档，支持目录、标题、页码、模板、图片替换等场景。 |
| `pdf` | 读取、拆分、合并、旋转、加水印、OCR、提取图片和生成 PDF。 |
| `pptx` | 创建、读取、编辑、合并和拆分 PPTX 演示文稿。 |
| `xlsx` | 读取、清洗、编辑、生成 Excel / CSV / TSV 表格。 |
| `drawio` | 创建流程图、架构图、ER 图、时序图、类图、网络图、线框图和 `.drawio` 文件。 |

### 内容生产和业务类

| Skill | 能力 |
| --- | --- |
| `guizang-ppt-skill` | 生成横向翻页网页 PPT，支持电子杂志风和瑞士国际主义风格。 |
| `po-skill` | 产品经理工具集，将 Wiki 或本地文档转换为结构化 PRD，并可创建 DPMP Story。 |
| `web-search` | Agent 模式的联网检索能力，适合查最新公开网页信息。 |

### Skill 管理和工作流类

| Skill | 能力 |
| --- | --- |
| `find-skills` | 帮你发现和安装适合某类任务的 Skill。 |
| `skill-creator` | 创建、修改、优化和评测 Skill。 |
| `tool-builder` | 交互式创建 Chat 模式下可用的自定义 HTTP 工具。 |
| `proma-coach` | 当你反复遇到同类摩擦时，帮助判断是否应该沉淀成 Skill。 |
| `brainstorming` | 在创造功能、组件或方案前先澄清需求和设计。 |
| `writing-plans` | 把明确需求拆成可执行实施计划。 |
| `executing-plans` | 按已有计划分步执行，并保留检查点。 |

### 如何启用、查看和编辑 Skill

进入 **设置 > Agent 配置 > Skills**：

- 左侧查看当前工作区已安装 Skills。
- 点击某个 Skill 查看名称、描述、版本和位置。
- 在“说明”页查看或编辑 `SKILL.md`。
- 在“资源文件”页查看该 Skill 的脚本、参考文档和模板文件。
- 可以启用、禁用、删除或从其他工作区导入 Skill。

建议只把稳定、反复出现的流程沉淀成 Skill。一次性的口味偏好直接告诉 Agent 即可，不必都写成 Skill。

## 华泰 SkillHub

华泰 SkillHub 是 Proma 内置的 Skill 市场入口，用来从远端 SkillHub 安装团队或组织维护的 Skills。

入口：**设置 > Agent 配置 > 华泰 SkillHub**。

主要操作：

- 搜索 Skill 名称或描述。
- 按“全部 / 已安装 / 未安装”筛选。
- 查看远端 `SKILL.md` 说明。
- 点击“安装”把 Skill 安装到当前工作区。
- 对已安装 Skill 可选择“覆盖安装”，用于同步远端新版本。
- 点击文件夹按钮打开已安装 Skill 目录。

注意事项：

- SkillHub 安装是工作区级别的，只影响当前工作区。
- 覆盖安装会替换当前工作区内同名 Skill 文件，改过本地文件时要谨慎。
- 安装后建议新开 Agent 会话，确保 Agent 读取到最新 Skill 状态。

## Agent 联网搜索

Agent 模式使用 `web-search` Skill 进行联网搜索。它会调用内置脚本 `scripts/search.mjs`，访问 Proma 内置的数智中台搜索服务，并且明确不走代理环境变量，避免内网检索接口被外部代理拦截。

### 什么时候会触发

当用户要求查找最新信息、公开网页资料、新闻、近期公告，或 Agent 判断自身知识可能过时时，`web-search` Skill 会被触发。

示例：

- “帮我查一下最近一个月关于某公司的新闻。”
- “重新检索张乐财富通的信息。”
- “查一下 2026 年最新政策原文，并给出处。”

### 默认有效期

联网检索默认查 **近一个月**。如果用户指定时间范围，Agent 会使用对应参数：

| 用户表达 | 参数 |
| --- | --- |
| 近一天 | `OneDay` |
| 近一周 | `OneWeek` |
| 近一个月 | `OneMonth` |
| 近一年 | `OneYear` |

Agent 内部命令示例：

```bash
node scripts/search.mjs --time-range OneWeek "搜索关键词"
```

### 摘要优先，必要时再看详情

为了避免检索内容过大导致模型网关报错，`web-search` 默认返回紧凑预览，每条结果最多保留约 600 字内容。需要进一步核对细节时，Agent 可以扩大预览长度：

```bash
node scripts/search.mjs --time-range OneMonth --max-content-chars 2000 "搜索关键词"
```

建议工作方式：

1. 先用关键词检索，得到标题、链接和摘要。
2. 让 Agent 先基于摘要判断哪些结果值得看。
3. 再针对少量关键结果扩大内容长度或重新检索。
4. 最终回答中保留 URL 来源。

### 常见问题

如果搜索返回 `count: 0`，说明接口连通但没有找到结果。可以尝试：

- 换更常见或更短的关键词。
- 把有效期从近一天扩大到近一个月或近一年。
- 使用公司全称、产品全称或官方名称。
- 避免一次输入多个不相关关键词。

如果搜索成功后模型报 `API returned an empty or malformed response (HTTP 200)`，通常是后续模型网关响应异常，不一定是搜索失败。可以切换模型、减少工具输出内容，或让 Agent 先输出摘要再继续查详情。

## Chat 工具和联网搜索

Chat 模式的工具入口在 **设置 > Chat 工具**。当前页面只保留内置 **联网搜索** 工具。

开启后，Chat 模式可以在需要实时信息时调用联网搜索。测试区支持输入关键词，并选择有效期：

- 近一天
- 近一周
- 近一个月
- 近一年

Chat 联网搜索和 Agent 联网搜索的区别：

| 能力 | Chat 联网搜索 | Agent 联网搜索 |
| --- | --- | --- |
| 入口 | 设置 > Chat 工具 | Agent 内置 `web-search` Skill |
| 适合 | 一次性问答、快速查资料 | 多步骤调研、写报告、结合文件和命令 |
| 执行方式 | Chat function calling | Agent 执行 Skill 脚本 |
| 是否能改文件 | 不适合 | 可以在工作区生成文档或报告 |

## MCP

MCP 用来给 Agent 接入外部系统或数据源，例如本地服务、知识库、数据库、搜索服务或业务系统。

入口：**设置 > Agent 配置 > MCP**。

Proma 支持常见 MCP 类型：

- `stdio`：通过本地命令启动 MCP Server。
- `http`：通过 HTTP 接入服务。
- `sse`：通过 SSE 接入服务。

配置建议：

- 每个工作区只启用当前任务真正需要的 MCP。
- 涉及密钥的环境变量尽量放在 MCP 配置里，不要写进对话。
- 不确定 MCP 怎么配时，可以在 Agent 模式里点击“AI 配置”，让 Agent 根据当前工作区帮你安装和写入配置。

## 记忆、提示词和外观

### 记忆

记忆用于跨会话保留长期偏好和事实，Chat 和 Agent 可以共享。适合存：

- 你的常用称呼、语言偏好、输出格式偏好。
- 长期项目背景。
- 反复使用的业务约定。

不建议把临时任务细节或敏感密钥写入记忆。

### 提示词管理

入口：**设置 > 提示词管理**。  
适合维护 Chat 模式常用系统提示词，例如翻译、写作、代码解释、审稿等。

### 外观和快捷键

入口：

- **设置 > 外观设置**：切换亮色、暗色或跟随系统主题。
- **设置 > 快捷键管理**：管理全局快捷键。
- **设置 > 磁盘管理**：清理缓存、预览文件和安装包等临时数据。

## 远程机器人

Proma 支持通过飞书 / Lark 等远程入口触发本机 Agent 工作流。适合在移动端、群聊或协作场景中让本机 Agent 继续处理任务。

使用建议：

- 远程机器人适合触发任务和查看进度，不适合输入大量敏感数据。
- 涉及文件修改、安装依赖、执行命令时，仍建议回到桌面端确认权限。

## 数据存储和迁移

Proma 采用本地优先存储，默认数据目录是 `~/.proma/`。

常见目录：

```text
~/.proma/
├── channels.json
├── conversations.json
├── conversations/
├── agent-sessions.json
├── agent-sessions/
├── agent-workspaces/
│   └── {workspace-slug}/
│       ├── workspace-files/
│       ├── mcp.json
│       └── skills/
├── attachments/
├── user-profile.json
├── settings.json
└── sdk-config/
```

Proma 不使用本地数据库，主要使用 JSON 配置和 JSONL 会话日志。迁移时可以备份整个 `~/.proma/` 目录。API Key 会加密保存，跨机器迁移后可能需要重新填写。

## 推荐工作流

### 快速问答

1. 进入 Chat。
2. 选择合适模型。
3. 直接提问。
4. 如需实时信息，打开 Chat 工具中的联网搜索。

### 复杂调研

1. 进入 Agent。
2. 选择工作区。
3. 明确告诉 Agent 目标、输出格式和时间范围。
4. 让 Agent 先用联网搜索找摘要。
5. 再让 Agent 深入少量关键来源。
6. 最后生成报告或保存到工作区文件。

### 沉淀重复流程

1. 在 Agent 模式中完成一次任务。
2. 如果发现以后还会反复做，告诉 Agent：“把这个流程沉淀成 Skill”。
3. 使用 `skill-creator` 创建或优化 Skill。
4. 下次类似任务让 Agent 自动按 Skill 执行。

### 给 Chat 增加 API 工具

1. 在 Agent 模式中告诉 Agent：“帮我创建一个 Chat 工具，用来调用某某 API”。
2. `tool-builder` 会引导你确认接口、参数和返回字段。
3. 工具写入 `~/.proma/chat-tools.json` 后，切回 Chat 模式测试。

## 常见排查

### Agent 不会调用某个 Skill

- 检查 Skill 是否安装在当前工作区。
- 检查 Skill 是否启用。
- 检查 `SKILL.md` 的 description 是否清楚描述了触发场景。
- 尝试更明确地描述任务，例如“用 web-search 查最近一个月的信息”。

### SkillHub 安装后没有生效

- 刷新 Agent 配置页。
- 新开一个 Agent 会话。
- 确认安装的是当前工作区，而不是另一个工作区。
- 如果本地改过同名 Skill，覆盖安装前先备份。

### 联网搜索没有结果

- 把有效期扩大到近一个月或近一年。
- 换更短、更标准的关键词。
- 使用官方名称、产品全称或英文名。
- 先让 Agent 搜索宽泛关键词，再逐步收窄。

### 模型可在 Chat 用，但 Agent 不可用

Agent 模式要求 Anthropic 或 Anthropic 兼容协议。OpenAI 兼容 Chat Completions 端点即使在 Chat 中可用，也不一定能用于 Agent。请在 **模型配置 > Agent 供应商** 中确认该渠道是否可启用。

### 什么时候该用 MCP，而不是 Skill

- 需要连接外部系统、数据库、文件服务或长驻服务：用 MCP。
- 需要固化操作流程、交付规范或领域方法：用 Skill。
- 需要调用单个 HTTP API 给 Chat 用：用 Chat 工具。

## 三、数据流对比

### Electron 版

```
User Input (Renderer)
    │
    ▼
window.electronAPI.sendMessage(input)
    │
    ▼ (IPC: contextBridge)
ipcMain.handle(AGENT_IPC_CHANNELS.SEND)
    │
    ▼
agent-service.runAgent(input, webContents)
    │
    ▼
orchestrator.sendMessage(input)
    │
    ▼
EventBus.emit(sessionId, payload)
    │
    ▼ (IPC 转发中间件)
webContents.send(STREAM_EVENT, payload)
    │
    ▼
Renderer Jotai atoms 更新
    │
    ▼
React UI 重新渲染
```

### 无头版

```
HTTP Client / IM Bridge
    │
    ▼
POST /v1/sessions/:id/messages  (HTTP API)
或
飞书/钉钉/微信消息                  (IM Bridge)
    │
    ▼
agent-service.runAgentHeadless(input, callbacks)
    │
    ▼
orchestrator.sendMessage(input)
    │
    ▼
EventBus.emit(sessionId, payload)
    │
    ├──▶ SSE 推送给 HTTP Client
    │    (local-api-server 订阅 EventBus)
    │
    └──▶ 飞书卡片推送
         (feishu-bridge 订阅 EventBus)
```

## 四、配置目录结构

无头版复用相同的配置目录结构，用户数据不冲突：

```
~/.workmate/
├── channels.json              # 渠道配置（API Key 等）
├── settings.json              # 应用设置
├── agent-sessions/            # Agent 会话数据
│   └── {session-id}.jsonl
├── agent-workspaces/          # 工作区
│   └── {workspace-slug}/
│       ├── mcp.json
│       ├── skills/
│       └── workspace-files/
├── conversations/             # 聊天会话
├── default-skills/            # 默认技能（启动时同步）
├── default-connectors/        # 默认连接器
├── default-plugins/           # 默认插件
├── connectors/                # 工作区连接器
├── automations/               # 自动化任务
├── memory/                    # 长期记忆
├── logs/                      # 日志文件
│   └── workmate-headless.log
└── cache/                     # 运行时缓存
    └── runtime.json
```

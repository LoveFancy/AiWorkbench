## 二、Phase 0：预初始化（`app.whenReady()` 之前）

所有操作在 `registerProtocolsAndHandlers()` 中完成。

### 步骤列表

| 序号 | 操作 | 耗时 | 说明 |
|------|------|------|------|
| P0.1 | `app.requestSingleInstanceLock()` | 毫秒级 | 单实例锁，防止重复启动。失败则 `app.quit()` |
| P0.2 | `app.setPath('userData')` | 毫秒级 | 开发模式用 `@proma/electron-dev`，正式版用 `HtAiWorkBench` |
| P0.3 | `app.setName('WorkMate')` | 毫秒级 | 设置应用名 |
| P0.4 | `installFileLogger()` | 毫秒级 | 安装文件日志到 `app.getPath('logs')` |
| P0.5 | `protocol.registerSchemesAsPrivileged()` | 毫秒级 | 注册 `proma-file://` 协议 |
| P0.6 | `app.commandLine.appendSwitch('disable-lcd-text')` | 毫秒级 | Windows 禁用 ClearType |
| P0.7 | 清理 `ANTHROPIC_*` 环境变量 | 毫秒级 | 防止终端环境变量干扰认证 |
| P0.8 | `app.on('open-file')` / `app.on('second-instance')` | 毫秒级 | 文件关联和双实例事件 |

**总耗时：< 10ms**

## 三、Phase 1：`bootstrap()` — 主进程初始化

代码位于 [index.ts](file:///d:/AiWorkbench/apps/electron/src/main/index.ts#L488-L620)。

所有步骤通过 `safeRun()` / `safeAwait()` 包裹，单点失败不阻断后续流程。

### 3.1 `setPromaVersion()` — 版本号设定

- **耗时**：毫秒级
- **说明**：将 `app.getVersion()` 写入全局供 User-Agent 使用

### 3.2 `protocol.handle('proma-file')` — 本地文件协议

- **耗时**：毫秒级
- **说明**：注册 `proma-file://` 的请求处理器

### 3.3 `initializeRuntime()` — 运行时环境检测 ⚠️ 首个阻塞点

- **耗时**：**数百ms ~ 数秒**（取决于系统环境）
- **类型**：`await safeAwait()` — **阻塞后续所有步骤**
- **详细子步骤**：

| 子步骤 | 平台 | 耗时 | 说明 |
|--------|------|------|------|
| 3.3.1 `loadShellEnv()` | macOS | 500ms~3s | 运行登录 Shell 提取环境变量（PATH 等） |
| 3.3.2 `loadWindowsEnv()` | Windows | 100ms~500ms | 从注册表读取系统/用户 PATH |
| 3.3.3 `detectNodeRuntime()` | 全平台 | 100ms~1s | `node --version` |
| 3.3.4 `detectBunRuntime()` | 全平台 | 100ms~1s | `bun --version`（多路径探测） |
| 3.3.5 `detectGitRuntime()` | 全平台 | 50ms~500ms | `git --version` |
| 3.3.6 `detectGitBash()` | Windows | 100ms~1s | 检测 Git Bash |
| 3.3.7 `detectWsl()` | Windows | 100ms~2s | `wsl --list --verbose` |

> **这是启动慢的最大瓶颈之一**。macOS 上 `loadShellEnv()` 需要 `execSync` 运行用户的登录 Shell；Windows 上 WSL 检测可能较慢。

代码位置：[runtime-init.ts](file:///d:/AiWorkbench/apps/electron/src/main/lib/runtime-init.ts#L26-L147)

### 3.4 `seedDefaultSkills()` — 同步默认 Skills

- **耗时**：100ms~500ms（首次）/ 50ms~200ms（后续）
- **类型**：`safeRun()` — 同步执行，不阻塞后续
- **说明**：从 app bundle 复制默认 Skills 到 `~/.workmate/default-skills/`
  - 首次启动：完整复制所有 Skill 目录
  - 后续启动：比较版本号，仅覆盖更新的 Skill
  - 开发模式从 `apps/electron/default-skills/` 读取，正式版从 `process.resourcesPath/default-skills`
- **潜在瓶颈**：涉及大量小文件复制（可能 4MB+），每个 Skill 比较 `SKILL.md` 中的 version

代码位置：[config-paths.ts](file:///d:/AiWorkbench/apps/electron/src/main/lib/config-paths.ts#L595-L665)

### 3.5 `seedDefaultPlugins()` — 同步默认插件

- **耗时**：50ms~200ms
- **类型**：同步执行（注意：此处**没有**包 `safeRun`，失败会抛异常）
- **说明**：复制默认插件到 `~/.workmate/default-plugins/`

代码位置：[config-paths.ts](file:///d:/AiWorkbench/apps/electron/src/main/lib/config-paths.ts#L667-L694)

### 3.6 `upgradeDefaultSkillsInWorkspaces()` — 升级工作区 Skills

- **耗时**：**100ms~数秒**（取决于工作区数量和 Skill 数量）
- **类型**：`safeRun()` — 同步执行
- **说明**：遍历所有工作区，逐个检查 Skills 是否需要版本升级
  - 读取所有工作区索引
  - 对每个工作区的 `skills/` 和 `skills-inactive/` 目录
  - 比较版本并 `rmSync + cpSync`
- **潜在瓶颈**：工作区越多越慢，涉及大量文件系统操作

代码位置：[agent-workspace-manager.ts](file:///d:/AiWorkbench/apps/electron/src/main/lib/agent-workspace-manager.ts#L329-L430)

### 3.7 `createApplicationMenu()` — 创建应用菜单

- **耗时**：毫秒级
- **说明**：构建 Electron 原生菜单

### 3.8—3.11 IPC Handlers 注册

| 函数 | 耗时 |
|------|------|
| `registerIpcHandlers()` | 毫秒级（注册大量 handler，但不执行） |
| `registerLogUploadIpc()` | 毫秒级 |
| `registerAuthIpcHandlers()` | 毫秒级 |
| `registerPlatformModelsIpcHandlers()` | 毫秒级 |

### 3.12 `initWorkmateServices()` — 观测上报

- **耗时**：毫秒级
- **说明**：注册全局异常捕获 + 初始化观测上报服务

代码位置：[workmate-init.ts](file:///d:/AiWorkbench/apps/electron/src/main/lib/workmate-init.ts#L1-L50)

### 3.13 `loadCacheFromDisk()` — 模型缓存加载

- **耗时**：毫秒级
- **说明**：从磁盘加载模型列表缓存（含加密 API Key），避免启动后立即网络请求

代码位置：`src/models/model-service.ts`

### 3.14 `initModelRefresh()` — 模型刷新定时器

- **耗时**：毫秒级
- **说明**：启动定时器，定期（默认 30 分钟）从服务端拉取模型列表

### 3.15 `createWindow()` — 创建主窗口

- **耗时**：100ms~500ms（取决于系统）
- **类型**：同步（窗口对象创建），内容异步加载
- **说明**：
  - 恢复上次窗口尺寸/位置 (`mainWindowState`)
  - 创建 `BrowserWindow`（`show: false`）
  - 开发模式：`loadURL('http://127.0.0.1:5173')`
  - 正式版：`loadFile('renderer/index.html')`
  - `ready-to-show` 事件后调用 `show()` + `maximize()`
  - 注册 `resize`/`move` 持久化（500ms 防抖）
  - macOS：关闭按钮隐藏窗口而非退出
  - Windows：关闭按钮隐藏到托盘而非退出
  - 拦截外部链接用系统浏览器打开

代码位置：[index.ts](file:///d:/AiWorkbench/apps/electron/src/main/index.ts#L321-L470)

### 3.16 `createTray()` — 系统托盘

- **耗时**：毫秒级
- **说明**：创建系统托盘图标和菜单

### 3.17 `startWorkspaceWatcher()` — 工作区文件监听

- **耗时**：毫秒级
- **说明**：递归监听 `agent-workspaces/` 目录，推送文件变更到渲染进程（防抖处理）

代码位置：[workspace-watcher.ts](file:///d:/AiWorkbench/apps/electron/src/main/lib/workspace-watcher.ts)

### 3.18 `startChatToolsWatcher()` — Chat 工具配置监听

- **耗时**：毫秒级
- **说明**：监听 `chat-tools.json` 变更

### 3.19 `initAutoUpdater()` — 自动更新

- **耗时**：毫秒级
- **说明**：启动更新检测定时器（首次 10s~60min 随机延迟，之后周期检查）

### 3.20 `createQuickTaskWindow()` — 预创建快速任务窗口

- **耗时**：100ms~500ms
- **说明**：预先创建隐藏的快速任务窗口（`show: false`），首次唤起秒开

### 3.21 `createVoiceDictationWindow()` — 语音输入窗口

- **耗时**：100ms~500ms（仅当 `voiceDictation.enabled === true`）
- **说明**：条件性预创建

### 3.22 `registerGlobalShortcut()` — 全局快捷键

- **耗时**：毫秒级
- **说明**：注册 3 个快捷键（快速任务、显示主窗口、语音输入）

### 3.23 `startAllBridges()` — 启动所有 Bridge ⚠️ 第二个阻塞点

- **耗时**：**数秒**（取决于启用的 Bridge 数量）
- **类型**：`await safeAwait()` — **阻塞后续步骤**
- **说明**：
  - 飞书 Bridge：每个启用的 Bot 独立启动 WebSocket 连接
  - 钉钉 Bridge：每个启用的 Bot 启动连接
  - 微信 Bridge：启动 WebSocket 连接
- **为什么阻塞**：Bridge 需要建立网络连接，可能需要多次重试

### 3.24 `startBridgeSelfHealing()` — Bridge 自愈

- **耗时**：毫秒级
- **说明**：启动定时器监控 Bridge 状态，异常时自动恢复

### 3.25 `startScheduler()` — 定时任务调度器

- **耗时**：毫秒级
- **说明**：恢复持久化的 active 任务，启动 10 秒 tick 定时器

## 六、耗时瓶颈汇总

| 排名 | 步骤 | 预估耗时 | 类型 | 是否阻塞 |
|------|------|----------|------|----------|
| 1 | `initializeRuntime()` — Shell 环境 + 运行时检测 | 500ms~5s | 异步阻塞 | **是** — 阻塞 Phase 1 所有后续步骤 |
| 2 | `startAllBridges()` — 飞书/钉钉/微信连接 | 1s~5s | 异步阻塞 | **是** — 阻塞 scheduler 等后续步骤 |
| 3 | `seedDefaultSkills()` + `seedDefaultPlugins()` + `upgradeDefaultSkillsInWorkspaces()` | 200ms~2s | 同步（safeRun） | 否 — 但不安全的插件同步可能抛异常 |
| 4 | 渲染进程加载 (HTML/JS/CSS) | 500ms~2s | 异步 | 窗口创建后才开始 |
| 5 | React mount + 15+ Initializers | 500ms~1.5s | 异步并行 | UI 就绪后才能交互 |
| 6 | `App` 组件 auth check | 100ms~500ms | 异步 | 显示 Login 或主界面 |

### 总启动耗时估算

| 场景 | 预估耗时 |
|------|----------|
| 最佳情况（macOS，各项检测命中缓存，少量工作区） | **1.5s ~ 3s** |
| 典型情况（Windows，Git Bash/WSL 检测，3-5 个工作区） | **3s ~ 6s** |
| 最差情况（首次启动，多项检测超时，多个 Bridge 重试） | **6s ~ 12s** |

## 七、优化建议

### 7.1 高优先级

1. **`initializeRuntime()` 非阻塞化**：Shell 环境加载和运行时检测改为后台进行，不阻塞窗口创建。当前 `await safeAwait()` 意味着用户必须等所有检测完成才能看到窗口。

2. **Skills/Plugins 同步异步化**：`seedDefaultSkills()`、`seedDefaultPlugins()`、`upgradeDefaultSkillsInWorkspaces()` 都是同步 IO 密集操作，可移到后台 worker 或 lazy 执行。

3. **Bridge 启动延迟**：`startAllBridges()` 可等窗口显示后再启动，或改为非阻塞 fire-and-forget。

### 7.2 中优先级

4. **渲染进程懒加载**：15+ Initializer 组件可分组延迟初始化，非关键组件使用 `requestIdleCallback` 或延迟 mount。

5. **模型缓存预加载优化**：`loadCacheFromDisk()` 虽然快，但后续 `AgentSettingsInitializer` 还要再调用 `listChannels()` + `getSettings()`，可合并为一次 IPC。

6. **预创建窗口按需**：`createQuickTaskWindow()` 和 `createVoiceDictationWindow()` 可在首次唤起时才创建，而非启动时预创建。

### 7.3 低优先级

7. **减少 settings.json 读取次数**：启动流程中多处独立读取 `settings.json`（`getSettings()` 被调用多次），可缓存首次读取结果。

8. **Tab 恢复可延迟**：`TabStatePersistenceInitializer` 的恢复操作可等主界面渲染后再执行。

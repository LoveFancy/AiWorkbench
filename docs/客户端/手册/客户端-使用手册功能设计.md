## 一、概述

### 1.1 做什么

将现有"使用教程"（`tutorial/tutorial.md` + `TutorialViewer.tsx`）和新增"使用手册"合并为统一的手册功能。

### 1.2 合并策略

| 项目 | 操作 |
|------|------|
| 设置页"使用教程" Tab | 保留，点击后打开 ManualPanel（与手册同一面板） |

| `TutorialBanner.tsx` | 删除 |
| `tutorial/tutorial.md` | 保留，作为内置 fallback |
| `tutorial-service.ts` | 保留 `createWelcomeConversation()`，`getTutorialContent()` 改为调用手册服务 |

### 1.3 三个入口

1. LeftSidebar 底部"使用手册"按钮（`BookOpen` 图标）
2. 设置 → 使用教程（打开同一 ManualPanel）
3. Agent 对话中通过 `workmate-manual` Skill 触发

## 三、接口对接

### 3.1 接口

```
GET /workmate/manual?version={客户端当前版本号}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `version` | Integer | 否 | 客户端缓存的手册版本号。首次请求不传 |

**响应（有更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": true,
    "title": "WorkMate 使用手册",
    "content": "# WorkMate 使用手册\n\n## 快速上手\n\n首次使用建议按这个顺序配置：\n\n1. 打开 WorkMate，完成**环境检查**\n2. 进入设置 > 模型配置，添加至少一个模型渠道\n\n![环境检查](data:image/png;base64,iVBORw0KGgo...)",
    "version": 4,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

**响应（无更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": false,
    "message": "已是最新版本"
  }
}
```

### 3.2 内容格式

服务端返回 **Markdown**，来自 `content_markdown` 字段（服务端 `turndown` 将 Tiptap 的 HTML 转为 Markdown）。图片以 base64 data URI 内嵌，无需额外下载。

内置 fallback（`tutorial/tutorial.md`）同为 Markdown，渲染链路完全一致。

### 3.3 版本比较

服务端完成比较，客户端只传本地版本号：

```
version 未传 或 == 0 → 返回最新
version < latestVersion → needUpdate: true
version >= latestVersion → needUpdate: false
```

### 3.4 鉴权

`GET /workmate/manual` 无需鉴权（已加入 `UserIdInterceptor` 排除列表）。

### 3.5 调用方式

```typescript
import { httpGet } from '../../shared/hteip-client'

const res = await httpGet<{ code: number; data: ManualApiResponse }>(
  '/workmate/manual',
  { params: { version: currentVersion } }
)
```

## 四、图片处理

服务端通过 `turndown` 将 Tiptap 中的图片转为 base64 data URI 内嵌在 Markdown 中：

```markdown
![环境检查](data:image/png;base64,iVBORw0KGgo...)
```

客户端 `react-markdown` 原生支持 base64 图片，直接渲染即可，无需额外处理。

内置 fallback 中的图片使用远程 URL，`react-markdown` 同样原生支持。

### 4.1 渲染

```tsx
// ManualView.tsx 自定义 img 组件
img: ({ src, alt }) => (
  <img
    src={src}           // data:image/... 或 https://
    alt={alt || ''}
    className="rounded-xl shadow-md max-w-full"
    loading="lazy"
  />
)
```

## 五、三级降级策略

### 5.1 降级链路

```
用户打开手册
  ↓
GET /workmate/manual?version=N
  ├── 成功 + needUpdate=true
  │     → 写入缓存 → 展示（source: server）
  │
  ├── 成功 + needUpdate=false
  │     → 展示缓存（source: cache）
  │
  └── 失败（网络/服务端异常）
        ├── 有缓存 → 展示缓存（source: cache）
        └── 无缓存 → 展示内置 fallback（source: builtin）
```

### 5.2 原则

- 全程不抛异常，不弹错误提示
- 内置 fallback 版本信息显示"内置版本"

### 5.3 缓存结构

```
~/.workmate/manual-cache/
├── version.json    # { "version": 4, "title": "...", "cachedAt": ... }
└── content.md      # Markdown（图片为 base64 内嵌）
```

### 5.4 内置 fallback

读取 `tutorial/tutorial.md`（通过 `electron-builder.yml` 的 `extraResources` 打入应用包）。

### 5.5 核心伪代码

```typescript
async function checkAndGetManual(): Promise<ManualContent | null> {
  const cachedVersion = readCachedVersion()

  // 1. 服务端
  try {
    const res = await httpGet('/workmate/manual', {
      params: { version: cachedVersion?.version ?? 0 }
    })
    if (res.data?.code === 0 && res.data.data?.needUpdate) {
      const d = res.data.data
      writeCache({ version: d.version!, title: d.title!, content: d.content! })
      return { version: d.version!, title: d.title!, content: d.content!, cachedAt: Date.now(), source: 'server' }
    }
    if (res.data?.code === 0 && !res.data.data?.needUpdate) {
      const cached = getCachedManual()
      if (cached) return { ...cached, source: 'cache' }
      const builtin = getBuiltInManual()
      return builtin ? { ...builtin, source: 'builtin' } : null
    }
  } catch {
    // 静默降级
  }

  // 2. 缓存
  const cached = getCachedManual()
  if (cached) return { ...cached, source: 'cache' }

  // 3. 内置 fallback
  const builtin = getBuiltInManual()
  return builtin ? { ...builtin, source: 'builtin' } : null
}
```

## 九、类型定义

```typescript
// packages/shared/src/types/manual.ts

export interface ManualApiResponse {
  needUpdate: boolean
  title?: string
  content?: string       // Markdown（服务端 content_markdown 字段）
  version?: number       // 整数递增
  updatedAt?: string     // ISO 8601
  message?: string
}

export interface ManualVersion {
  version: number        // 0 = 内置版本
  title: string
  cachedAt: number       // 内置版本为 0
}

export interface ManualContent {
  version: number
  title: string
  content: string        // Markdown（base64 内嵌图片）
  cachedAt: number
  source: 'server' | 'cache' | 'builtin'
}

export const MANUAL_IPC_CHANNELS = {
  CHECK_AND_GET: 'manual:check-and-get',
  GET_BUILT_IN: 'manual:get-built-in',
} as const
```

## 十一、实现要点

### 11.1 方法职责

| 方法 | 职责 |
|------|------|
| `checkAndGetManual()` | 三级降级编排 |
| `fetchFromServer()` | 调后端接口 |
| `getCachedManual()` | 读缓存 |
| `writeCache()` | 写缓存 |

### 11.2 规范

- 类型集中在 `packages/shared/src/types/manual.ts`
- IPC 通道使用 `MANUAL_IPC_CHANNELS` 常量
- `ManualContent.source` 标记内容来源
- 所有新增文件有 JSDoc 注释
- 不修改现有业务组件内部逻辑

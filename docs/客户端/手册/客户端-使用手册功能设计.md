## 一、概述

### 1.1 做什么

将现有"使用教程"（`tutorial/tutorial.md` + `TutorialViewer.tsx`）和新增"使用手册"合并为统一的手册功能。

### 1.2 合并策略

| 项目 | 操作 |
|------|------|
| 设置页"使用教程" Tab | 保留，点击后打开 ManualPanel（与手册同一面板） |
| `TutorialViewer.tsx` | 删除，改造为 `ManualView.tsx` |
| `TutorialBanner.tsx` | 删除 |
| `tutorial/tutorial.md` | 保留，作为内置 fallback |
| `tutorial-service.ts` | 保留 `createWelcomeConversation()`，`getTutorialContent()` 改为调用手册服务 |

### 1.3 三个入口

1. LeftSidebar 底部"使用手册"按钮（`BookOpen` 图标）
2. 设置 → 使用教程（打开同一 ManualPanel）
3. Agent 对话中通过 `workmate-manual` Skill 触发

### 1.4 查阅模式

手册提供两种查阅方式：

| 模式 | 接口 | 内容 | 图片 | 打开方式 |
|------|------|------|------|---------|
| 直接预览 | `GET /workmate/manual?version=N` | Markdown 纯文本 | 无 | 客户端内 react-markdown 渲染 |
| 图文版 | `GET /workmate/manual/html` | 完整 HTML | base64 内嵌 | 系统默认浏览器打开 |

> 两者分离：Markdown 接口不含图片，保证客户端内渲染流畅；图文版走 `shell.openPath` 在浏览器中打开，图片在浏览器侧解码。

## 三、接口对接

### 3.1 Markdown 接口（客户端内渲染）

```
GET /workmate/manual?version={客户端当前版本号}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `version` | Integer | 否 | 客户端缓存的手册版本号。首次请求不传 |

**响应（有更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": true,
    "title": "WorkMate 使用手册",
    "content": "# WorkMate 使用手册\n\n## 快速上手\n\n...",
    "version": 4,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

> `content` 为 Markdown 纯文本，**不含图片**。图片需通过图文版查阅。

**响应（无更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": false,
    "message": "已是最新版本"
  }
}
```

### 3.2 HTML 图文接口（浏览器打开）

```
GET /workmate/manual/html
```

**响应**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": true,
    "title": "WorkMate 使用手册",
    "content": "<h1>WorkMate 使用手册</h1><p>...</p><img src=\"data:image/png;base64,...\">",
    "version": 4,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

> `content` 为完整 HTML，含 base64 图片。客户端将其包裹为完整页面后写入临时文件，通过 `shell.openPath()` 在系统默认浏览器中打开。

### 3.3 版本比较

服务端完成比较，客户端只传本地版本号：

```
version 未传 或 == 0 → 返回最新
version < latestVersion → needUpdate: true
version >= latestVersion → needUpdate: false
```

### 3.4 鉴权

- `GET /workmate/manual` — 无需鉴权
- `GET /workmate/manual/html` — 无需鉴权

### 3.5 调用方式

```typescript
import { httpGet } from '../../shared/hteip-client'

// Markdown
const res = await httpGet<{ code: number; data: ManualApiResponse }>(
  '/workmate/manual',
  { params: { version: currentVersion } }
)

// HTML 图文
const res = await httpGet<{ code: number; data: ManualApiResponse }>(
  '/workmate/manual/html'
)
```

## 四、双模式查阅

### 4.1 Markdown 直接预览（客户端内）

- `GET /workmate/manual` 返回纯 Markdown（无图片），`react-markdown` 渲染
- 内容轻量（通常几十KB），渲染快
- 图片仅保留 `alt` 文本，点击「查看图文版」跳转浏览器

### 4.2 HTML 图文版（浏览器打开）

流程图：

```
用户点击「查看图文版」
  → IPC manual:open-html
  → openManualHtml()
    → httpGet('/workmate/manual/html')
    → wrapManualHtml(title, content)  // 包裹为完整 HTML 页面
    → writeFile(tempDir/workmate-manual-v{n}.html)
    → shell.openPath(filePath)         // 系统默认浏览器打开
```

包裹样式：

```css
body { max-width: 860px; margin: 0 auto; padding: 32px 24px;
       font-size: 16px; line-height: 1.8; color: #1a1a1a; }
img { max-width: 100%; border-radius: 8px; box-shadow: 0 2px 8px rgba(0,0,0,.08); }
/* ... 完整样式见 manual-service.ts wrapManualHtml() */
```

### 4.3 设计动机

| 维度 | 旧方案（base64 在 Markdown） | 新方案（双模式分离） |
|------|---------------------------|-------------------|
| Markdown 体积 | 7.6MB（含 base64 图片） | ~50KB（纯文本） |
| 客户端渲染 | 慢，需 base64→Blob 预处理 | 快，直接渲染 |
| 图片查阅 | 内嵌在 Markdown 中 | 「查看图文版」→ 浏览器 |
| 图片解码 | 阻塞客户端渲染线程 | 浏览器侧独立解码 |

## 五、三级降级策略

### 5.1 降级链路

```
用户打开手册
  ↓
GET /workmate/manual?version=N
  ├── 成功 + needUpdate=true
  │     → 写入缓存 → 展示（source: server）
  │
  ├── 成功 + needUpdate=false
  │     → 展示缓存（source: cache）
  │
  └── 失败（网络/服务端异常）
        ├── 有缓存 → 展示缓存（source: cache）
        └── 无缓存 → 展示内置 fallback（source: builtin）
```

### 5.2 请求去重与内存缓存

```
checkAndGetManual()
  ├── inflightRequest 存在 → 直接复用 Promise（避免 StrictMode 双重调用）
  ├── memoryCache 存在 → 直接返回（同次会话已成功获取）
  └── 无缓存 → 执行三级降级链路，成功后写入 memoryCache
```

### 5.3 原则

- 全程不抛异常，不弹错误提示
- `checkAndGetManual()` 静默降级，`openManualHtml()` 失败时抛异常给渲染端处理
- 内置 fallback 版本信息显示"内置版本"

### 5.4 缓存结构

```
~/.proma/manual-cache/
├── version.json    # { "version": 4, "title": "...", "cachedAt": ... }
└── content.md      # Markdown（纯文本，不含图片）
```

### 5.5 内置 fallback

读取 `tutorial/tutorial.md`（通过 `electron-builder.yml` 的 `extraResources` 打入应用包）。

### 5.6 核心伪代码

```typescript
async function checkAndGetManual(): Promise<ManualContent | null> {
  // 请求去重
  if (inflightRequest) return inflightRequest

  // 内存缓存
  if (memoryCache) return memoryCache

  inflightRequest = (async () => {
    const cachedVersion = readCachedVersion()

    // 1. 服务端
    try {
      const serverResult = await fetchFromServer(cachedVersion?.version ?? 0)
      if (serverResult) { memoryCache = serverResult; return serverResult }
      // needUpdate=false：使用缓存或 fallback
      if (cachedVersion) {
        const cached = getCachedManual()
        if (cached) { memoryCache = { ...cached, source: 'cache' }; return memoryCache }
      }
      const builtin = getBuiltInManual()
      const result = builtin ? { ...builtin, source: 'builtin' } : null
      if (result) memoryCache = result
      return result
    } catch { /* 静默降级 */ }

    // 2. 缓存
    const cached = getCachedManual()
    if (cached) { memoryCache = { ...cached, source: 'cache' }; return memoryCache }

    // 3. 内置 fallback
    const builtin = getBuiltInManual()
    const result = builtin ? { ...builtin, source: 'builtin' } : null
    if (result) memoryCache = result
    return result
  })()

  try { return await inflightRequest } finally { inflightRequest = null }
}

async function openManualHtml(): Promise<void> {
  const res = await httpGet('/workmate/manual/html')
  // 校验 code === 0
  const html = wrapManualHtml(title, content)
  await writeFile(tempPath, html, 'utf-8')
  await shell.openPath(tempPath)
}
```

## 九、类型定义

```typescript
// packages/shared/src/types/manual.ts

/** 客户端两种查阅方式：
 *    GET /workmate/manual  → Markdown 纯文本（不含图片），客户端内渲染
 *    GET /workmate/manual/html → HTML 图文版，浏览器中打开
 */

export interface ManualApiResponse {
  needUpdate: boolean
  title?: string
  content?: string       // Markdown 纯文本 或 HTML 图文
  version?: number       // 整数递增
  updatedAt?: string     // ISO 8601
  message?: string
}

export interface ManualVersion {
  version: number        // 0 = 内置版本
  title: string
  cachedAt: number       // 内置版本为 0
}

export interface ManualContent {
  version: number
  title: string
  content: string        // Markdown 纯文本（不含图片）
  cachedAt: number
  source: 'server' | 'cache' | 'builtin'
}

export const MANUAL_IPC_CHANNELS = {
  CHECK_AND_GET: 'manual:check-and-get',   // Markdown 内容
  GET_BUILT_IN: 'manual:get-built-in',     // 内置 fallback
  OPEN_HTML: 'manual:open-html',           // 图文版 → 浏览器打开
} as const
```

## 十一、实现要点

### 11.1 方法职责

| 方法 | 职责 |
|------|------|
| `checkAndGetManual()` | 三级降级编排 + 飞行中请求去重 + 内存缓存 |
| `fetchFromServer()` | 调 `GET /workmate/manual?version=N` |
| `openManualHtml()` | 调 `GET /workmate/manual/html` → 写临时文件 → `shell.openPath` |
| `wrapManualHtml()` | 将服务端 HTML 包裹为完整页面（含样式） |
| `getCachedManual()` | 读本地缓存 |
| `writeCache()` | 写本地缓存 |
| `getBuiltInManual()` | 读内置 fallback |

### 11.2 性能优化

| 优化点 | 说明 |
|------|------|
| `inflightRequest` 去重 | 避免 React StrictMode 双重 useEffect 导致重复网络请求 |
| `memoryCache` | 同次会话内已成功获取，后续调用零开销返回 |
| 静态 import | `ipc.ts` 顶部 import `manual-service`，避免每次 IPC 调用动态 `await import()` |
| `useMemo` 缓存渲染 | `ManualView.tsx` 中 `<Markdown>` 渲染用 `useMemo`，滚动导致的 `activeId` 变化不触发重新解析 |
| Markdown 不含图片 | 内容从 ~7.6MB 降至 ~50KB，IPC 传输和渲染都快 |

### 11.3 规范

- 类型集中在 `packages/shared/src/types/manual.ts`
- IPC 通道使用 `MANUAL_IPC_CHANNELS` 常量
- `ManualContent.source` 标记内容来源
- 所有新增文件有 JSDoc 注释
- 不修改现有业务组件内部逻辑

## 一、需求概述

### 1.1 功能目标

为 WorkMate 客户端提供统一的使用手册管理能力：

1. **管理台双Tab编辑**：左侧 Markdown 编辑器（编辑+预览）、右侧 Tiptap 富文本编辑器（支持 Word 粘贴、截图），两者独立编辑
2. **客户端 Markdown 接口**：返回 Markdown 文本，前端渲染（无图片 base64，避免卡顿）
3. **客户端 HTML 图文接口**：返回完整 HTML，新窗口打开查看图文版式
4. **版本管理**：HTML 和 Markdown 独立版本号，保存时比较新旧内容，仅对变化的格式升版本

### 1.2 核心流程

```
┌─────────────────────────────────────────────────────────┐
│                     管理台 (Admin)                        │
├─────────────────────────────────────────────────────────┤
│  Tab: Markdown 编辑          │  Tab: 富文本编辑           │
│  ┌──────────┬──────────┐    │  ┌────────────────────┐   │
│  │ 编辑列    │ 预览列    │    │  │ Tiptap 编辑器       │   │
│  │ (textarea)│ (渲染)    │    │  │  ├ Word粘贴(图文)   │   │
│  │          │          │    │  │  └ 截图粘贴          │   │
│  └──────────┴──────────┘    │  └────────────────────┘   │
│         ↓                         ↓                     │
│     markdownContent            htmlContent               │
│         └─────────┬──────────────┘                      │
│                   ↓                                     │
│         POST /console/manual                             │
│         {title, htmlContent, markdownContent}            │
└─────────────────────────────────────────────────────────┘
                           │
                           ▼
              ┌───────────────────────┐
              │  MySQL: manuals 元数据  │
              │  单行 id, html_version  │
              │   md_version            │
              │  文件: v{n}.html / .md  │
              └───────────────────────┘
                           │
              ┌────────────┴────────────┐
              ▼                         ▼
┌───────────────────────┐  ┌───────────────────────┐
│  GET /manual?version  │  │ GET /manual/html?version│
│  → Markdown（无图片）   │  │ → HTML 图文             │
│  客户端渲染 (快)        │  │ 新窗口打开 (图文链接)     │
└───────────────────────┘  └───────────────────────┘
```

## 二、存储设计

### 2.1 存储架构

```
文件系统 (${workmate.manual.storage-path})
├── v1.html     ← HTML 图文内容（富文本编辑器产出，html_version=1）
├── v1.md       ← Markdown 内容（Markdown 编辑器产出，md_version=1）
├── v2.html     ← 仅 HTML 更新，md_version 仍为 1
├── v3.md       ← 仅 MD 更新，html_version 仍为 2
└── ...

MySQL (manuals 表) 单行记录
│ id │ title │ html_version │ md_version │ created_at │ updated_at │
│  1 │ 使用手册 │    2         │    3       │    ...     │    ...     │
```

> HTML 和 Markdown **独立版本号**，各自递增，互不干扰。

### 2.2 数据库表结构

```sql
-- V8__manuals.sql（合并后唯一迁移脚本）
CREATE TABLE `manuals` (
    `id`           INTEGER      NOT NULL AUTO_INCREMENT,
    `title`        VARCHAR(256) NOT NULL DEFAULT '使用手册',
    `html_version` INTEGER      NOT NULL DEFAULT 0,
    `md_version`   INTEGER      NOT NULL DEFAULT 0,
    `created_at`   DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`   DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    PRIMARY KEY (`id`)
) DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2.3 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 主键，单行记录（始终 UPDATE，不新增） |
| `title` | VARCHAR(256) | 手册标题 |
| `html_version` | INTEGER | HTML 版本号，从 1 开始，仅 HTML 变化时递增 |
| `md_version` | INTEGER | Markdown 版本号，从 1 开始，仅 MD 变化时递增 |
| `created_at` | DATETIME(3) | 创建时间 |
| `updated_at` | DATETIME(3) | 更新时间 |

### 2.4 版本策略

- **独立版本**：HTML 和 Markdown 各自有版本号，保存时比较新旧内容，仅对变化的格式升版本
- **内容不变不升版本**：如果只改了 Markdown，`html_version` 不变，`md_version` +1
- 文件系统按各自版本号保留最近 10 个文件，超出自动清理
- 单文件最大 10MB

## 三、API 接口设计

### 3.1 管理台接口

#### 获取最新手册（用于编辑）

```
GET /workmate/console/manual
```

**响应**：

```json
{
  "code": 0,
  "data": {
    "id": 1,
    "title": "使用手册",
    "htmlContent": "<h1>欢迎</h1><p>图文内容...</p>",
    "markdownContent": "# 欢迎\n\nMarkdown 内容...",
    "htmlVersion": 3,
    "mdVersion": 5,
    "updatedAt": "2026-06-12T10:30:00"
  }
}
```

> 无记录时返回 `htmlContent: ""`, `markdownContent: ""`, `htmlVersion: 0`, `mdVersion: 0`

#### 保存手册

```
POST /workmate/console/manual
Content-Type: application/json

{
  "title": "使用手册",
  "htmlContent": "<h1>欢迎</h1>...",
  "markdownContent": "# 欢迎\n..."
}
```

**响应**：

```json
{
  "code": 0,
  "message": "保存成功",
  "data": {
    "id": 1,
    "title": "使用手册",
    "htmlVersion": 3,
    "mdVersion": 5,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

### 3.2 客户端接口

#### Markdown 手册（前端渲染）

```
GET /workmate/manual?version={版本号}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `version` | Integer | 否 | 客户端当前缓存版本号 |

**响应（需要更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": true,
    "title": "使用手册",
    "content": "# 欢迎\n\nMarkdown 内容...",
    "version": 5,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

> `content` 为 Markdown 纯文本，**不含 base64 图片**，客户端渲染流畅。
> `version` 为 `md_version`，客户端下次请求时传入此值以增量更新。

**响应（无需更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": false,
    "message": "已是最新版本"
  }
}
```

#### HTML 图文手册（新窗口打开）

```
GET /workmate/manual/html?version={版本号}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `version` | Integer | 否 | 客户端当前缓存版本号 |

**响应（需要更新）**：

```json
{
  "code": 0,
  "data": {
    "needUpdate": true,
    "title": "使用手册",
    "content": "<h1>欢迎</h1><img src=\"data:image/png;base64,...\">...",
    "version": 3,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

> `content` 为完整 HTML，含 base64 图片，新窗口中的 `document.body.innerHTML = content` 即可渲染。
> `version` 为 `html_version`，客户端下次请求时传入此值以增量更新。

### 3.3 版本比较逻辑

```
客户端 Markdown 接口：
  clientVersion vs 服务端 md_version
    ├── version == null       → 返回最新手册
    ├── version < md_version  → 返回最新手册
    └── version >= md_version → 返回 needUpdate: false

客户端 HTML 接口：
  clientVersion vs 服务端 html_version
    ├── version == null        → 返回最新手册
    ├── version < html_version → 返回最新手册
    └── version >= html_version → 返回 needUpdate: false
```

### 3.4 鉴权说明

| 接口 | 鉴权 |
|------|------|
| `GET/POST /workmate/console/manual` | `AdminWhitelistInterceptor`（管理台白名单） |
| `GET /workmate/manual` | 无需鉴权 |
| `GET /workmate/manual/html` | 无需鉴权 |

### 4.2 ManualService 核心逻辑

```java

    }
    public Optional<Manual> getManualForClientHtml(Integer clientVersion) {
        // 比较 clientVersion vs html_version
    }

    public String readHtml(int version) { ... }
    public String readMarkdown(int version) { ... }
}
```

## 六、Word 粘贴支持

（仅右侧富文本编辑器 Tab）

### 6.1 粘贴流程

```
用户 Ctrl+C (Word) → 剪贴板含 HTML + 图片文件
        ↓
handlePaste 拦截
  ├── items 来源（截图）
  ├── files 来源（Word 图文混排）
  └── 去重：name:size 作为 key
        ↓
图片转 base64 → 替换 HTML 中 img src
        ↓
cleanWordHtml 清理（mso-* / 空span / font标签）
        ↓
insertContent 插入编辑器
```

### 6.2 清理规则

| Word 元素 | 处理方式 |
|-----------|----------|
| `mso-*` / `o:*` / `w:*` 属性 | 移除 |
| `style` 中的 `mso-` 声明 | 过滤 |
| 空 `<span>` | 删除 |
| `<font>` 标签 | 替换为纯文本 |
| `v:imagedata` | 转为 `<img>` |

## 七、客户端接入指南

### 7.1 Markdown 渲染（推荐）

首次加载不传 `version`，后续带缓存版本号增量更新：

```
GET /workmate/manual?version=3
```

- `needUpdate: true` → 拿到 Markdown 文本，客户端渲染，**缓存返回的 `version` 值供下次请求用**
- `needUpdate: false` → 无需更新

### 7.2 HTML 图文查看

Markdown 内容中提供「查看图文版」链接，点击后请求 HTML 接口：

```
GET /workmate/manual/html
```

返回的 `content` 为完整 HTML，客户端用 WebView 加载：

```html
<!DOCTYPE html><html><head><meta charset="utf-8">
<style>body{font-size:16px;line-height:1.8;padding:16px} img{max-width:100%}</style>
</head><body>{{content}}</body></html>
```

### 7.3 渲染建议样式

```css
body { font-size: 16px; line-height: 1.8; color: #333; padding: 16px; }
h1 { font-size: 28px; font-weight: 700; margin: 24px 0 16px; }
h2 { font-size: 24px; font-weight: 600; margin: 20px 0 12px; }
h3 { font-size: 20px; font-weight: 600; margin: 16px 0 8px; }
img { max-width: 100%; height: auto; border-radius: 4px; }
table { border-collapse: collapse; width: 100%; }
th, td { border: 1px solid #d9d9d9; padding: 8px 12px; }
pre { background: #1e1e1e; color: #d4d4d4; padding: 16px; border-radius: 6px; overflow-x: auto; }
code { background: #f5f5f5; padding: 2px 6px; border-radius: 3px; color: #d4380d; }
blockquote { border-left: 4px solid #1677ff; padding: 12px 16px; margin: 12px 0; background: #f0f5ff; }
```

## 八、版本管理策略

- HTML 和 Markdown **独立版本号**，各自从 1 开始，互不干扰
- 保存时比较新旧内容，**仅对变化的格式升版本**
- 场景示例：

| 操作 | html_version | md_version | 写入文件 |
|------|:-----------:|:---------:|---------|
| 首次保存 | 1 | 1 | v1.html, v1.md |
| 只改 HTML | 2 | 1 | v2.html |
| 只改 MD | 2 | 2 | v2.md |
| 只改 MD | 2 | 3 | v3.md |
| 两者都改 | 3 | 4 | v3.html, v4.md |

- 文件系统按各自版本号保留最近 10 个文件，超出自动清理
- 单文件最大 10MB`

## 九、数据库恢复指南

### 9.1 当前状态

数据库迁移已合并为单个 `V8__manuals.sql`，旧文件（V8/V9/V10）已删除。

### 9.2 如果旧环境已执行过旧迁移

执行以下 SQL 回退，然后重新启动让 Flyway 执行新 V8：

```sql
-- 1. 清空 Flyway 迁移记录（替换为实际数据库名）
DELETE FROM `your_db`.`flyway_schema_history` WHERE version IN ('8', '9', '10');

-- 2. 删除旧表
DROP TABLE IF EXISTS `manuals`;

-- 3. 删除残留文件（可选）
-- rm -rf /app/workmate/manual/
```

> 执行后重启服务，Flyway 将自动执行新的 `V8__manuals.sql` 建表。
> 也可直接执行 `db/migration/V8__manuals_recovery.sql` 作为恢复脚本。

### 9.3 首次部署

无需任何恢复操作，Flyway 自动创建表。

## 一、需求概述

### 1.1 功能目标

为 WorkMate 客户端提供统一的使用手册管理能力：

1. **管理台编辑**：支持富文本编辑（标题、图片、表格、代码块等），支持从 Word 直接粘贴图文混排内容，保存后自动更新版本号
2. **客户端查询**：客户端按版本号查询手册，服务端比较版本后按需返回最新内容（Markdown 格式）
3. **版本管理**：每次保存生成新版本，旧版本保留在数据库中
4. **双格式存储**：同时存储 HTML（管理台编辑）和 Markdown（客户端渲染）

### 1.2 核心流程

```
┌──────────────────────────┐        ┌──────────────────────────┐
│      管理台 (Admin)        │        │      客户端 (Client)       │
├──────────────────────────┤        ├──────────────────────────┤
│                          │        │                          │
│  Tiptap 编辑器            │        │                          │
│  ├─ 手动编辑              │        │                          │
│  ├─ Word 粘贴(图文混排)    │        │                          │
│  └─ 截图粘贴              │        │                          │
│         ↓                │        │                          │
│  保存时 turndown 转 MD    │        │                          │
│         ↓                │        │                          │
│  POST /console/manual    │        │  GET /manual?version     │
│  → {title,content,md}    │        │  ← Markdown + 版本号      │
│                          │        │                          │
└──────────────────────────┘        └──────────────────────────┘
              │                              │
              └────────────┬─────────────────┘
                           ▼
                   ┌──────────────┐
                   │   manuals 表  │
                   │ HTML + MD 双存│
                   └──────────────┘
```

## 二、数据库设计

### 2.1 表结构

```sql
-- V8__manuals.sql（建表）
CREATE TABLE `manuals` (
    `id`               INTEGER      NOT NULL AUTO_INCREMENT,
    `title`            VARCHAR(256) NOT NULL DEFAULT '使用手册',
    `content`          MEDIUMTEXT   NOT NULL COMMENT 'HTML 富文本内容',
    `content_markdown` MEDIUMTEXT   NULL     COMMENT 'Markdown 格式内容',
    `version`          INTEGER      NOT NULL DEFAULT 1 COMMENT '版本号，每次保存 +1',
    `created_at`       DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    `updated_at`       DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3) ON UPDATE CURRENT_TIMESTAMP(3),
    PRIMARY KEY (`id`)
) DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- V9__manual_markdown.sql（追加 Markdown 字段）
ALTER TABLE `manuals`
  ADD COLUMN `content_markdown` MEDIUMTEXT NULL COMMENT 'Markdown 格式内容' AFTER `content`;
```

### 2.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER | 主键，自增 |
| `title` | VARCHAR(256) | 手册标题，默认"使用手册" |
| `content` | MEDIUMTEXT | 富文本 HTML 内容，管理台编辑使用 |
| `content_markdown` | MEDIUMTEXT | Markdown 格式内容，客户端接口返回 |
| `version` | INTEGER | 版本号，从 1 开始，每次保存 +1 |
| `created_at` | DATETIME(3) | 创建时间 |
| `updated_at` | DATETIME(3) | 更新时间，自动更新 |

### 2.3 版本策略

- 每次保存生成一条**新记录**（非 UPDATE 旧记录），保留完整历史
- 版本号从 1 开始递增
- 查询最新版本：`SELECT * FROM manuals ORDER BY version DESC LIMIT 1`

## 三、API 接口设计

### 3.1 管理台接口

#### 获取最新手册

```
GET /workmate/console/manual
```

**响应**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 3,
    "title": "使用手册",
    "content": "<h1>欢迎使用 WorkMate</h1><p>...</p>",
    "version": 3,
    "updatedAt": "2026-06-12T10:30:00"
  }
}
```

> 数据库无记录时返回 `content: ""`, `version: 0`

#### 保存手册

```
POST /workmate/console/manual
Content-Type: application/json

{
  "title": "使用手册",
  "content": "<h1>欢迎使用 WorkMate</h1><p>这是更新后的内容...</p>",
  "contentMarkdown": "# 欢迎使用 WorkMate\n\n这是更新后的内容..."
}
```

**响应**：

```json
{
  "code": 0,
  "message": "保存成功",
  "data": {
    "id": 4,
    "title": "使用手册",
    "version": 4,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

### 3.2 客户端接口

#### 查询手册

```
GET /workmate/manual?version={客户端当前版本号}
```

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| `version` | Integer | 否 | 客户端当前缓存的手册版本号 |

**响应（需要更新）**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "needUpdate": true,
    "title": "使用手册",
    "content": "# 欢迎使用 WorkMate\n\n这是 Markdown 内容...\n\n![图片](data:image/png;base64,...)",
    "version": 4,
    "updatedAt": "2026-06-12T11:00:00"
  }
}
```

> `content` 字段返回 Markdown 格式，图片以 base64 data URI 内嵌

**响应（无需更新）**：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "needUpdate": false,
    "message": "已是最新版本"
  }
}
```

### 3.3 版本比较逻辑

```
客户端传入 version → 服务端查询最新版本号 latestVersion
  ├── version == null       → 返回最新手册
  ├── version < latestVersion → 返回最新手册
  └── version >= latestVersion → 返回 needUpdate: false
```

### 3.4 鉴权说明

| 接口 | 鉴权 |
|------|------|
| `GET/POST /workmate/console/manual` | 需通过 `AdminWhitelistInterceptor`（管理台白名单） |
| `GET /workmate/manual` | 无需鉴权（已加入 `UserIdInterceptor` 排除列表） |

### 4.2 ManualService 核心逻辑

```java
@Service
public class ManualService {

    /**
     * 保存手册：每次版本号 +1，同时存储 HTML 和 Markdown
     */
    @Transactional
    public Manual save(String title, String content, String contentMarkdown) {
        int newVersion = manualMapper.findLatest()
                .map(m -> m.getVersion() + 1)
                .orElse(1);
        Manual manual = Manual.builder()
                .title(title)
                .content(content)
                .contentMarkdown(contentMarkdown)
                .version(newVersion)
                .build();
        manualMapper.insert(manual);
        return manual;
    }

    /**
     * 客户端查询：按版本号比较，决定是否返回新内容
     */
    public Optional<Manual> getManualForClient(Integer clientVersion) {
        Optional<Manual> latest = manualMapper.findLatest();
        if (latest.isEmpty()) return Optional.empty();
        Manual m = latest.get();
        if (clientVersion == null || clientVersion < m.getVersion()) {
            return Optional.of(m);
        }
        return Optional.empty();
    }
}
```

## 六、HTML → Markdown 转换

### 6.1 转换时机

管理台点击「保存」时，前端自动将 Tiptap 输出的 HTML 转为 Markdown，随请求一起发送到后端。

### 6.2 转换工具

使用 `turndown` 库，配置如下：

```typescript
const turndown = new TurndownService({
  headingStyle: 'atx',      // # 标题格式
  codeBlockStyle: 'fenced', // ``` 代码块
  emDelimiter: '*',         // *斜体*
})

// 自定义图片规则：保留 base64 src
turndown.addRule('image', {
  filter: 'img',
  replacement: (_content, node) => {
    const img = node as HTMLImageElement
    const alt = img.alt || ''
    const src = img.getAttribute('src') || ''
    return `![${alt}](${src})`
  },
})
```

### 6.3 转换示例

| HTML | Markdown |
|------|----------|
| `<h1>标题</h1>` | `# 标题` |
| `<p><strong>加粗</strong></p>` | `**加粗**` |
| `<img src="data:image/png;base64,...">` | `![](data:image/png;base64,...)` |
| `<table>...</table>` | 表格语法 |
| `<pre><code>...</code></pre>` | ` ``` ... ``` ` |

### 6.4 存储策略

| 字段 | 格式 | 用途 |
|------|------|------|
| `content` | HTML | 管理台加载回编辑器，保留完整格式 |
| `content_markdown` | Markdown | 客户端接口返回，轻量、易渲染 |

## 七、Word 粘贴支持

### 7.1 粘贴流程

```
用户 Ctrl+C (Word) → 剪贴板含 HTML + 图片文件
        ↓
handlePaste 拦截
        ↓
  ├── 从 clipboardData.items 读图片（截图场景）
  ├── 从 clipboardData.files 读图片（Word 图文混排场景）
  └── 图片转 base64 → 替换 HTML 中 img src
        ↓
cleanWordHtml 清理
  ├── 移除 mso-* / o:* / w:* 属性
  ├── 清理 style 中的 mso- 声明
  ├── 删除空 <span>
  └── <font> 转纯文本
        ↓
transformPastedHTML 自动清理（纯文本/HTML 粘贴）
        ↓
insertContent 插入编辑器
```

### 7.2 关键实现

```typescript
editorProps: {
  // 纯文本/HTML 粘贴：自动清理 Word 样式
  transformPastedHTML: (html: string) => cleanWordHtml(html),

  // 图片粘贴：截图 / Word 图文混排
  handlePaste: (view, event) => {
    const cd = event.clipboardData

    // 两个来源都要检查
    // 1. cd.items — 截图、单独复制图片
    // 2. cd.files — Word 全选复制图文混排

    // 纯图片 → 直接插入 <img>
    // HTML + 图片 → 替换 img src 为 base64 → insertContent
    // 纯 HTML/文本 → return false，走默认流程
  },
}
```

### 7.3 清理规则

| Word 元素 | 处理方式 |
|-----------|----------|
| `mso-*` 属性 | 全部移除 |
| `o:*` / `w:*` 命名空间 | 全部移除 |
| `style` 中的 `mso-` 声明 | 过滤掉，保留正常 CSS |
| 空 `<span>` | 删除 |
| `<font>` 标签 | 替换为纯文本 |
| `v:imagedata` | 转为 `<img>` 标签 |
| `class` 属性 | 移除（Word 自带样式类） |

## 八、客户端接入指南

### 8.1 首次加载

客户端启动时，不传 `version` 参数（或传 `0`），获取最新手册：

```
GET /workmate/manual
```

### 8.2 增量更新

客户端缓存上次获取的 `version`，后续请求携带：

```
GET /workmate/manual?version=3
```

- 返回 `needUpdate: true` → 更新本地缓存，渲染新内容
- 返回 `needUpdate: false` → 无需更新，继续使用缓存

### 8.3 渲染建议

服务端返回的 `content` 为 **Markdown 格式**，客户端可使用 Markdown 渲染器展示。推荐使用 `react-markdown`（Web）或平台原生 Markdown 组件。

图片以 base64 data URI 内嵌在 Markdown 中，无需额外下载：

```markdown
![截图说明](data:image/png;base64,iVBORw0KGgo...)
```

## 九、版本管理策略

### 9.1 版本号规则

- 初始版本号为 `1`
- 每次在管理台点击「保存」，版本号 +1
- 版本号是全局递增的整数，不与环境关联

### 9.2 历史版本

- 每次保存插入新记录，旧记录保留不删除
- 可通过 `manuals` 表按 `version DESC` 查看所有历史版本
- 如需回退到历史版本，可手动将旧版本内容复制到编辑器重新保存

### 9.3 扩展建议

后续可扩展的功能：

- 管理台增加「版本历史」列表，支持预览和回退
- 支持多语言手册（增加 `language` 字段）
- 支持草稿/发布状态（增加 `status` 字段）
- 图片上传到服务器独立存储（减少数据库体积）
- 客户端支持增量 diff 更新（减少传输量）

# 专家团布局修改方案

> 编辑时间：2026年6月15日

---

## 一、现状分析

### 1.1 当前架构问题

| 问题 | 现状 | 影响 |
|------|------|------|
| 专家团藏在设置里 | 用户需进入 设置 → 专家团 才能看到 | 发现性极差，新用户不知道专家团存在 |
| 设置页与使用场景割裂 | 在 Agent 模式工作时要回设置页召唤专家 | 操作路径长：Agent 会话 → 设置 → 找专家 → 召唤 → 回到会话 |
| 无快速访问机制 | 39 个专家团全部平铺，无收藏/常用标记 | 用户高频使用 3-5 个专家，每次都要从头翻 |
| Chat 模式也显示 | 设置页的专家团在 Chat 模式下也能看到 | Chat 模式下专家团无意义（专家团仅用于 Agent 模式） |
| 无关注/收藏能力 | WorkBuddy 也无此功能 | 用户无法快速筛选常用专家 |

### 1.2 WorkBuddy 的专家团布局（参考）

WorkBuddy 将专家团放在**侧边栏**中，作为独立入口：

```
WorkBuddy 侧边栏布局：
├── 🔍 搜索
├── 📁 文件浏览器
├── 🤖 Agent 会话列表
├── ⚡ Automation（自动任务）
├── 👥 专家团 ← 侧边栏直接可见
│   ├── ai-engineer
│   ├── backend-architect
│   ├── data-analysis
│   └── ...
└── ⚙️ 设置
```

优点：专家团与会话在同一视觉平面，召唤路径短。

### 1.3 业界参考

| 产品 | 设计特点 | 可借鉴点 |
|------|---------|---------|
| **Cursor 2.0/3.0** | Agent-centric sidebar，agents/plans/runs 为一级对象 | 侧边栏按功能分区，Agent 是核心入口 |
| **VS Code Extensions** | 侧边栏独立视图，支持搜索/分类/已安装筛选 | 搜索+筛选+列表的标准模式 |
| **GitHub Copilot** | Agent picker 下拉菜单 + 快速切换 | 快捷选择器减少操作步数 |
| **Linear** | Project favorites with star toggle | 星标/关注机制简单直观 |
| **Figma** | 团队库 + 收藏夹双层结构 | 收藏夹与全部列表分离 |

## 二、整体布局方案

### 2.1 核心原则

1. **Agent 模式专属**：专家团仅在 Agent 模式下可见，Chat 模式隐藏
2. **侧边栏就近访问**：从设置页迁出，嵌入左侧边栏，紧贴工作流
3. **关注优先**：关注的专家团置顶，减少查找成本
4. **渐进式展现**：侧边栏紧凑展示 → 展开完整面板 → 详情弹窗，三层递进

### 2.2 目标布局

```
LeftSidebar (Agent 模式)
├── 🔍 搜索
├── 🤖 Agent 会话列表（日期分组）
│   ├── 今天
│   ├── 昨天
│   └── 更早
├── ⏰ 自动任务                       ← 现有，位置不变
│
├── 👥 专家团  [展开/收起]  [管理]    ← 新增，Agent 模式专属
│   ├── ⭐ 已关注 (3)
│   │   ├── 🏗️ 后端架构师      ⭐ ●  →
│   │   ├── 📊 数据分析专家     ⭐ ●  →
│   │   └── 🔍 Code Review     ⭐ ●  →
│   ├── 🕐 最近使用 (5)
│   │   ├── 🧠 AI工程师         ☆ ●  →
│   │   └── ...
│   └── 📋 全部专家团 (39)
│       ├── ...
│       └── ...
│
├── ───────── 分隔线 ─────────
│
└── 📌 置顶会话
```

### 2.3 Chat 模式下的布局

Chat 模式下专家团区域**完全不渲染**：

```

```

## 四、关注（Follow/Favorite）机制

### 4.1 设计思路

借鉴 **GitHub Stars** + **Linear Favorites** 模式：

- 每个专家团有一个星标按钮（☆/⭐）
- 关注的专家团置顶显示
- 关注状态跨会话/工作区持久化
- 支持在侧边栏和管理面板两处切换关注状态（双向同步）

### 4.2 存储方案

```typescript
// 关注列表存储在本地，不在插件系统内（用户偏好而非项目配置）
interface FollowedExpertsState {
  // key: expertGroupId（如 "backend-architect"）
  // value: 关注时间戳
  followed: Record<string, number>
}

// 存储位置：~/.proma/user-preferences.json 或 electron-store
// 与工作区无关，跟随用户账号
```

使用 `electron-store` 或 `atomWithStorage`（localStorage 兜底）：

```typescript
const followedExpertGroupsAtom = atomWithStorage<Record<string, number>>(
  'followed-expert-groups',
  {}
)
```

### 4.3 UI 交互

| 操作 | 位置 | 行为 |
|------|------|------|
| 点击 ☆ | 侧边栏条目 / 管理面板卡片 | 切换为 ⭐，加入置顶区 |
| 点击 ⭐ | 侧边栏条目 / 管理面板卡片 | 切换为 ☆，从置顶区移除 |
| 关注 | 侧边栏 hover 时显示 | 即时响应，toast 提示"已关注" |
| 取消关注 | 侧边栏 hover 时显示 | 即时响应，无 toast（减少打扰） |

### 4.4 关注列表排序

已关注专家团按**关注时间倒序**排列（最近关注的在前），方便用户快速找到刚关注的专家。

## 六、交互流程

### 6.1 侧边栏快速召唤

```
用户在 Agent 会话中
  │
  ├─→ 侧边栏找到关注的专家团
  │     └─→ 点击 [召唤] 按钮
  │           └─→ 创建新专家会话，自动跳转
  │
  ├─→ 侧边栏浏览全部专家团
  │     └─→ hover 条目 → 点击 [召唤]
  │           └─→ 创建新专家会话
  │
  └─→ 点击 [管理专家团] → 进入管理面板
        └─→ 搜索/筛选 → 点击 [召唤]
              └─→ 创建新专家会话
```

### 6.2 AgentView 工具栏召唤

保留现有 `ExpertSummonButton` 在 AgentView 工具栏：

```
AgentView 顶部工具栏：
[模型选择器] [思考模式] [显示选项] [召唤专家▼] [设置]
```

点击"召唤专家"打开 `ExpertGroupPicker` 弹窗（保持现有逻辑），弹窗中的卡片也新增关注星标。

### 6.3 关注/取消关注

```
用户 hover 专家条目
  └─→ 显示 ☆ 星标
        └─→ 点击 ☆ → 切换为 ⭐
              ├─→ toast "已关注 {专家名}"
              ├─→ 条目移至"已关注"分组顶部
              └─→ 持久化到本地存储

已关注条目 hover
  └─→ 显示 ⭐（实心）
        └─→ 点击 ⭐ → 切换为 ☆
              ├─→ 条目从"已关注"分组移除
              └─→ 持久化更新
```

### 7.3 新增 atom

```typescript
// renderer/atoms/expert-group-follow.ts
import { atomWithStorage } from 'jotai/utils'

/** 关注的专家团 ID → 关注时间戳 */
export const followedExpertGroupsAtom = atomWithStorage<Record<string, number>>(
  'followed-expert-groups',
  {}
)

/** 切换关注状态 */
export const toggleFollowExpertGroupAtom = atom(
  null,
  (get, set, expertGroupId: string) => {
    const followed = { ...get(followedExpertGroupsAtom) }
    if (followed[expertGroupId]) {
      delete followed[expertGroupId]
    } else {
      followed[expertGroupId] = Date.now()
    }
    set(followedExpertGroupsAtom, followed)
  }
)
```

### 7.4 排序逻辑

```typescript
function sortExpertGroups(
  groups: AgentExpertGroupInfo[],
  followed: Record<string, number>
): AgentExpertGroupInfo[] {
  return [...groups].sort((a, b) => {
    const aFollowed = followed[a.id] ?? 0
    const bFollowed = followed[b.id] ?? 0
    // 1. 已关注优先
    if (aFollowed && !bFollowed) return -1
    if (!aFollowed && bFollowed) return 1
    // 2. 同关注状态下，最近关注的在前
    if (aFollowed && bFollowed) return bFollowed - aFollowed
    // 3. 未关注的按名称排序
    return a.name.localeCompare(b.name, 'zh')
  })
}
```

## 九、边界情况

### 9.1 零专家团

当没有任何专家团时，侧边栏区域显示空态：

```
👥 专家团                    0
─────────────────────────────
  暂无专家团
  [导入 WorkBuddy] [上传 Zip]
```

### 9.2 全部不可用

当所有专家团都不可用时，侧边栏仍显示列表，但召唤按钮 disabled。

### 9.3 模式切换

从 Agent 模式切换到 Chat 模式时：
- 侧边栏专家团区域立即隐藏
- 关注状态保留（不丢失）
- 切回 Agent 模式时恢复显示

### 9.4 关注列表持久化

- 存储键名：`followed-expert-groups`
- 存储位置：`~/.proma/user-preferences.json`
- 工作区隔离：否（全局偏好，跨工作区共享）
- 账号隔离：跟随本地用户 profile

### 9.5 已关注专家被卸载

如果关注的专家团对应的插件被卸载：
- 关注记录保留（不清除）
- 列表中不显示（因为无此专家）
- 重新安装后自动恢复关注状态

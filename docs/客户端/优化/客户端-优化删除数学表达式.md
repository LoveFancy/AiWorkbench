# 客户端优化 - 删除数学表达式（$ / $$）渲染 --
# 未实施

## 背景

当前客户端在消息渲染和富文本编辑器中集成了数学公式支持：
- **消息渲染**：`react-markdown` + `remark-math` + `rehype-katex` 将 `$...$` / `$$...$$` 渲染为排版数学公式
- **富文本编辑器**：`markdown-it` 自定义 `addMathSupport()` 识别 `$...$` / `$$...$$` 并渲染为 math-block/inline 元素
- **LaTeX 规范化**：`normalizeLatexDelimiters()` 将 `\(...\)` / `\[...\]` 转换为 `$...$` / `$$...$$`

移除这些能力以简化依赖和维护成本。

## 变更清单

### 1. 修改 `markdown-rich-text.ts`

- 删除 `addMathSupport()` 函数及调用
- 删除 html→markdown 反向转换中的 `math-inline` / `math-block` 还原逻辑
- 清理 `PREVIEW_BLOCK_RE` 中的 `math-block` 引用

### 2. 修改 `message.tsx`

- 移除 `remark-math` / `rehype-katex` / `normalizeLatexDelimiters` import
- `REMARK_PLUGINS` 改为仅保留 `remarkGfm`，`REHYPE_PLUGINS` 改为空数组
- 移除 `normalizeLatexDelimiters()` 调用

### 3. 修改 `reasoning.tsx`

- 移除 `remark-math` / `rehype-katex` / `normalizeLatexDelimiters` import
- Markdown 组件 props 移除相关插件，移除 `normalizeLatexDelimiters()` 包裹

### 4. 删除 `normalize-latex.ts`

- 此文件仅被 message.tsx 和 reasoning.tsx 引用，移除后无其他依赖

### 5. 移除 npm 依赖

- `remark-math`（`^6`）
- `rehype-katex`（`^7`）

## 影响分析

| 层面 | 变更前 | 变更后 |
|------|--------|--------|
| 输入框 | `$x^2$` 以公式样式预览 | `$x^2$` 显示为纯文本 |
| AI 消息渲染 | 数学公式排版渲染 | `$...$` / `$$...$$` 原样显示为文本 |
| 推理内容渲染 | 同上 | 同上 |
| 模型调用 | 无影响 | — |
| Agent 执行 | 无影响 | — |

## 不影响的功能

- 基本对话和模型调用流程不受影响
- Agent 执行逻辑不受影响
- Markdown 其他特性（GFM、代码高亮等）不受影响
- `$` 符号本身仍可在输入框中正常输入和发送

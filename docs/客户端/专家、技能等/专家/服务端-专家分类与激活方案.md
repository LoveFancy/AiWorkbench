# 服务端：专家分类与激活方案

编辑时间：2026年6月18日

> **关联文档**：
> - [专家团对workmate接口](./专家团对workmate接口.md)
> - [服务端-专家团管理后台方案](./服务端-专家团管理后台方案.md)
>
> 本方案在现有专家团功能基础上，增加**多分类**支持和**激活状态**管理。分类数据来源于 `expert-groups/{id}.json` 中的 `categories` 字段（数组）。

---

## 一、现状分析

### 1.1 现有分类机制

| 项目 | 现状 |
|------|------|
| 数据库列 | `expert_groups.category` — 单个 VARCHAR(64) |
| 数据来源 | `plugin.json` → `keywords` 数组的第二项（跳过首项 `expert-group`） |
| 取值逻辑 | `extractCategory()` 只取一个值 |
| 管理台筛选 | 基于 `category` 单值等值匹配 |
| 管理台前端 | 从列表数据中动态提取去重后作为下拉选项 |

### 1.2 现有激活机制

| 项目 | 现状 |
|------|------|
| 数据库列 | `expert_groups.is_active` BOOLEAN NOT NULL DEFAULT TRUE（V11 已有） |
| workmate 客户端列表 | `findAllActive()` — 只查 `is_active = TRUE` |
| workmate 客户端详情 | `getClientDetail()` — 校验 `is_active = true`，否则 404 |
| 管理台 | 列表支持按 `isActive` 筛选，toggle 切换接口已就绪 |

> **结论**：激活字段 `is_active` 已存在且功能完整，需求 #4（未激活的不返回给 workmate）**当前已满足**，本方案不再额外改动。

### 1.3 需要变更的点

| 需求 | 当前状态 | 需要做什么 |
|------|---------|-----------|
| 多分类 | 仅支持单分类（`category` VARCHAR） | 新建关联表 `expert_group_categories`，读取 JSON 的 `categories` 数组 |
| workmate 分类筛选 | 客户端接口不支持分类筛选 | `GET /workmate/expert-groups` 增加 `?category=` 参数 |
| workmate 返回分类 | DTO 无 categories 字段 | `ExpertGroupClientDTO` 新增 `categories: List<String>` |
| 管理台分类接口 | 无分类列表接口 | 新增 `GET /workmate/console/expert-groups/categories` |
| 管理台分类筛选 | 基于单值 `category` 列 | 改为基于关联表的多分类筛选 |
| 管理台编辑分类 | 编辑表单仅支持单个分类 | 改为多分类编辑（标签输入） |
| 激活字段 | ✅ 已存在 `is_active` | 无需变更 |

## 二、数据库设计

### 2.1 新增表：`expert_group_categories`

```sql
-- V14__expert_group_categories.sql
CREATE TABLE `expert_group_categories` (
    `id`               BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `expert_group_id`  VARCHAR(64)  NOT NULL COMMENT '专家团 ID',
    `category_name`    VARCHAR(64)  NOT NULL COMMENT '分类名称',
    `created_at`       DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_group_category` (`expert_group_id`, `category_name`),
    INDEX `idx_expert_group_id` (`expert_group_id`),
    INDEX `idx_category_name` (`category_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='专家团多分类关联表';
```

### 2.2 旧 `category` 列处理策略

**废弃 `expert_groups.category` 列**，分类数据完全由 `expert_group_categories` 表承载。

| 场景 | 处理方式 |
|------|---------|
| 新旧数据 | 分类数据统一走 `expert_group_categories` 表 |
| 管理台筛选 | 关联 `expert_group_categories` 表查询 |
| workmate 返回 | DTO 只保留 `categories` 字段，移除旧的 `category` 字段 |
| V14 迁移 | 先迁移旧数据到 `expert_group_categories`，再 DROP `category` 列 |
| ExpertGroup 实体 | 移除 `category` 字段 |
| ResultMap | 移除 `category` 映射 |

### 2.3 表关系

```
expert_groups (1) ──── (N) expert_group_categories
     id  ←──FK──  expert_group_id
```

## 三、JSON 数据读取变更

### 3.1 `expert-groups/{id}.json` 新增字段

```json
{
  "id": "ai-engineer",
  "name": "深网网",
  "categories": ["开发", "AI", "模型部署"],
  ...
}
```

### 3.2 字段提取逻辑

| 步骤 | 操作 |
|------|------|
| 1 | 读取 `expert-groups/{id}.json` |
| 2 | 解析 `$.categories` 数组 |
| 3 | 校验：必须是字符串数组，空数组则跳过 |
| 4 | 写入 `expert_group_categories` 表（先 DELETE 旧分类，再逐条 INSERT 新分类） |

### 3.3 上传覆盖模式：全量刷新

上传相同 ID 的专家团时，所有数据都要刷新：

```
processSingle() 步骤 5「合并字段」之后，覆盖模式下：
  1. DELETE FROM expert_group_categories WHERE expert_group_id = ?
  2. 遍历 categories 数组，逐条 INSERT INTO expert_group_categories
  3. ExpertGroup 实体不再设置 category 字段
```

> 覆盖模式下，`expert_groups` 表走 `existingOrInsert()` → `update()`，关联表分类数据**全量删旧插新**，确保与 JSON 完全一致。

## 四、API 设计

### 4.1 管理台接口变更（/workmate/console）

#### 新增接口：获取分类列表

```
GET /workmate/console/expert-groups/categories

说明：返回所有已使用的分类名称列表，供管理台前端下拉筛选使用

响应：
{
  "code": 0,
  "data": {
    "categories": ["开发", "AI", "模型部署", "内容创作", "测试"]
  }
}
```

数据来源：`SELECT DISTINCT category_name FROM expert_group_categories ORDER BY category_name`

#### 变更接口：分页列表

```
GET /workmate/console/expert-groups
    ?page=1
    &pageSize=20
    &keyword=代码
    &category=AI          ← 变更：现在基于 expert_group_categories 表做多分类筛选
    &isActive=true

说明：
  - category 筛选改为关联查询 expert_group_categories 表
  - 如果一个专家团有多个分类，任一匹配即返回（但去重）
  - 不传 category 则不做分类筛选

响应：每个 items[i] 新增 categories 字段
{
  "code": 0,
  "data": {
    "total": 100,
    "page": 1,
    "pageSize": 20,
    "items": [
      {
      "id": "code-expert",
      "name": "代码专家团",
      "categories": ["开发", "AI"],   // 多分类
      ...
      }
    ]
  }
}
```

SQL 变更（`ExpertGroupMapper.xml` → `findPage`）：

```sql
SELECT DISTINCT eg.* FROM expert_groups eg
LEFT JOIN expert_group_categories egc ON eg.id = egc.expert_group_id
<where>
    <if test="keyword != null and keyword != ''">
        AND (eg.name LIKE CONCAT('%', #{keyword}, '%') OR eg.description LIKE CONCAT('%', #{keyword}, '%'))
    </if>
    <if test="category != null and category != ''">
        AND egc.category_name = #{category}
    </if>
    <if test="isActive != null">
        AND eg.is_active = #{isActive}
    </if>
</where>
ORDER BY eg.updated_at DESC
```

#### 变更接口：更新元数据

```
PUT /workmate/console/expert-groups/:id
Content-Type: application/json

Body（新增 categories 字段）:
{
  "name": "新名称",
  "description": "新描述",
  "categories": ["开发", "AI", "工具"],   // 新增：多分类编辑
  "tags": ["new-tag"],
  "iconUrl": "https://..."
}

处理逻辑：
  - 如果传了 categories 字段：
    1. DELETE FROM expert_group_categories WHERE expert_group_id = :id
    2. 逐条 INSERT 新分类
  - 如果不传 categories 字段：不做分类变更
```

### 4.2 客户端接口变更（/workmate）

#### 变更接口：专家团列表

```

  - 传 category 则筛选包含该分类的专家团（关联 expert_group_categories 表）

响应（每个 items[i] 新增 categories 字段）：

      {
        "id": "ai-engineer",

```

```

## 八、数据迁移

V14 脚本执行时：

```sql
-- V14__expert_group_categories.sql

-- 1. 建表
CREATE TABLE `expert_group_categories` (
    `id`               BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
    `expert_group_id`  VARCHAR(64)  NOT NULL COMMENT '专家团 ID',
    `category_name`    VARCHAR(64)  NOT NULL COMMENT '分类名称',
    `created_at`       DATETIME(3)  NOT NULL DEFAULT CURRENT_TIMESTAMP(3),
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_group_category` (`expert_group_id`, `category_name`),
    INDEX `idx_expert_group_id` (`expert_group_id`),
    INDEX `idx_category_name` (`category_name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='专家团多分类关联表';

-- 2. 将旧 category 数据迁移到新表
INSERT INTO expert_group_categories (expert_group_id, category_name)
SELECT id, category
FROM expert_groups
WHERE category IS NOT NULL AND category != '';

-- 3. 删除旧列
ALTER TABLE expert_groups DROP COLUMN `category`;
ALTER TABLE expert_groups DROP INDEX `idx_category`;

## 九、总结

| 维度 | 核心变更 |
|------|---------|
| 数据库 | 新增 `expert_group_categories` 关联表（V14）；DROP `expert_groups.category` 列及索引 |
| 上传流程 | 从 `expert-groups/{id}.json` 读取 `categories` 数组；覆盖模式下 DELETE + INSERT 全量刷新分类 |
| 管理台 API | 新增 `GET /console/expert-groups/categories`；列表/详情响应增加 `categories`（移除旧 `category`）；更新接口支持多分类编辑 |
| workmate API | 列表接口新增 `?category=` 筛选参数；响应增加 `categories` 字段（移除旧 `category`） |
| 前端 | 分类下拉从独立 API 获取；列表展示多分类 Tag；编辑表单改为 Tags 输入 |
| 激活字段 | 无需变更（V11 已有，功能已完整） |

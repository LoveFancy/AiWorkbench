## 二、场景分类定义

### 2.1 分类表

| 场景 ID | 名称 | 匹配标签 |
|---------|------|---------|
| `content` | 内容创作 | 写作、文案、翻译 |
| `office` | 办公场景 | 文档、邮件、日程 |
| `legal` | 法律咨询 | 法律、合同、合规 |
| `research` | 科研学术 | 研究、论文、数据分析 |
| `uiux` | UI/UX | 设计、前端、交互 |
| `data` | 数据分析 | 数据、可视化、统计 |
| `devops` | DevOps | 部署、CI/CD、运维 |
| `edu` | 教育培训 | 教学、课程、培训 |

### 2.2 匹配逻辑

场景筛选采用**双重匹配**策略，按优先级依次尝试：

```
┌─────────────────────────────────────────────────────┐
│  filterByScene(groups, sceneTags)                    │
│                                                     │
│  对每个 group：                                      │
│    1. tags 匹配（优先）                               │
│       group.tags 中任一 tag 包含 sceneTags 中任一标签   │
│       或 sceneTags 中任一标签包含 group.tags 中任一 tag │
│       → 命中则保留                                   │
│                                                     │
│    2. 名称/描述兜底（tags 为空或未命中时）              │
│       拼接 group.name + group.description             │
│       + group.mainRole.name                          │
│       → 任一 sceneTag 出现在文本中则保留               │
└─────────────────────────────────────────────────────┘
```

```typescript
// experts/utils/filter.ts

/** 按场景 tags 过滤：任一 tag 匹配即命中 */
export function filterByScene(
  groups: AgentExpertGroupInfo[],
  sceneTags: string[]
): AgentExpertGroupInfo[] {
  if (!sceneTags || sceneTags.length === 0) return groups
  return groups.filter(g => {
    // 1. tags 直接匹配
    if (g.tags?.some(t =>
      sceneTags.some(st => t.includes(st) || st.includes(t))
    )) return true
    // 2. 名称/描述兜底
    const text = [g.name, g.description, g.mainRole?.name]
      .filter(Boolean).join(' ')
    return sceneTags.some(st => text.includes(st))
  })
}
```

### 2.3 场景计数

场景卡片上的数字（如"3 位专家"）使用 `countByScene()` 实时计算，逻辑与 `filterByScene` 一致，仅返回数量而非过滤后的数组。

```typescript
/** 按 tags 匹配场景下的专家数量 */
export function countByScene(
  groups: AgentExpertGroupInfo[],
  sceneTags: string[]
): number {
  return groups.filter(g => {
    if (g.tags?.some(t =>
      sceneTags.some(st => t.includes(st) || st.includes(t))
    )) return true
    const text = [g.name, g.description, g.mainRole?.name]
      .filter(Boolean).join(' ')
    return sceneTags.some(st => text.includes(st))
  }).length
}
```

## 三、数据流

### 3.1 状态管理

场景筛选使用独立的 `sceneFilter` 状态，与文本搜索（`query`）和标签筛选（`filterTag`）分离：

```typescript
// ExpertPageView.tsx

const [query, setQuery] = React.useState('')
const [filterTag, setFilterTag] = React.useState<FilterTag>(initialFilter)
const [sortBy, setSortBy] = React.useState<SortBy>('smart')
const [sceneFilter, setSceneFilter] = React.useState<string[] | null>(null)
```

### 3.2 数据管道

```
allGroups
    │
    ├── filterByTag(result, filterTag, followed, recent)
    │       ↓ 按关注/最近/可用/不可用筛选
    │
    ├── filterByScene(result, sceneFilter ?? [])
    │       ↓ 按场景 tags 筛选（sceneFilter 为 null 时不过滤）
    │
    ├── searchByName(result, query)
    │       ↓ 按搜索词匹配名称/描述/标签/角色
    │
    └── sortExpertGroups(result, followed, recent, sortBy)
            ↓ 按智能/名称/最近使用排序
            ↓
        displayGroups → ExpertCardGrid
```

### 3.3 场景点击处理

```typescript
// ExpertPageView.tsx — onSceneClick 回调

<ExpertFeaturedScenes
  allGroups={allGroups}
  onSceneClick={(tags) => {
    setSceneFilter(tags)  // 设置场景筛选 tags，非文本搜索
  }}
/>
```

### 3.4 清除场景筛选

用户可通过以下方式清除场景筛选：
- 点击筛选标签切换到其他状态（`filterTag` 变化时 `sceneFilter` 不变，但 `filterByTag` 先执行）
- 点击空态中的"清除筛选"按钮（同时清除 `query`、`filterTag`、`sceneFilter`）
- 切换到其他侧边栏入口（`initialFilter` 变化触发 `useEffect` 重置）

```typescript
// ExpertEmptyState — onClear 回调
onClear={() => {
  setQuery('')
  setFilterTag('all')
  setSceneFilter(null)
}}
```

## 四、显示条件

精选场景区域仅在以下条件**全部满足**时显示：

| 条件 | 说明 |
|------|------|
| `showFeaturedScenes === true` | 仅"专家/专家团"页面传入（已关注/最近使用页面不显示） |
| `filterTag === 'all'` | 未选择收藏/最近/可用/不可用筛选 |
| `!query.trim()` | 搜索框为空 |
| `displayGroups.length > 0` | 有可展示的专家团 |

```typescript
// ExpertPageView.tsx — 条件渲染

{showFeaturedScenes && filterTag === 'all' && !query.trim() && displayGroups.length > 0 && (
  <div className="mb-8">
    <ExpertFeaturedScenes
      allGroups={allGroups}
      onSceneClick={(tags) => setSceneFilter(tags)}
    />
  </div>

```

## 六、关键设计决策

### 6.1 为什么用独立 `sceneFilter` 而非文本搜索

**问题**：最初实现将场景 tags 拼接为搜索词（`setQuery('写作 文案 翻译')`），但 `searchByName` 检查的是"某个搜索词是否包含整个查询字符串"——没有任何单个 tag 会包含 `"写作 文案 翻译"` 这个完整字符串，导致永远返回空。

**方案**：新增独立的 `sceneFilter` 状态 + `filterByScene()` 函数，直接按 tags 逐个匹配（任一 tag 命中即保留），不走文本搜索管道。

### 6.2 为什么场景计数和筛选使用相同逻辑

确保场景卡片上显示的数字与实际点击后看到的结果一致，避免"显示 5 位专家但点进去只有 3 个"的体验割裂。

### 6.3 为什么 tags 为空时兜底名称/描述

WorkBuddy 转换来的专家团可能没有 `tags` 字段，此时通过名称和描述文本匹配仍能正确归类到场景中。

# 客户端：专家分类查看方案

编辑时间：2026年6月18日

> 本方案基于服务端已有的多分类设计，在客户端实现分类筛选能力。

***

## 一、现有筛选机制回顾

### 1.1 当前筛选流程

```
数据管道：allGroups → tag筛选 → scene筛选 → search搜索 → displayGroups
```

| 筛选层    | 实现                                       | 互斥/叠加       |
| ------ | ---------------------------------------- | ----------- |
| Tag 筛选 | `ExpertFilterPills`（已关注/最近使用/专家/专家团/未下载） | 五选一，点选即替换   |
| 场景筛选   | `ExpertFeaturedScenes`（点击场景卡片）           | 独立，与 tag 叠加 |
| 搜索     | `ExpertSearchBar`（文本输入）                  | 独立，与以上叠加    |

### 1.2 当前数据模型

`ServerExpertGroupSummary`（服务端返回）和 `AgentExpertGroupInfo`（客户端使用）均**无** **`categories`** **字段**。服务端多分类方案要求新增此字段。

***

## 二、服务端接口审查

### 2.1 服务端方案中定义的接口

根据服务端方案，客户端相关接口变更如下：

| # | 方法  | 路径                                           | 变更类型 | 说明                                      |
| - | --- | -------------------------------------------- | ---- | --------------------------------------- |
| 1 | GET | `/workmate/expert-groups`                    | 修改   | 新增 `?category=` 入参；响应增加 `categories` 字段 |
| 2 | GET | `/workmate/expert-groups/group-detail/:id`   | 修改   | 响应增加 `categories` 字段                    |
| 3 | GET | `/workmate/console/expert-groups/categories` | 新增   | 管理台用，返回所有分类名称列表                         |

### 2.2 客户端当前对接情况

```typescript
// expert-remote-service.ts
const path = '/workmate/expert-groups'  // 仅调用列表接口，无 category 参数
const res = await httpGet<{ code: number; data: { items: ...; total: number } }>(path)
```

### 2.3 接口缺口

| 缺口                 | 说明                                                                                                |
| ------------------ | ------------------------------------------------------------------------------------------------- |
| 缺失 `categories` 字段 | `ServerExpertGroupSummary` 类型和 `AgentExpertGroupManifest` 类型均无此字段                                 |
| 缺失分类列表接口           | 客户端无 `fetchServerExpertGroupCategories()`，需对接 `GET /workmate/expert-groups/categories`（或复用列表数据提取） |
| 缺失分类筛选参数           | `fetchServerExpertGroups()` 不支持传 `?category=` 参数                                                  |
| 缺失 IPC 通道          | 分类列表拉取无对应 IPC channel                                                                             |
| 合并逻辑脱漏             | `mergeExpertGroups()` 未映射 `categories`                                                            |

> **结论**：服务端接口方案是完整的，但客户端代码尚未对接 `categories` 字段和分类列表接口。

## 三、分类筛选交互设计

### 3.1 布局位置

分类筛选位于现有 Tag 筛选按钮**下方**，与 Tag 筛选**并列叠加**（非互斥）：

```
┌──────────────────────────────────────────────────────┐
│  [搜索框]                          [导入] [刷新]      │
│                                                      │
│  [已关注] [最近使用] [专家] [专家团] [未下载]          │  ← Tag 筛选（互斥）
│                                                      │
│  分类：[开发 ▾]                                       │  ← 分类筛选（叠加）
│                                                      │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐               │
│  │ 卡片  │ │ 卡片  │ │ 卡片  │ │ 卡片  │               │
│  └──────┘ └──────┘ └──────┘ └──────┘               │
└──────────────────────────────────────────────────────┘
```

### 3.2 交互规则

| 场景     | Tag 筛选   | 分类筛选   | 结果                     |
| ------ | -------- | ------ | ---------------------- |
| 初始     | `all`    | 未选（全部） | 全部专家团                  |
| 只看分类   | `all`    | 选"AI"  | 分类包含 "AI" 的所有专家团       |
| 只看 Tag | `expert` | 未选（全部） | 所有专家类型的专家团             |
| 叠加筛选   | `team`   | 选"开发"  | 专家团 + 分类包含"开发"（AND 逻辑） |

### 3.3 组件选型

采用**平铺一行 Pill 标签**形式，与现有 `ExpertFilterPills` 风格保持一致：

- "全部"为默认选中项
- 各分类标签等宽排列，点击切换
- 标签数量超出容器宽度时，左右出现箭头按钮，点击滚动
- 与 Tag 筛选（已关注/最近使用等）并列叠加，非互斥

```
┌──────────────────────────────────────────────────────────────┐
│ 分类：[◀] [全部] [开发] [AI] [模型部署] [内容创作] [测试] [▶] │
└──────────────────────────────────────────────────────────────┘
```

滚动交互：

- 左右箭头仅在内容溢出时才显示
- 点击箭头滚动一个标签宽度（或固定像素量），露出隐藏的标签
- scroll 容器设置 `overflow: hidden` + `scroll-behavior: smooth`，用 `scrollBy()` 控制

## 四、数据流设计

### 4.1 分类列表获取

**方案 B（已实现）：通过独立接口获取**

服务端新增了 `/workmate/expert-groups/categories` 接口（入参为空，返回所有分类列表），实现走此路径：

```
数据流：
  expert-remote-service.ts
    → httpGet(/workmate/expert-groups/categories)
    → 写磁盘缓存 (~/.workmate/expert-group-categories-cache.json)
    → IPC main handler
    → preload bridge
    → expertCategoriesAtom (jotai)
    → ExpertPageView / ExpertPicker 消费
```

额外工作：
  - 新增 `EXPERT_IPC_CHANNELS.FETCH_CATEGORIES` 通道
  - 新增 `fetchServerExpertGroupCategories()` in expert-remote-service.ts（含缓存）
  - 新增 `expertCategoriesAtom` + `fetchExpertCategoriesAtom` in expert-remote.ts
  - `loadRemoteExpertDataAtom` 同步拉取分类

### 4.2 筛选管道变更

```typescript
// ExpertPageView.tsx — displayGroups 变更

const [category, setCategory] = React.useState<string>('all')

const displayGroups = React.useMemo(() => {
  let result = allGroups
  // 1. Tag 筛选（现有）
  result = filterByTag(result, filterTag, followed, recent)
  // 2. 分类筛选（新增）
  if (category !== 'all') {
    result = result.filter(g => g.categories?.includes(category))
  }
  // 3. 场景筛选（现有）
  if (sceneFilter) {
    result = result.filter(g => sceneFilter.has(g.id))
  }
  // 4. 搜索（现有）
  result = searchByName(result, query)
  return result
}, [allGroups, filterTag, category, query, followed, recent, sceneFilter])
```

#### 5.4.3 `ExpertCategoryFilter.tsx` — 新组件

```tsx
import * as React from 'react'
import { ChevronLeft, ChevronRight, Tag } from 'lucide-react'
import { cn } from '@/lib/utils'
import { Button } from '@/components/ui/button'

interface ExpertCategoryFilterProps {
  categories: string[]
  value: string
  onChange: (category: string) => void
}

export function ExpertCategoryFilter({ categories, value, onChange }: ExpertCategoryFilterProps) {
  const scrollRef = React.useRef<HTMLDivElement>(null)
  const [canScrollLeft, setCanScrollLeft] = React.useState(false)
  const [canScrollRight, setCanScrollRight] = React.useState(false)

  const checkScroll = React.useCallback(() => {
    const el = scrollRef.current
    if (!el) return
    setCanScrollLeft(el.scrollLeft > 0)
    setCanScrollRight(el.scrollLeft + el.clientWidth < el.scrollWidth - 1)
  }, [])

  React.useEffect(() => {
    checkScroll()
    const el = scrollRef.current
    if (!el) return
    const obs = new ResizeObserver(checkScroll)
    obs.observe(el)
    return () => obs.disconnect()
  }, [categories, checkScroll])

  const scroll = (direction: 'left' | 'right') => {
    const el = scrollRef.current
    if (!el) return
    el.scrollBy({ left: direction === 'left' ? -200 : 200, behavior: 'smooth' })
  }

  if (categories.length === 0) return null

  return (
    <div className="flex items-center gap-1">
      <Tag size={14} className="text-muted-foreground shrink-0" />
      {canScrollLeft && (
        <Button variant="ghost" size="icon" className="h-7 w-7 shrink-0" onClick={() => scroll('left')}>
          <ChevronLeft size={14} />
        </Button>
      )}
      <div
        ref={scrollRef}
        className="flex items-center gap-1 overflow-hidden"
        onScroll={checkScroll}
      >
        <button
          onClick={() => onChange('all')}
          className={cn(
            'inline-flex items-center rounded-full px-3 py-1 text-xs font-medium transition-colors shrink-0',
            value === 'all'
              ? 'bg-primary text-primary-foreground'
              : 'bg-muted text-muted-foreground hover:bg-muted/80 hover:text-foreground',
          )}
        >
          全部
        </button>
        {categories.map(cat => (
          <button
            key={cat}
            onClick={() => onChange(cat)}
            className={cn(
              'inline-flex items-center rounded-full px-3 py-1 text-xs font-medium transition-colors shrink-0',
              value === cat
                ? 'bg-primary text-primary-foreground'
                : 'bg-muted text-muted-foreground hover:bg-muted/80 hover:text-foreground',
            )}
          >
            {cat}
          </button>
        ))}
      </div>
      {canScrollRight && (
        <Button variant="ghost" size="icon" className="h-7 w-7 shrink-0" onClick={() => scroll('right')}>
          <ChevronRight size={14} />
        </Button>
      )}
    </div>
  )
}
```

## 七、清空筛选行为

当用户切换 Tag 筛选或清空搜索时，分类筛选**保持不变**（不自动重置），保持用户意图。仅在点击"全部"选项或手动清空时重置分类。

## 八、总结

| 维度    | 核心变更                                                                       |
| ----- | -------------------------------------------------------------------------- |
| 类型层   | `ServerExpertGroupSummary` 和 `AgentExpertGroupManifest` 新增 `categories` 字段 |

| 渲染进程  | 新增 `ExpertCategoryFilter` 组件；`ExpertPageView` / `ExpertPicker` 增加分类筛选      |
| 交互    | Tag 筛选 + 分类筛选 AND 叠加；分类单选下拉，位于 Tag 筛选下方                                    |
| 服务端缺口 | `categories` 字段和列表接口已在服务端方案定义，客户端只需配套对接                                    |

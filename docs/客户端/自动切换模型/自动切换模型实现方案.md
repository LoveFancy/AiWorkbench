### 4.2 agent-session-manager.ts — 新增 activeModelId 字段

**文件**：`apps/electron/src/main/lib/agent-session-manager.ts`

AgentSessionMeta 从 `@proma/shared` 导入，类型新增 `activeModelId` 字段后，读写无需改动——存储的 JSON 天然支持新增可选字段。

关键在于 `getAgentSessionMeta()` / `updateAgentSessionMeta()` 的实现：

```typescript
// updateAgentSessionMeta 内部（现有逻辑不变，新字段自动支持）
export function updateAgentSessionMeta(
  sessionId: string,
  updates: Partial<AgentSessionMeta>,
): void {
  const index = readIndex()
  const session = index.sessions.find(s => s.id === sessionId)
  if (!session) return
  Object.assign(session, updates)  // ← activeModelId 自动合并
  session.updatedAt = Date.now()
  writeIndex(index)
}
```

### 4.3 agent-orchestrator.ts — 核心切换逻辑

**文件**：`apps/electron/src/main/lib/agent-orchestrator.ts`

这是本次改动最大的文件。改造集中在 `sendMessage()` 方法内的重试循环（第 1703 行起）。

#### 4.3.1 新增常量

```typescript
/** 同模型最大重试次数 */
const MAX_SAME_MODEL_RETRIES = 1
```

#### 4.3.2 新增工具函数

```typescript
/**
 * 从候选池中选择下一个可用模型
 *
 * @param currentModelId  当前使用的模型 ID
 * @param candidatePool   候选池 ID 列表（按模型列表排序）
 * @param excludeModelIds 本轮已失败的模型 ID 集合
 * @param availableModels 当前系统实际可用模型 ID 集合
 * @returns 下一个候选模型 ID，或 null（耗尽）
 */
function selectNextCandidateModel(
  currentModelId: string,
  candidatePool: string[],
  excludeModelIds: Set<string>,
  availableModels: Set<string>,
): string | null {
  const startIdx = candidatePool.indexOf(currentModelId)
  // 从当前位置的下一个开始遍历
  for (let i = (startIdx >= 0 ? startIdx + 1 : 0); i < candidatePool.length; i++) {
    const candidate = candidatePool[i]
    if (candidate && !excludeModelIds.has(candidate) && availableModels.has(candidate)) {
      return candidate
    }
  }
  return null
}
```

#### 4.3.3 sendMessage() 改造要点

在 `sendMessage()` 方法中添加以下逻辑：

**A. 启动时读取配置 + 确定初始模型**

在模型解析阶段（第 1129 行附近，`resolvedModel` 变量位置）：

```typescript
// ===== 🆕 Auto Mode：模型解析 =====
const appSettings = getSettings()
const autoModeEnabled = appSettings.autoModeEnabled ?? false
const candidatePool = appSettings.autoSwitchCandidateModels ?? []

// 获取系统当前可用模型 ID 集合（用于下线兜底）
const availableModels = new Set<string>()
const channels = listChannels()
for (const c of channels) {
  if (c.enabled) {
    for (const m of c.models) {
      if (m.enabled) availableModels.add(m.id)
    }
  }
}
// 也加入平台模型
const platformChannel = getPlatformChannel()
if (platformChannel) {
  for (const m of platformChannel.models) {
    if (m.enabled) availableModels.add(m.id)
  }
}

// 确定初始模型
let activeModelId = modelId || DEFAULT_MODEL_ID
if (autoModeEnabled && candidatePool.length > 0) {
  // 优先级：sessionMeta.activeModelId > 用户手动选的在候选池中的模型 > 候选池第一个
  const sessionMeta = getAgentSessionMeta(sessionId)
  activeModelId = sessionMeta?.activeModelId
              ?? (modelId && candidatePool.includes(modelId) ? modelId : null)
              ?? candidatePool[0]
              ?? DEFAULT_MODEL_ID
}
// ===== Auto Mode 模型解析结束 =====
```

**B. retry 循环改造**（第 1703 行起）

在现有 `for (attempt = 1; ...)` 循环的 `shouldRetryFromError = true; break` 之后，`continue` 之前，加入切换判定：

```typescript
// ===== 🆕 Auto Mode 跨模型切换 =====
if (autoModeEnabled && shouldRetryFromError) {
  const sameModelAttempts = /* 同模型连续失败的 attempt 数 */ _sameModelAttempts
  
  if (sameModelAttempts > MAX_SAME_MODEL_RETRIES) {
    // 已重试 1 次仍失败，切换模型
    const excludeSet = new Set(_triedModelIds)
    excludeSet.add(activeModelId)
    
    const nextModel = selectNextCandidateModel(
      activeModelId, candidatePool, excludeSet, availableModels
    )
    
    if (nextModel) {
      console.log(`[Agent 编排] Auto Mode 切换模型: ${activeModelId} → ${nextModel}`)
      
      // 通知前端
      this.eventBus.emit(sessionId, {
        kind: 'proma_event',
        event: {
          type: 'model_switched' as any,
          fromModel: activeModelId,
          toModel: nextModel,
        },
      })
      
      activeModelId = nextModel
      _sameModelAttempts = 0
      _triedModelIds.add(nextModel)
      
      // 更新 sessionMeta
      try {
        updateAgentSessionMeta(sessionId, { activeModelId: nextModel })
      } catch { /* ignore */ }
      
      // 更新 queryOptions 的 model
      queryOptions.model = nextModel
      
      shouldRetryFromError = false
      retryDelayElapsedMs = 0  // 复位退避延迟
      continue  // 用新模型重试
    } else {
      // 候选池耗尽
      console.log(`[Agent 编排] Auto Mode 候选池已耗尽`)
      break
    }
  } else {
    _sameModelAttempts++
  }
}
// ===== Auto Mode 跨模型切换结束 =====
```

**C. 同模型计数追踪**

新增局部变量：

```typescript
let _sameModelAttempts = 0
const _triedModelIds = new Set<string>([activeModelId])
```

**D. 成功时记录记忆**

在正常完成的消息持久化之后（第 1160 行附近 `resolvedModel` 位置，或 break 后）：

## 七、测试要点

### 7.1 单元测试

| 测试场景 | 预期 |
|----------|------|
| `selectNextCandidateModel` — 正常选取下一个 | 返回候选池中 current 之后第一个非排除且在线的模型 |
| `selectNextCandidateModel` — 候选池耗尽 | 返回 null |
| `selectNextCandidateModel` — 跳过已下线模型 | 跳过不在 availableModels 中的模型 |
| `selectNextCandidateModel` — currentModelId 不在候选池中 | 从候选池第一个开始遍历 |

### 7.2 集成测试

| 测试场景 | 预期 |
|----------|------|
| Auto OFF 发送消息，模型报 503 | 直接报错，不切换模型 |
| Auto ON，候选池 [A, B]，A 成功 | 正常完成，记录 A 为 activeModelId |
| Auto ON，候选池 [A, B]，A 失败 2 次（首次+重试 1 次） | 切换到 B，通知前端 |
| Auto ON，候选池 [A]，A 失败 2 次 | 候选池耗尽，报错停止（共 2 次 attempt） |
| Auto ON，候选池 [A, B, C]，全部失败 | 共 6 次 attempt 后停止 |
| Auto ON，下一轮消息 | 使用上一轮成功的 activeModelId |
| Auto OFF 后 | activeModelId 不再生效，回到手动选择模型 |
| 候选池为空时打开 Auto Mode | 开关置灰，不可操作 |
| 清空候选池时 Auto Mode ON | 开关自动回拨 OFF 并置灰 |

## 八、实施建议

### 8.1 实施顺序

1. **Phase 1 — 类型 + 存储**：`AppSettings` / `AgentSessionMeta` / `PromaEvent` 类型扩展 + atoms
2. **Phase 2 — 后端核心**：`agent-orchestrator.ts` 跨模型切换 + 跨轮记忆
3. **Phase 3 — 前端 UI**：`ModelSelector.tsx` / `CandidateModelDialog.tsx` / `AgentView.tsx`
4. **Phase 4 — 通知 + 联动**：`useGlobalAgentListeners.ts` toast + Auto 标识

### 8.2 风险点

| 风险 | 缓解措施 |
|------|----------|
| 跨模型切换时 SDK session 不兼容（thinking signature 等） | 利用现有的 `session_not_found` / `thinking_signature_incompatible` 恢复路径，先回填上下文再切换 |
| 不同模型的 API Key 不同（切换渠道时） | 本期仅支持同渠道内模型切换；候选池中模型均来自当前渠道 |
| 候选池配置与当前渠道不匹配 | `selectNextCandidateModel` 中 `availableModels` 过滤，同时切换时重新获取 channel |

# 账号区交互优化设计

## 背景

当前左侧栏底部在未登录状态下同时展示账号入口和独立设置齿轮；打开账号弹层后，弹层顶部展示“未登录”，菜单内又展示“登录 OA 账号”和“设置”。用户在同一视觉区域会看到 2 个设置入口、多个登录语义入口，操作层级不清晰。

## 目标

- 底部侧栏只保留一个账户相关触发器。
- 未登录弹层只保留一个明确的主登录动作。
- 设置入口只在弹层内出现一次。
- 保留外观快捷切换、使用手册、帮助与反馈、检查更新等低频系统操作。
- 展开侧栏和收起侧栏复用同一套账户菜单组件，不引入第二套菜单结构。

## 推荐方案：账户中心

底部触发器从单一“登录 OA 账号”语义调整为更中性的“账户与设置”。它负责打开账户中心弹层，不直接表达具体动作。

未登录弹层顶部改为登录主操作区：

- 使用一个醒目的“登录 OA 账号”主按钮作为唯一登录动作。
- 将“登录后同步 OA 账号状态”作为按钮下方辅助说明。
- 不再在菜单列表里重复展示“登录 OA 账号”。

弹层系统操作区按使用频率排列：

1. 设置
2. 外观快捷切换
3. 使用手册
4. 帮助与反馈
5. 检查更新

已登录弹层保持账号信息头部，但设置入口仍只保留在菜单列表中。底部独立设置齿轮移除，更新或环境异常红点挂在账户触发器上。

## 组件边界

- `UserAccountMenu.tsx`
  - 继续承载已登录和未登录账户菜单。
  - 调整 `GuestAccountMenu` 的触发器文案与弹层结构。
  - 保留 `ThemeQuickSwitch`，但避免登录动作在列表内重复出现。

- `LeftSidebar.tsx`
  - 展开状态底部只渲染 `UserAccountMenu` 或 `GuestAccountMenu`。
  - 移除展开状态底部的独立设置按钮。
  - 收起状态继续使用同一菜单组件。

- `UserAccountMenu.test.ts`
  - 更新行为断言：展开侧栏不再包含底部独立设置按钮。
  - 更新未登录菜单断言：登录主动作只出现一次，设置入口只出现一次。

## 交互细节

- 点击底部“账户与设置”打开弹层。
- 未登录时，点击弹层顶部主按钮触发 `onLogin`。
- 点击“设置”打开通用设置页。
- 点击“检查更新”打开关于页并触发检查更新。
- 主题快捷切换保持当前逻辑：浅色切换到 `special + slate-light`，深色切换到 `dark + default`。

## 验证

- 运行账户菜单相关测试。
- 如修改影响 JSX 结构，运行 renderer 相关类型检查或全量 `bun run typecheck`。
- 视觉检查未登录展开侧栏：同屏只看到一个底部触发器，弹层内只看到一个登录主按钮和一个设置入口。

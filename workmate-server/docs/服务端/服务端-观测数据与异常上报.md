## 一、错误指纹（Error Fingerprinting）


**问题**：相同的错误因不同用户、不同时间上报产生大量重复记录，难以统计和聚合。

**优化**：在 `observability_events` 表增加 `error_fingerprint` 字段。

```prisma
model ObservabilityEvent {
  // ... 现有字段 ...
  errorFingerprint String? @map("error_fingerprint") @db.VarChar(64)  // 错误指纹，用于聚合相同错误
  breadcrumbs      String? @db.Text  // JSON: 事件发生前的用户操作路径
  tags             String? @db.Text  // JSON: 自定义标签 key:value 对

  @@index([errorFingerprint])
}
```

**指纹生成算法**（服务端）：

```typescript
import { createHash } from 'node:crypto'

function generateErrorFingerprint(event: { errorType?: string; errorMessage?: string; errorStack?: string }): string {
  const raw = [event.errorType ?? '', extractTopFrame(event.errorStack) ?? '', event.errorMessage ?? ''].join('|')
  return createHash('md5').update(raw).digest('hex').slice(0, 16)
}

function extractTopFrame(stack?: string): string | null {
  if (!stack) return null
  const match = stack.match(/at\s+(.+?)(?:\s+\(|$)/)
  return match ? match[1] : null
}
```

**客户端侧**：在 `error` 事件中增加 `fingerprint` 字段，由客户端根据 `error.type + error.message` 生成，服务端再根据 `errorType + errorStack 第一帧` 补全校验。

## 二、面包屑（Breadcrumbs）

**问题**：仅捕获异常堆栈难以还原用户操作上下文。

**优化**：客户端维护最近 20 条面包屑（用户操作记录），在异常上报时附加。

```typescript
export interface Breadcrumb {
  type: 'navigation' | 'user_action' | 'system' | 'http'
  category: string       // 如 'ui.click', 'navigation', 'api.call'
  message: string        // 人类可读描述
  timestamp: number
  data?: Record<string, unknown>  // 附加数据（URL、按钮ID等）
}
```

客户端 `ObservabilityService` 增加面包屑功能：

```typescript
private breadcrumbs: Breadcrumb[] = []
private static MAX_BREADCRUMBS = 20

addBreadcrumb(crumb: Breadcrumb): void {
  this.breadcrumbs.push(crumb)
  if (this.breadcrumbs.length > ObservabilityService.MAX_BREADCRUMBS) {
    this.breadcrumbs.shift()
  }
}

reportErrorEvent(error: Error, context?: Record<string, unknown>): void {
  this.reportEvent({
    type: 'error',
    error: { type: error.name, message: error.message, stack: error.stack },
    breadcrumbs: [...this.breadcrumbs],   // 附加面包屑
    ...context,
  })
}
```

集成点：
- 路由切换：`addBreadcrumb({ type: 'navigation', category: 'route', message: '进入设置页' })`
- 按钮点击：`addBreadcrumb({ type: 'user_action', category: 'ui.click', message: '点击发送按钮' })`
- API 调用：`addBreadcrumb({ type: 'http', category: 'api.call', message: 'POST /workmate/observability/events' })`

## 三、客户端上报采样与限流

**问题**：高并发场景下客户端可能产生大量事件，冲击服务端。

**优化**：客户端增加采样率和限流机制。

```typescript
export interface ObservabilityConfig {
  enabled: boolean
  url: string              // 完整上报地址，如 https://host/workmate/observability/events
  timeoutMs?: number
  maxQueueSize?: number
  flushIntervalMs?: number
  maxBatchSize?: number
  sampleRate?: number         // 采样率 0.0~1.0，默认 1.0（全量上报）
  maxEventsPerMinute?: number // 每分钟最大事件数，默认 60
  enableBreadcrumbs?: boolean // 是否启用面包屑，默认 true
}
```

**采样逻辑**：

```typescript
private eventsThisMinute = 0
private minuteWindowStart = Date.now()

reportEvent(item: ObservabilityEventItem): void {
  // error 类型始终上报，不受采样率和限流影响
  if (item.type !== 'error') {
    // 限流：每分钟上限
    const now = Date.now()
    if (now - this.minuteWindowStart > 60_000) {
      this.eventsThisMinute = 0
      this.minuteWindowStart = now
    }
    this.eventsThisMinute++
    if (this.eventsThisMinute > (this.config.maxEventsPerMinute ?? 60)) return

    // 采样
    if (Math.random() > (this.config.sampleRate ?? 1.0)) return
  }

  this.eventQueue.push(item)
  if (this.eventQueue.length >= (this.config.maxQueueSize ?? 100)) {
    this.flushQueue()
  }
}
```

**注意**：`error` 类型事件始终全量上报，不受采样率和限流影响（仅受队列大小限制）。

## 四、服务端事件去重

**问题**：客户端可能因网络超时重试导致同一事件重复写入。

**优化**：客户端为每个事件生成 `eventId`，服务端做幂等去重。

```typescript
// 客户端
import { randomUUID } from 'node:crypto'

export interface ObservabilityEventItem {
  eventId: string          // 事件唯一标识，用于服务端去重
  type: ObservabilityEventType
  // ... 其余字段 ...
}

// 服务端（observability.service.ts）
async function writeEvents(events: ObservabilityEventItem[]): Promise<{ received: number; duplicated: number }> {
  const existingIds = await prisma.observabilityEvent.findMany({
    where: { eventId: { in: events.map(e => e.eventId).filter(Boolean) } },
    select: { eventId: true },
  })
  const existingSet = new Set(existingIds.map(e => e.eventId))
  const newEvents = events.filter(e => !existingSet.has(e.eventId))

  if (newEvents.length > 0) {
    await prisma.observabilityEvent.createMany({ data: newEvents.map(transformEvent) })
  }

  return { received: events.length, duplicated: events.length - newEvents.length }
}
```

> **注**：`eventId` 字段设为 `UNIQUE` 索引，实际实现中可使用 `INSERT IGNORE` / `ON DUPLICATE KEY UPDATE` 实现高效去重。

- 批量插入采用 Prisma `createMany`，单条失败不阻塞其他事件

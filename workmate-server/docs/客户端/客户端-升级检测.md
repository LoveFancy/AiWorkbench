## 一、设计思路

应用启动时调用 `GET /workmate/upgrade/check`，服务端根据**活跃的升级策略**返回是否有更新。客户端收到响应后需二次校验版本号，防止异常数据。

### 服务端行为

| 条件 | 结果 |
|------|------|
| 有 ACTIVE 策略 | 返回策略信息（targetVersion / downloadUrl / releaseType），白名单按策略下的规则匹配 |
| 无 ACTIVE 策略但有活跃发布 | 返回发布表信息 + 该发布下的白名单 |
| 都没有 | `hasUpdate=false` |

> **关键**：只有管理台创建升级策略并激活后，客户端方能检测到更新。服务端优先读取策略数据，发布表仅作降级兜底。

### 客户端行为

收到 `hasUpdate=true` 后，客户端需执行**二次校验**：

```
releaseType = 'upgrade':
  ✓ targetVersion > currentVersion → 允许升级
  ✗ targetVersion ≤ currentVersion → 忽略（异常数据）

releaseType = 'rollback':
  ✓ targetVersion < currentVersion → 允许回退
  ✗ targetVersion ≥ currentVersion → 忽略（异常数据）
```

## 二、数据模型

### 2.1 升级检测结果类型

```typescript
export interface UpgradeCheckResult {
  hasUpgrade: boolean
  releaseType?: 'upgrade' | 'rollback'
  upgradeAllowed?: boolean
  currentVersion: string
  targetVersion?: string
  releaseNotes?: string
  downloadUrl?: string
  minVersion?: string
  hint?: string
  publishedAt?: string
}
```

### 2.2 后端响应格式

`GET /workmate/upgrade/check?currentVersion=1.0.0&platform=win32`

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "hasUpgrade": true,
    "releaseType": "upgrade",
    "upgradeAllowed": true,
    "currentVersion": "1.0.0",
    "targetVersion": "1.2.0",
    "releaseNotes": "修复了若干已知问题，优化了性能",
    "downloadUrl": "https://example.com/releases/1.2.0/installer.exe",
    "minVersion": "1.0.0",
    "hint": "有新版本可用，建议更新",
    "publishedAt": "2026-05-20T10:00:00Z"
  }
}
```

**字段说明**：

| 字段 | 说明 |
|------|------|
| `hasUpgrade` | 是否有更新（包括升级和回退） |
| `releaseType` | `upgrade` 表示升级，`rollback` 表示回退 |
| `upgradeAllowed` | 当前用户是否在白名单内，可进行升级 |
| `targetVersion` | 目标版本号 |
| `minVersion` | 最低允许的当前版本 |
| `downloadUrl` | 下载地址 |

## 三、服务层设计

新增文件：`apps/electron/src/main/lib/upgrade-check-service.ts`

```
函数列表：
├── initUpgradeCheck(config): void
├── checkForUpgrade(): Promise<UpgradeCheckResult>
├── showUpgradeNotification(result): void
└── scheduleNextCheck(): void
```

```typescript
import { getToken } from './auth-service'
import { getSettings } from './settings-service'
import { app } from 'electron'

export async function checkForUpgrade(): Promise<UpgradeCheckResult> {
  const settings = getSettings()
  if (!settings.serverUrl) {
    return { hasUpgrade: false, currentVersion: app.getVersion() }
  }

  const token = getToken()
  if (!token) {
    console.warn('[升级检测] 未登录，跳过检测')
    return { hasUpgrade: false, currentVersion: app.getVersion() }
  }

  const currentVersion = app.getVersion()
  const platform = process.platform
  const url = `${settings.serverUrl}/workmate/upgrade/check?currentVersion=${currentVersion}&platform=${platform}`

  try {
    const response = await fetch(url, {
      headers: { 'Cookie': `EIPGW-TOKEN=${token}` },
      signal: AbortSignal.timeout(10000),
    })
    // 请求经过 EIP 网关，Gateway 自动注入 X-EIPGW-USERID Header

    if (!response.ok) {
      return { hasUpgrade: false, currentVersion }
    }

    const json = await response.json()
    if (json.code !== 0) {
      return { hasUpgrade: false, currentVersion }
    }

    return {
      hasUpgrade: json.data.hasUpgrade,
      releaseType: json.data.releaseType,
      upgradeAllowed: json.data.upgradeAllowed,
      currentVersion,
      targetVersion: json.data.targetVersion,
      releaseNotes: json.data.releaseNotes,
      downloadUrl: json.data.downloadUrl,
      minVersion: json.data.minVersion,
      hint: json.data.hint,
      publishedAt: json.data.publishedAt,
    }
  } catch (error) {
    console.warn('[升级检测] 网络请求失败:', error)
    return { hasUpgrade: false, currentVersion }
  }
}
```

## 四、升级检测流程

```
应用启动 (main/index.ts)
    ↓
checkForUpgrade()
    ↓
GET {serverUrl}/workmate/upgrade/check?currentVersion=X&platform=Y
Cookie: EIPGW-TOKEN=<jwt>
    ↓ (经过 EIP 网关，自动注入 X-EIPGW-USERID)
    ↓
响应:
  hasUpgrade=false       → 静默（已是最新）
  upgradeAllowed=true    → 弹窗通知（下载链接）
  upgradeAllowed=false   → 记录日志（不在白名单或版本过低）
    ↓
每 N 小时（默认 12h）重复检测
```

## 五、前端升级通知弹窗

`UpgradeNotification.tsx`：根据 `releaseType` 显示不同标题（"有新版本可用"/"建议回退版本"），展示目标版本号、更新日志，"立即下载"打开系统浏览器下载。升级和回退仅下载 URL 不同，客户端下载后由用户手动安装。

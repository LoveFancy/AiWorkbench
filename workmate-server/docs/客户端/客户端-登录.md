## 一、设计思路

登录不依赖后端 WorkMate Server，而是通过企业 **EIP 网关** 完成认证。客户端只需携带 Cookie 请求，EIP 网关会自动在 Header 中注入加密的工号给后端。

### 与后端关系

```
┌──────────────────────────────────────────────────┐
│              WorkMate 客户端 (Electron)           │
│                                                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 登录模块  │ │ 上报模块  │ │  升级检测模块     │  │
│  │(EIP网关) │ │(观测上报) │ │  (版本检查)      │  │
│  └────┬─────┘ └────┬─────┘ └────────┬─────────┘  │
│       │            │               │              │
│       │  请求均经过 EIP 网关，Cookie 自动携带       │
│       │  EIP 网关自动注入 X-EIPGW-USERID Header   │
│       │            │               │              │
└───────┼────────────┼───────────────┼──────────────┘
        │            │               │
        ▼            ▼               ▼
┌──────────────────────────────────────────────────┐
│                  EIP 网关                         │
│     校验 Cookie → 注入 X-EIPGW-USERID 密文        │
└──────────────────────┬───────────────────────────┘
                       │

┌──────────────────────────────────────────────────┐
│              WorkMate Server                     │
│  /workmate/models  /workmate/observability/events│

│  (从 X-EIPGW-USERID 解密获取工号)                 │
└──────────────────────────────────────────────────┘
```

### 整体流程

```
1. 用户在登录页输入工号 + 密码
          ↓
2. 客户端 POST http://eip.htsc.com.cn/gateway/login
   Body: { "username": "022480", "password": "123123" }
          ↓
3. EIP 网关返回 200:
   Set-Cookie: EIPGW-TOKEN=<short_term_jwt>   ← 短期 Token，有效期 4 小时
          ↓
4. 客户端携带 EIPGW-TOKEN Cookie，请求长期 Token:
   GET http://eip.htsc.com.cn/gateway/manage/user/token/generate?days=365
          ↓
5. EIP 网关返回长期 Token（字符串格式）:
   EIPGW-TOKEN:您的token为：eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
   有效期 365天
          ↓
6. 客户端安全存储：长期 EIPGW-TOKEN + 过期时间 + 工号
          ↓
7. 所有后续请求经 EIP 网关，网关自动：
   - 携带 Cookie（EIPGW-TOKEN）
   - 注入 Header X-EIPGW-USERID（AES-128-GCM 加密工号）
```

## 三、服务层设计

### 3.1 认证服务（新增文件：`apps/electron/src/main/lib/auth-service.ts`）

```
函数列表（按调用顺序排列）：
┌── loginWithEipGateway(username, password, days)   — ★ 总入口：完整登录流程
│   ├── login(username, password)                   — Step 1: EIP 网关登录
│   ├── getLongTermToken(shortToken, days)          — Step 2: 获取长期 Token
│   └── saveToken(token, jobId, days)               — Step 3: 安全存储
├── getToken(): string | null                       — 读取 Token（供其他模块调用）
├── getJobId(): string | null                       — 读取工号
├── isLoggedIn(): boolean                           — 是否已登录
└── logout(): void                                  — 清除登录状态
```

```typescript
// auth-service.ts 核心实现概要

import { safeStorage } from 'electron'
import { readFileSync, writeFileSync, existsSync } from 'node:fs'
import { join } from 'node:path'
import { getConfigDir } from './config-paths'

const AUTH_FILE = 'auth.json'
const EIP_GATEWAY_BASE = 'http://eip.htsc.com.cn/gateway'

function getAuthFilePath(): string {
  return join(getConfigDir(), AUTH_FILE)
}

// ===== 总入口：完整登录流程 =====

/**
 * 完整登录 + 获取长期 Token 流程（对外唯一入口）
 *
 * 调用链：login() → getLongTermToken() → saveToken()
 */
export async function loginWithEipGateway(
  username: string, password: string, longTermDays: number = 365,
): Promise<{ success: boolean; message: string; jobId?: string; tokenExpiresAt?: number }> {
  // Step 1: EIP 网关登录
  const loginResult = await login(username, password)
  if (!loginResult.success || !loginResult.jobId || !loginResult.shortToken) {
    return { success: false, message: loginResult.message }
  }

  // Step 2: 换取长期 Token
  const longTerm = await getLongTermToken(loginResult.shortToken, longTermDays)
  if (!longTerm) {
    return { success: false, message: '获取长期 Token 失败' }
  }

  // Step 3: 安全存储长期 Token
  saveToken(longTerm.token, loginResult.jobId, longTermDays)

  return {
    success: true,
    message: '登录成功',
    jobId: loginResult.jobId,
    tokenExpiresAt: Date.now() + longTermDays * 24 * 60 * 60 * 1000,
  }
}

// ===== Step 1: EIP 网关登录 =====

async function login(
  username: string, password: string,
): Promise<{ success: boolean; message: string; jobId?: string; shortToken?: string }> {
  try {
    const response = await fetch(`${EIP_GATEWAY_BASE}/login`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Origin': 'http://eip.htsc.com.cn',
        'Host': 'eip.htsc.com.cn',
      },
      body: JSON.stringify({ username, password }),
      redirect: 'manual',
    })

    if (response.status !== 200) {
      return { success: false, message: `登录失败: HTTP ${response.status}` }
    }

    const setCookie = response.headers.get('set-cookie') ?? ''
    const token = extractToken(setCookie)
    if (!token) {
      return { success: false, message: '登录失败: 未获取到 EIPGW-TOKEN' }
    }

    const jobId = parseJobId(token)
    return { success: true, message: '登录成功', jobId: jobId ?? username, shortToken: token }
  } catch (error) {
    return { success: false, message: `登录请求异常: ${(error as Error).message}` }
  }
}

// ===== Step 2: 获取长期 Token =====

async function getLongTermToken(
  shortTermToken: string, days: number = 365,
): Promise<{ token: string; days: number } | null> {
  try {
    const response = await fetch(
      `${EIP_GATEWAY_BASE}/manage/user/token/generate?days=${days}`,
      { headers: { 'Cookie': `EIPGW-TOKEN=${shortTermToken}` } },
    )
    if (!response.ok) return null

    const text = await response.text()
    const match = text.match(/EIPGW-TOKEN[：:]\s*您的token为[：:]\s*([A-Za-z0-9\-_.]+)/)
    if (!match) return null
    return { token: match[1], days }
  } catch {
    return null
  }
}

// ===== Step 3: 本地安全存储 =====

function saveToken(token: string, jobId: string, days: number): void {
  const expiresAt = Date.now() + days * 24 * 60 * 60 * 1000
  const authData: Record<string, unknown> = { jobId, expiresAt, days, lastLoginAt: Date.now() }

  if (safeStorage.isEncryptionAvailable()) {
    authData.encryptedToken = safeStorage.encryptString(token).toString('base64')
  } else {
    authData.token = token
  }

  writeFileSync(getAuthFilePath(), JSON.stringify(authData, null, 2), 'utf-8')
}

// ===== 读取方法（供其他模块调用）=====

export function getToken(): string | null {
  const filePath = getAuthFilePath()
  if (!existsSync(filePath)) return null

  try {
    const data = JSON.parse(readFileSync(filePath, 'utf-8'))
    if (data.expiresAt && Date.now() > data.expiresAt) return null

    if (data.encryptedToken && safeStorage.isEncryptionAvailable()) {
      return safeStorage.decryptString(Buffer.from(data.encryptedToken, 'base64'))
    }
    return data.token ?? null
  } catch { return null }
}

export function getJobId(): string | null {
  try {
    const data = JSON.parse(readFileSync(getAuthFilePath(), 'utf-8'))
    return data.jobId ?? null
  } catch { return null }
}

export function isLoggedIn(): boolean {
  const token = getToken()
  if (!token) return false
  return parseJobId(token) !== null
}

export function logout(): void {
  writeFileSync(getAuthFilePath(), JSON.stringify({}), 'utf-8')
}

// ===== 工具函数 =====

/** 从 Set-Cookie 响应头中提取 EIPGW-TOKEN 值 */
function extractToken(header: string): string | null {
  const match = header.match(/EIPGW-TOKEN=([^;]+)/)
  return match ? match[1] : null
}

/** 从 JWT payload 中解析 mid（工号） */
function parseJobId(jwt: string): string | null {
  try {
    const parts = jwt.split('.')
    if (parts.length !== 3) return null
    const payload = JSON.parse(Buffer.from(parts[1], 'base64').toString('utf-8'))
    return payload.mid ?? null
  } catch { return null }
}
```

### 3.2 Cookie 传递约定

客户端请求经过 EIP 网关时，**网关自动完成以下操作**：

1. 校验请求携带的 Cookie（`EIPGW-TOKEN`）
2. 校验通过后，在 Header 中注入 `X-EIPGW-USERID`（AES-128-GCM 加密的工号）

客户端只需要确保请求携带 Cookie 即可，**无需自行处理工号加密**。在 Electron 中，使用 `fetch` 时通过 `headers.Cookie` 或 `credentials: 'include'` 传递 Cookie：

```typescript
// 构建请求的辅助函数
export function buildAuthHeaders(): Record<string, string> {
  const token = getToken()
  if (token) {
    return { 'Cookie': `EIPGW-TOKEN=${token}` }
  }
  return {}
}
```

## 六、安全注意事项

| 项 | 措施 |
|----|------|
| Token 本地存储 | `safeStorage.encryptString()` 加密 |
| 传输 | Cookie 携带 EIPGW-TOKEN，EIP 网关自动注入加密工号 |
| Token 管理 | 短期 Token（4h）换取长期 Token（可配置天数） |
| Token 泄露 | 长期 Token 为个人令牌，需妥善保管 |

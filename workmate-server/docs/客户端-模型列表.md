# 客户端：获取用户 API Key 与模型列表方案

编辑时间：2026年6月3日

> **关联文档**：对应后端服务设计见 [服务端设计.md](./服务端设计.md)。客户端登录方案见 [客户端-登录.md](./客户端-登录.md)。请求经过 EIP 网关，网关自动注入 `X-EIPGW-USERID` Header。

## 一、设计思路

客户端登录 EIP 网关后，调用 `GET /workmate/models` 接口获取当前用户的大模型平台 API Key 和可用模型列表。服务端根据 EIP 网关注入的工号，代理查询大模型平台，返回用户的 API Key 和模型信息。

### 为什么走服务端代理

- **API Key 安全**：Key 不需要下发到客户端本地存储，或者即使下发也通过服务端管控
- **模型权限管控**：由大模型平台统一管理用户可见的模型范围，服务端透传
- **无额外认证**：客户端已通过 EIP 网关登录，无需再对接大模型平台的认证体系
- **访问控制**：服务端可按需增加额外逻辑（如审计日志、限流等）

### 整体流程


## 二、接口定义

### 2.1 客户端 → 服务端

```
GET /workmate/models
Cookie: EIPGW-TOKEN=<jwt>
```

**说明**：无需传参，工号由 EIP 网关注入 `X-EIPGW-USERID` Header。

### 2.2 服务端 → 大模型平台

```
GET {MODEL_PLATFORM_API_URL}/users/{jobId}/credentials
```

**入参**：`jobId` — 从 `X-EIPGW-USERID` 解密得到的工号

**出参**：

```json
{
  "apiKey": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "models": [
    {
      "id": "gpt-4",
      "name": "GPT-4",
      "description": "OpenAI GPT-4 模型",
      "provider": "openai",
      "maxTokens": 8192,
      "enabled": true
    },
    {
      "id": "deepseek-v3",
      "name": "DeepSeek V3",
      "description": "深度求索 V3 模型",
      "provider": "deepseek",
      "maxTokens": 32768,
      "enabled": true
    }
  ]
}
```

### 2.3 服务端 → 客户端

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "apiKey": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "models": [
      {
        "id": "gpt-4",
        "name": "GPT-4",
        "description": "OpenAI GPT-4 模型",
        "provider": "openai",
        "maxTokens": 8192,
        "enabled": true
      }
    ],
    "total": 1
  },
  "timestamp": 1717300000000
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `apiKey` | string | 用户的大模型平台 API Key，用于后续直接调用大模型 API |
| `models` | ModelInfo[] | 用户有权限使用的模型列表 |
| `models[].id` | string | 模型唯一标识 |
| `models[].name` | string | 模型显示名称 |
| `models[].provider` | string | 模型提供商 |
| `models[].maxTokens` | number | 最大 Token 数 |
| `models[].enabled` | boolean | 是否启用（服务端仅返回 enabled=true 的模型） |
| `total` | number | 可用模型总数 |

### 3.2 客户端模型服务

新增文件：`apps/electron/src/main/lib/model-service.ts`

```
函数列表：
┌── fetchUserModels()                              — 总入口：获取 API Key + 模型列表
│   ├── 构建请求 (Cookie: EIPGW-TOKEN)
│   ├── GET {serverUrl}/workmate/models            — 通过 EIP 网关
│   └── 返回 { apiKey, models }                     — API Key + 模型列表
├── getApiKey(): string | null                     — 获取缓存的 API Key
├── getModels(): ModelInfo[]                       — 获取缓存的模型列表
└── clearCache(): void                             — 清除缓存
```

```typescript
// model-service.ts 核心实现概要

import { getToken } from './auth-service'
import { getSettings } from './settings-service'

let cachedApiKey: string | null = null
let cachedModels: ModelInfo[] = []

/** 获取用户 API Key 和模型列表 */
export async function fetchUserModels(): Promise<ModelListResponse> {
  const settings = getSettings()
  const token = getToken()

  if (!settings.serverUrl || !token) {
    throw new Error('未登录或未配置服务端地址')
  }

  const response = await fetch(`${settings.serverUrl}/workmate/models`, {
    headers: {
      'Cookie': `EIPGW-TOKEN=${token}`,
      'Content-Type': 'application/json',
    },
    signal: AbortSignal.timeout(10000),
  })

  if (!response.ok) {
    throw new Error(`获取模型列表失败: HTTP ${response.status}`)
  }

  const json = await response.json()
  if (json.code !== 0) {
    throw new Error(json.message ?? '获取模型列表失败')
  }

  const { apiKey, models, total } = json.data

  // 缓存到内存
  cachedApiKey = apiKey
  cachedModels = models

  return { apiKey, models, total }
}

/** 获取缓存的 API Key */
export function getApiKey(): string | null {
  return cachedApiKey
}

/** 获取缓存的模型列表 */
export function getModels(): ModelInfo[] {
  return cachedModels
}

/** 清除缓存 */
export function clearCache(): void {
  cachedApiKey = null
  cachedModels = []
}
```

### 3.4 前端 Jotai Atom

新增文件：`apps/electron/src/renderer/atoms/models.ts`

```typescript
import { atom } from 'jotai'

export const modelsAtom = atom<ModelInfo[]>([])
export const apiKeyAtom = atom<string | null>(null)
export const modelsLoadingAtom = atom<boolean>(false)

// 获取模型列表（带加载状态）
export const fetchModelsAtom = atom(
  null,
  async (get, set) => {
    set(modelsLoadingAtom, true)
    try {
      const result = await window.electronAPI.fetchModels()
      set(modelsAtom, result.models)
      set(apiKeyAtom, result.apiKey)
    } catch (error) {
      console.error('获取模型列表失败:', error)
    } finally {
      set(modelsLoadingAtom, false)
    }
  }
)
```

## 四、服务端实现

### 4.1 模型平台服务

文件：`src/services/model-platform.service.ts`

```typescript
import { config } from '../config'
import { logger } from '../utils/logger'
import type { ModelInfo } from '../types'

interface ModelPlatformCredentialsResponse {
  apiKey: string
  models: Array<{
    id: string
    name: string
    description?: string
    provider?: string
    maxTokens?: number
    enabled: boolean
  }>
}

export interface UserCredentials {
  apiKey: string
  models: ModelInfo[]
}

export async function getUserCredentials(userId: string): Promise<UserCredentials> {
  const url = `${config.modelPlatformApiUrl}/users/${userId}/credentials`

  logger.info('查询用户凭证和模型列表', { userId, url })

  try {
    const controller = new AbortController()
    const timeoutId = setTimeout(() => controller.abort(), config.modelPlatformTimeoutMs)

    const response = await fetch(url, {
      method: 'GET',
      headers: { 'Content-Type': 'application/json' },
      signal: controller.signal,
    })

    clearTimeout(timeoutId)

    if (!response.ok) {
      logger.error('大模型平台返回错误', { status: response.status, userId })
      return { apiKey: '', models: [] }
    }

    const data: ModelPlatformCredentialsResponse = await response.json()
    return {
      apiKey: data.apiKey,
      models: data.models.filter((m) => m.enabled),
    }
  } catch (error) {
    if (error instanceof Error && error.name === 'AbortError') {
      logger.error('大模型平台请求超时', { userId })
    } else {
      logger.error('查询大模型平台失败', { error, userId })
    }
    return { apiKey: '', models: [] }
  }
}
```

### 4.2 控制器

文件：`src/controllers/models.controller.ts`

```typescript
import { Request, Response, NextFunction } from 'express'
import { getUserCredentials } from '../services/model-platform.service'
import { sendSuccess, sendError } from '../utils/response'
import { logger } from '../utils/logger'

export async function getUserModelsHandler(
  req: Request,
  res: Response,
  next: NextFunction
): Promise<void> {
  try {
    const jobId = req.jobId!
    const { apiKey, models } = await getUserCredentials(jobId)
    sendSuccess(res, { apiKey, models, total: models.length })
  } catch (error) {
    logger.error('获取模型列表失败', { error })
    next(error)
  }
}
```

文件：`src/routes/models.routes.ts`

## 七、安全注意事项

| 项 | 措施 |
|----|------|
| API Key 传输 | 通过 EIP 网关，请求链路全程经企业网关保护 |
| API Key 缓存 | 仅缓存在主进程内存中，不持久化到磁盘 |
| 工号不可伪造 | 工号由 EIP 网关加密注入，服务端解密后作为查询条件 |
| 模型权限 | 由大模型平台统一下发，服务端不做额外过滤（仅过滤 enabled=false） |
| 日志脱敏 | 日志中不输出完整的 apiKey 值 |

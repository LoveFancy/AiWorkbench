## 一、方案概述

本文档为 WorkMate 伴行（Proma）客户端设计以下四个功能的实施方案：

| 序号 | 功能 | 目标 |
|------|------|------|
| 1 | 登录功能 | EIP 网关登录，获取长期 Cookie，与后端通信时经过 EIP 网关 |
| 2 | 用户提问观测上报 | 用户登录、提问时将事件上报到后端 `POST /workmate/observability/events` |
| 3 | 升级检测 | 应用启动时调用 `GET /workmate/upgrade/check`，支持白名单灰度发布 |
| 4 | 异常捕获与上报 | 全局异常捕获，上报异常详情到后端观测接口 |

### 设计原则

- **符合现有架构**：遵循 IPC 通信模式、服务层分层、Jotai 状态管理等既有约定
- **本地优先**：EIPGW-TOKEN 使用 `safeStorage` 加密存储到本地文件
- **EIP 网关透明**：客户端请求经过 EIP 网关，网关自动处理 Cookie 和工号加密注入
- **渐进式集成**：在现有 chat-service / agent-orchestrator 的流程中织入上报和升级检测逻辑
- **非侵入**：上报和升级检测失败不影响主业务流程

## 二、方案一：登录功能（EIP 网关认证）

### 2.1 设计思路

登录不依赖后端 WorkMate Server，而是通过企业 **EIP 网关** 完成认证。客户端只需携带 Cookie 请求，EIP 网关会自动在 Header 中注入加密的工号给后端。

整体流程：

```
1. 用户在登录页输入工号 + 密码
          ↓
2. 客户端 POST http://eip.htsc.com.cn/gateway/login
   Body: { "username": "022480", "password": "123123" }
          ↓
3. EIP 网关返回 200:
   Set-Cookie: EIPGW-TOKEN=<short_term_jwt>   ← 短期 Token，有效期 4 小时
          ↓
4. 客户端携带 EIPGW-TOKEN Cookie，请求长期 Token:
   GET http://eip.htsc.com.cn/gateway/manage/user/token/generate?days=365
          ↓
5. EIP 网关返回长期 Token（字符串格式）:
   EIPGW-TOKEN:您的token为：eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
   有效期 365天
          ↓
6. 客户端安全存储：长期 EIPGW-TOKEN + 过期时间 + 工号
          ↓
7. 所有后续请求经 EIP 网关，网关自动：
   - 携带 Cookie（EIPGW-TOKEN）
   - 注入 Header X-EIPGW-USERID（AES-128-GCM 加密工号）
```

### 2.3 服务层设计

#### 2.3.1 认证服务（新增文件：`apps/electron/src/main/lib/auth-service.ts`）

```
函数列表（按调用顺序排列）：
┌── loginWithEipGateway(username, password, days)   — ★ 总入口：完整登录流程
│   ├── login(username, password)                   — Step 1: EIP 网关登录
│   ├── getLongTermToken(shortToken, days)          — Step 2: 获取长期 Token
│   └── saveToken(token, jobId, days)               — Step 3: 安全存储
├── getToken(): string | null                       — 读取 Token（供其他模块调用）
├── getJobId(): string | null                       — 读取工号

└── logout(): void                                  — 清除登录状态
```

```typescript
// auth-service.ts 核心实现概要

import { safeStorage } from 'electron'
import { readFileSync, writeFileSync, existsSync } from 'node:fs'
import { join } from 'node:path'
import { getConfigDir } from './config-paths'

const AUTH_FILE = 'auth.json'
const EIP_GATEWAY_BASE = 'http://eip.htsc.com.cn/gateway'

function getAuthFilePath(): string {
  return join(getConfigDir(), AUTH_FILE)
}

// ===== 总入口：完整登录流程 =====

/**
 * 完整登录 + 获取长期 Token 流程（对外唯一入口）
 *
 * 调用链：login() → getLongTermToken() → saveToken()
 */
export async function loginWithEipGateway(
  username: string, password: string, longTermDays: number = 365,
): Promise<{ success: boolean; message: string; jobId?: string; tokenExpiresAt?: number }> {
  // Step 1: EIP 网关登录
  const loginResult = await login(username, password)
  if (!loginResult.success || !loginResult.jobId || !loginResult.shortToken) {
    return { success: false, message: loginResult.message }
  }

  // Step 2: 换取长期 Token
  const longTerm = await getLongTermToken(loginResult.shortToken, longTermDays)
  if (!longTerm) {
    return { success: false, message: '获取长期 Token 失败' }
  }

  // Step 3: 安全存储长期 Token
  saveToken(longTerm.token, loginResult.jobId, longTermDays)

  return {
    success: true,
    message: '登录成功',
    jobId: loginResult.jobId,
    tokenExpiresAt: Date.now() + longTermDays * 24 * 60 * 60 * 1000,
  }
}

// ===== Step 1: EIP 网关登录 =====

async function login(
  username: string, password: string,
): Promise<{ success: boolean; message: string; jobId?: string; shortToken?: string }> {
  try {
    const response = await fetch(`${EIP_GATEWAY_BASE}/login`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Origin': 'http://eip.htsc.com.cn',
        'Host': 'eip.htsc.com.cn',
      },
      body: JSON.stringify({ username, password }),
      redirect: 'manual',
    })

    if (response.status !== 200) {
      return { success: false, message: `登录失败: HTTP ${response.status}` }
    }

    const setCookie = response.headers.get('set-cookie') ?? ''
    const token = extractToken(setCookie)
    if (!token) {
      return { success: false, message: '登录失败: 未获取到 EIPGW-TOKEN' }
    }

    const jobId = parseJobId(token)
    return { success: true, message: '登录成功', jobId: jobId ?? username, shortToken: token }
  } catch (error) {
    return { success: false, message: `登录请求异常: ${(error as Error).message}` }
  }
}

// ===== Step 2: 获取长期 Token =====

async function getLongTermToken(
  shortTermToken: string, days: number = 365,
): Promise<{ token: string; days: number } | null> {
  try {
    const response = await fetch(
      `${EIP_GATEWAY_BASE}/manage/user/token/generate?days=${days}`,
      { headers: { 'Cookie': `EIPGW-TOKEN=${shortTermToken}` } },
    )
    if (!response.ok) return null

    const text = await response.text()
    const match = text.match(/EIPGW-TOKEN[：:]\s*您的token为[：:]\s*([A-Za-z0-9\-_.]+)/)
    if (!match) return null
    return { token: match[1], days }
  } catch {
    return null
  }
}

// ===== Step 3: 本地安全存储 =====

function saveToken(token: string, jobId: string, days: number): void {
  const expiresAt = Date.now() + days * 24 * 60 * 60 * 1000
  const authData: Record<string, unknown> = { jobId, expiresAt, days, lastLoginAt: Date.now() }

  if (safeStorage.isEncryptionAvailable()) {
    authData.encryptedToken = safeStorage.encryptString(token).toString('base64')
  } else {
    authData.token = token
  }

  writeFileSync(getAuthFilePath(), JSON.stringify(authData, null, 2), 'utf-8')
}

// ===== 读取方法（供其他模块调用）=====

export function getToken(): string | null {
  const filePath = getAuthFilePath()
  if (!existsSync(filePath)) return null

  try {
    const data = JSON.parse(readFileSync(filePath, 'utf-8'))
    if (data.expiresAt && Date.now() > data.expiresAt) return null

    if (data.encryptedToken && safeStorage.isEncryptionAvailable()) {
      return safeStorage.decryptString(Buffer.from(data.encryptedToken, 'base64'))
    }
    return data.token ?? null
  } catch { return null }
}

export function getJobId(): string | null {
  try {
    const data = JSON.parse(readFileSync(getAuthFilePath(), 'utf-8'))
    return data.jobId ?? null
  } catch { return null }
}

export function isLoggedIn(): boolean {
  const token = getToken()
  if (!token) return false
  return parseJobId(token) !== null
}

export function logout(): void {
  writeFileSync(getAuthFilePath(), JSON.stringify({}), 'utf-8')
}

// ===== 工具函数 =====

/** 从 Set-Cookie 响应头中提取 EIPGW-TOKEN 值 */
function extractToken(header: string): string | null {
  const match = header.match(/EIPGW-TOKEN=([^;]+)/)
  return match ? match[1] : null
}

/** 从 JWT payload 中解析 mid（工号） */
function parseJobId(jwt: string): string | null {
  try {
    const parts = jwt.split('.')
    if (parts.length !== 3) return null
    const payload = JSON.parse(Buffer.from(parts[1], 'base64').toString('utf-8'))
    return payload.mid ?? null
  } catch { return null }
}
```

#### 2.3.2 Cookie 传递约定

客户端请求经过 EIP 网关时，**网关自动完成以下操作**：

1. 校验请求携带的 Cookie（`EIPGW-TOKEN`）
2. 校验通过后，在 Header 中注入 `X-EIPGW-USERID`（AES-128-GCM 加密的工号）

客户端只需要确保请求携带 Cookie 即可，**无需自行处理工号加密**。在 Electron 中，使用 `fetch` 时通过 `headers.Cookie` 或 `credentials: 'include'` 传递 Cookie：

```typescript
// 构建请求的辅助函数
export function buildAuthHeaders(): Record<string, string> {
  const token = getToken()
  if (token) {
    return { 'Cookie': `EIPGW-TOKEN=${token}` }
  }
  return {}
}
```

## 三、方案二：用户提问观测上报

### 3.1 设计思路

客户端内聚一个 `ObservabilityService`，收集事件，定时批量 POST 到服务端 `POST /workmate/observability/events`。请求经过 EIP 网关，网关自动注入 `X-EIPGW-USERID`。

### 3.2 数据模型（新增文件：`packages/shared/src/types/observability.ts`）

```typescript
export type ObservabilityEventType =
  | 'user_login' | 'user_logout' | 'chat_question' | 'agent_question'
  | 'error' | 'upgrade_check'

export interface ObservabilityEventItem {
  eventId: string            // 事件唯一标识（UUID），用于服务端去重
  type: ObservabilityEventType
  userId: string             // 用户工号（由 auth-service.getJobId() 填充，服务端以 req.jobId 覆盖）
  userName?: string
  timestamp: number
  question?: string
  questionLength?: number
  modelId?: string
  channelId?: string
  sessionId?: string
  workspaceId?: string
  result?: 'success' | 'failure' | 'pending'
  responseDurationMs?: number
  error?: { type: string; message: string; stack?: string; statusCode?: number; fingerprint?: string }
  breadcrumbs?: Breadcrumb[] // 事件发生前的用户操作路径（error 事件自动附加）
  client: { appVersion: string; platform: string; osVersion: string }
}

export interface Breadcrumb {
  type: 'navigation' | 'user_action' | 'system' | 'http'
  category: string
  message: string
  timestamp: number
  data?: Record<string, unknown>
}

export interface ObservabilityReportBody {
  events: ObservabilityEventItem[]
}

export interface ObservabilityConfig {
  enabled: boolean
  url: string              // 完整上报地址，如 https://host/workmate/observability/events
  timeoutMs?: number
  maxQueueSize?: number
  flushIntervalMs?: number
  maxBatchSize?: number
  sampleRate?: number         // 采样率 0.0~1.0，默认 1.0（全量上报）
  maxEventsPerMinute?: number // 每分钟最大事件数，默认 60
  enableBreadcrumbs?: boolean // 是否启用面包屑，默认 true
}
```

### 3.3 服务层设计

#### 3.3.1 观测上报服务（新增文件：`apps/electron/src/main/lib/observability-service.ts`）

```
函数列表：
├── init(config): void
├── addBreadcrumb(crumb): void             — 记录面包屑（用户操作上下文）
├── reportEvent(item): void                — 通用事件入队（含采样/限流）
├── reportLoginEvent(jobId, name, result, error?): void
├── reportLogoutEvent(jobId, name): void
├── reportChatEvent(params): void
├── reportAgentEvent(params): void
├── reportUpgradeCheckEvent(result): void  — 升级检测结果上报
├── reportErrorEvent(error, context?): void — 异常上报（自动附加面包屑，不受采样限制）
├── flushQueue(): Promise<void>
└── shutdown(): void
```

**架构图**：

```
┌─────────────────────────────────────────────────────────┐
│                 ObservabilityService                     │
├─────────────────────────────────────────────────────────┤
│  面包屑 (breadcrumbs: Breadcrumb[], 最近 20 条)         │
│       │  addBreadcrumb() 入队                            │
│       ▼  reportErrorEvent() 自动附加                     │
│                                                          │
│  采样 & 限流层                                           │
│  ┌──────────────────────────────────────────────────┐   │
│  │ error 事件 → 直通（全量上报）                       │   │
│  │ 其他事件 → 采样率检查 + 每分钟限流                   │   │
│  └──────────────────────────────────────────────────┘   │
│       │ 通过                                             │
│       ▼                                                  │
│  内存队列 (eventQueue: ObservabilityEventItem[])        │
│       │  每事件携带 eventId (UUID) 用于去重              │
│       ▼                                                  │
│  ┌──────────────┐   失败    ┌──────────────────────┐    │
│  │ 定时冲刷      │ ────────→ │ 本地磁盘缓存          │    │
│  │ (5s 间隔)     │           │ ~/.proma/            │    │
│  │ 最多 50 条/批 │ ←──────── │ obs-queue.jsonl      │    │
│  └──────┬───────┘   恢复     └──────────────────────┘    │
│         │ 成功                                            │
│         ▼                                                 │
│  ┌──────────────────────────────────────────────────┐   │
│  │ POST {serverUrl}/workmate/observability/events   │   │
│  │ Headers:                                          │   │
│  │   Content-Type: application/json                  │   │
│  │   Cookie: EIPGW-TOKEN=<jwt>                      │   │
│  │ (EIP 网关自动注入 X-EIPGW-USERID)                  │   │
│  │ Body: { events: [...] }                           │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

核心逻辑：
1. `reportEvent()` 将事件放入内存队列，不阻塞调用方（非 error 事件受采样率和限流控制）
2. `addBreadcrumb()` 维护最近 20 条操作上下文，`reportErrorEvent()` 自动附加面包屑
3. 定时器每 5 秒冲刷队列，构建 `{ events: [...] }` POST 到服务端
4. 请求携带 Cookie，EIP 网关自动注入加密工号
5. 上报失败时序列化追加到磁盘缓存
6. 应用启动时从磁盘恢复未上报事件
7. 每个事件携带 `eventId`（UUID），服务端据此去重
8. `error` 类型事件始终全量上报，不受采样率和限流影响

### 3.4 集成点

#### 登录事件上报

在 `auth-service.ts` 的 `loginWithEipGateway()` 成功后调用 `reportLoginEvent(jobId, username, 'success')`。

#### Chat 提问上报

在 `chat-service.ts` 的 `sendMessage` finally 块中调用 `reportChatEvent({...})`。

#### Agent 提问上报

在 `agent-orchestrator.ts` 的 `run` 方法 finally 块中调用 `reportAgentEvent({...})`。

#### 升级检测上报

在 `upgrade-check-service.ts` 的 `checkForUpgrade()` 返回结果后调用 `reportUpgradeCheckEvent(result)`，记录升级检测结果。

## 四、方案三：升级检测

### 4.1 设计思路

客户端在应用启动时（及之后按固定间隔）调用服务端 `GET /workmate/upgrade/check` 接口。请求经过 EIP 网关，Cookie + 加密工号自动注入。

### 4.2 数据模型

```typescript
  upgradeAllowed?: boolean
  currentVersion: string
  targetVersion?: string

  minVersion?: string                   // 升级场景的最低版本约束（回退场景忽略）

  publishedAt?: string
```

```typescript
import { getJobId, getToken } from './auth-service'
import { getSettings } from './settings-service'
import { app } from 'electron'

export async function checkForUpgrade(): Promise<UpgradeCheckResult> {
  const settings = getSettings()
  if (!settings.serverUrl) {
    return { hasUpgrade: false, currentVersion: app.getVersion() }
  }

  const token = getToken()
  if (!token) {
    console.warn('[升级检测] 未登录，跳过检测')
    return { hasUpgrade: false, currentVersion: app.getVersion() }
  }

  const currentVersion = app.getVersion()
  const platform = process.platform
  const url = `${settings.serverUrl}/upgrade/check?currentVersion=${currentVersion}&platform=${platform}`

  try {
    const response = await fetch(url, {
      headers: { 'Cookie': `EIPGW-TOKEN=${token}` },
      signal: AbortSignal.timeout(10000),
    })
    // 请求经过 EIP 网关，Gateway 自动注入 X-EIPGW-USERID Header

    if (!response.ok) {
      return { hasUpgrade: false, currentVersion }
    }

    const json = await response.json()
    if (json.code !== 0) {
      return { hasUpgrade: false, currentVersion }
    }

    return {
      hasUpgrade: json.data.hasUpgrade,
      releaseType: json.data.releaseType,
      upgradeAllowed: json.data.upgradeAllowed,
      currentVersion,
      targetVersion: json.data.targetVersion,
      releaseNotes: json.data.releaseNotes,
      downloadUrl: json.data.downloadUrl,
      minVersion: json.data.minVersion,
      hint: json.data.hint,
      publishedAt: json.data.publishedAt,
    }
  } catch (error) {
    console.warn('[升级检测] 网络请求失败:', error)
    return { hasUpgrade: false, currentVersion }
  }
}
```

### 4.4 升级检测流程

```
应用启动 (main/index.ts)
    ↓
checkForUpgrade()
    ↓
GET {serverUrl}/workmate/upgrade/check?currentVersion=X&platform=Y
Cookie: EIPGW-TOKEN=<jwt>
    ↓ (经过 EIP 网关，自动注入 X-EIPGW-USERID)
    ↓
响应:
  hasUpgrade=false       → 静默（已是最新）
  upgradeAllowed=true    → 弹窗通知（下载链接）
  upgradeAllowed=false   → 记录日志（不在白名单或版本过低）
    ↓
每 N 小时（默认 12h）重复检测
```

### 4.5 前端升级通知弹窗

`UpgradeNotification.tsx`：根据 `releaseType` 显示不同标题（"有新版本可用"/"建议回退版本"），展示目标版本号、更新日志，"立即下载"打开系统浏览器下载。升级和回退仅下载 URL 不同，客户端下载后由用户手动安装。

### 5.2 防递归保护

```typescript
let isReportingError = false

export function reportErrorEvent(error: Error, context?: Record<string, unknown>): void {
  if (isReportingError) return
  isReportingError = true
  try {
    const fingerprint = generateFingerprint(error.name, error.message)
    reportEvent({
      type: 'error',
      error: { type: error.name, message: error.message, stack: error.stack, fingerprint },
      breadcrumbs: [...breadcrumbs],
      ...context,
    })
  } finally { isReportingError = false }
}

function generateFingerprint(errorType: string, errorMessage: string): string {
  const raw = `${errorType}:${errorMessage.slice(0, 100)}`
  return createHash('md5').update(raw).digest('hex').slice(0, 16)
}
```

## 九、安全注意事项汇总

| 项 | 措施 |
|----|------|
| Token 本地存储 | `safeStorage.encryptString()` 加密 |
| 网络传输 | 请求经过 EIP 网关，Cookie 携带 EIPGW-TOKEN |
| 工号注入 | EIP 网关自动 AES-128-GCM 加密注入 `X-EIPGW-USERID` |
| 客户端无需密钥 | 客户端不持有加解密密钥，密钥仅在 EIP 网关和服务端之间共享 |
| 身份不可伪造 | 服务端入库时用 `req.jobId` 覆盖客户端上报的 `userId`，确保身份真实 |
| Token 过期 | 到期后需重新通过 EIP 网关登录获取 |
| 上报内容脱敏 | 异常堆栈中过滤本地文件路径中的用户名 |
| 防递归上报 | `isReportingError` 标记防止上报异常触发自身递归 |
| 上报失败 | fire-and-forget，失败写入本地磁盘缓存 |
| 事件去重 | 每事件携带 `eventId`（UUID），服务端幂等去重 |
| 采样与限流 | 非 error 事件受采样率/限流控制，error 事件全量上报 |

# WorkMate Server — 管理台前端设计

编辑时间：2026年6月3日

> **关联文档**：[API与数据库设计](./服务端-API与数据库设计.md) | [开发与部署](./服务端-开发与部署.md) | [后续迭代计划](./服务端-后续迭代计划.md)

## 设计决策

| 决策 | 原因 |
|------|------|
| **不设独立登录页** | 管理台通过 EIP 网关访问，用户身份由网关透传的工号 + `admin_whitelist` 表校验，无需额外登录步骤 |
| **React 18 + Vite** | 与客户端主项目保持一致的技术栈；Vite 的 HMR 和快速冷启动提升开发体验 |
| **Ant Design 5** | 企业级 UI 组件库，提供开箱即用的表格、表单、弹窗等组件，适合管理后台类场景 |
| **Axios** | 成熟稳定的 HTTP 客户端，支持拦截器，便于统一处理认证和错误 |

## 一、页面路由

```
/workmate/admin/dashboard          → 仪表盘（通过 /workmate/console/verify 校验白名单，无权限则提示）
/workmate/admin/upgrade            → 升级管理（白名单策略 + 发布版本）
/workmate/admin/observability      → 观测数据查看
/workmate/admin/settings           → 权限设置（管理台访问白名单）
```

### 2.1 管理台入口

管理台**不设独立登录页**，用户身份由 EIP 网关透传。打开 `/workmate/admin` 后：

1. 前端自动调用 `GET /workmate/console/verify`，拦截器解密 `X-EIPGW-USERID` 获取工号并校验 `admin_whitelist`
2. 白名单通过 → 进入仪表盘
3. 白名单不通过 → 显示 "无管理台访问权限，请联系管理员将你的工号加入白名单"

### 2.2 仪表盘 (`/admin/dashboard`)

以卡片形式展示关键统计指标：

- 总事件数 / 今日事件 / 成功率 / 活跃用户数
- 模型使用排行 / 用户活跃排行
- 错误趋势
- 进行中策略数 / 活跃发布版本数

### 2.3 升级管理 (`/admin/upgrade`)

多个 Tab 页：

**Tab 1: 发布版本管理**

- 表格：版本号、类型（升级/回退）、平台、是否激活、发布时间、操作
- 创建发布版本弹窗：版本号、类型、下载链接、发布说明、平台、最低版本

**Tab 2: 白名单规则管理**

- 表格：ID、规则类型、规则值、目标版本、平台、状态、操作
- 添加弹窗：选择 list/range/prefix/suffix，输入规则值
- 操作：启用/禁用、删除

**Tab 3: 升级策略管理**

- 策略列表：策略名称、目标版本、当前阶段/总阶段、状态（draft/active/paused/finished）、平台
- 操作按钮：
  - `draft` → [启动] [编辑] [删除]
  - `active` → [推进下一阶段] [暂停] [一键回退]
  - `paused` → [继续] [一键回退]
  - `finished` → [查看] [一键回退]
- 策略编辑器（创建/编辑弹窗）：
  - 策略名称、目标版本、下载链接、发布说明、平台、minVersion
  - 浸泡时间、自动暂停错误率阈值
  - 动态阶段列表（每阶段含名称 + 规则列表，支持添加/删除阶段和规则）
- 阶段推进确认弹窗：
  - 显示当前阶段信息、已覆盖用户数
  - 健康度指标：错误率、活跃率、平均响应时长
  - 浸泡进度：已过 X 分钟 / 需 Y 分钟
  - 下一阶段的规则变化预览

### 2.4 观测数据查看 (`/admin/observability`)

- 统计卡片：总事件数、错误事件、错误率、高频错误数
- 筛选条件：事件类型、用户工号、错误指纹、时间范围
- 事件列表表格（分页）：ID、事件ID、类型、用户工号、用户名、提问、模型ID、错误信息、错误指纹、响应耗时、客户端版本、平台、时间

### 2.5 权限设置 (`/admin/settings`)

- 管理台访问白名单表格：ID、类型、规则值、备注、状态、操作
- 添加白名单弹窗：规则类型、规则值、备注

## 三、技术实现

### 3.1 技术栈

| 层级 | 技术 |
|------|------|
| 框架 | React 18 |
| 构建 | Vite |
| UI 组件 | Ant Design 5 |
| 路由 | React Router v6 |
| HTTP 客户端 | Axios |

### 3.2 目录结构

```
admin/
├── package.json
├── tsconfig.json
├── vite.config.ts
├── index.html
└── src/
    ├── main.tsx                   # React 入口
    ├── App.tsx                    # 根组件（路由配置 + AntdApp 包裹）
    ├── api/
    │   ├── client.ts              # Axios 实例 + 拦截器
    │   ├── dashboard.api.ts       # 仪表盘统计 API
    │   ├── upgrade.api.ts         # 升级管理 API
    │   ├── strategy.api.ts        # 策略管理 API
    │   ├── observability.api.ts   # 观测数据 API
    │   └── whitelist.api.ts       # 白名单管理 API
    ├── pages/
    │   ├── Dashboard.tsx          # 仪表盘
    │   ├── UpgradeManagement.tsx  # 升级管理
    │   ├── ObservabilityView.tsx  # 观测数据查看
    │   └── Settings.tsx           # 权限设置
    ├── components/
    │   ├── Layout.tsx             # 管理台布局（侧边栏 + 顶栏）
    │   └── StatCard.tsx           # 统计卡片组件
    └── styles/
        └── global.css             # 全局样式
```

### 3.3 Antd v5 注意事项

- 使用 `App` 组件包裹 + `App.useApp()` hook 获取 `message`/`modal`/`notification` 实例，避免静态方法无法消费动态主题的 warning
- `App.tsx` 的包裹顺序：`ConfigProvider` → `AntdApp` → `BrowserRouter` → `Routes`

## 四、相关 API 接口

### 4.1 权限校验


#### 4.2.4 管理台模块

> 管理台统一使用 `/workmate/console` 前缀，通过拦截器 `app.use('/workmate/console', extractUserId, adminWhitelist, adminRouter)` 统一处理所有管理台接口的身份解密和白名单校验。所有请求经 EIP 网关后，`X-EIPGW-USERID` Header 自动注入，拦截器解密获取工号并校验白名单。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/workmate/console/verify` | 验证当前工号是否有管理台权限 |
| GET | `/workmate/console/upgrade/whitelist` | 获取升级白名单列表 |
| POST | `/workmate/console/upgrade/whitelist` | 添加升级白名单规则 |
| PUT | `/workmate/console/upgrade/whitelist/:id` | 更新白名单规则 |
| DELETE | `/workmate/console/upgrade/whitelist/:id` | 删除白名单规则 |
| GET | `/workmate/console/upgrade/releases` | 获取发布版本列表 |
| GET | `/workmate/console/upgrade/releases/history` | 获取历史版本（供回退选择） |
| POST | `/workmate/console/upgrade/releases` | 创建发布版本 |
| PUT | `/workmate/console/upgrade/releases/:id` | 更新发布版本 |
| POST | `/workmate/console/upgrade/rollback` | 一键回退到上一版本 |
| GET | `/workmate/console/upgrade/strategies` | 获取升级策略列表 |
| POST | `/workmate/console/upgrade/strategies` | 创建升级策略 |
| PUT | `/workmate/console/upgrade/strategies/:id` | 更新升级策略 |
| DELETE | `/workmate/console/upgrade/strategies/:id` | 删除升级策略 |
| POST | `/workmate/console/upgrade/strategies/:id/start` | 启动策略（进入第一阶段） |
| POST | `/workmate/console/upgrade/strategies/:id/next-stage` | 推进到下一阶段 |
| POST | `/workmate/console/upgrade/strategies/:id/pause` | 暂停策略 |
| GET | `/workmate/console/upgrade/strategies/:id/stages` | 获取策略的所有阶段及规则 |
| POST | `/workmate/console/upgrade/strategies/:id/stages` | 添加阶段 |
| PUT | `/workmate/console/upgrade/strategies/stages/:id` | 更新阶段 |
| DELETE | `/workmate/console/upgrade/strategies/stages/:id` | 删除阶段 |
| POST | `/workmate/console/upgrade/strategies/stages/:id/rules` | 为阶段添加一条白名单规则 |
| DELETE | `/workmate/console/upgrade/strategies/rules/:id` | 删除阶段规则 |
| GET | `/workmate/console/upgrade/strategies/:id/health` | 获取策略当前健康度 |
| GET | `/workmate/console/observability/events` | 查询观测事件（分页） |
| GET | `/workmate/console/observability/stats` | 观测统计概览 |
| GET | `/workmate/console/whitelist` | 获取管理台白名单列表 |
| POST | `/workmate/console/whitelist` | 添加管理台白名单规则 |
| POST | `/workmate/console/whitelist/batch` | 批量添加白名单规则 |
| DELETE | `/workmate/console/whitelist/:id` | 删除管理台白名单规则 |

**GET `/workmate/console/verify`**

```typescript
// Response（有权限）
{
  "code": 0,
  "message": "ok",
  "data": { "jobId": "022480", "hasAccess": true }
}

// Response（无权限）
{ "code": 403, "message": "无管理台访问权限", "timestamp": 1717300000000 }
```

**POST `/workmate/console/upgrade/whitelist`**

```typescript
// Request
{
  "ruleType": "prefix",
  "ruleValue": "022*",
  "targetVersion": "0.11.0",   // 可选，不传=全版本

}

// Response
{
  "code": 0,
  "message": "添加成功",
  "data": { "id": 42, "ruleType": "prefix", "ruleValue": "022*" }
}
```

│      admin_whitelist         │

│ id (PK)                      │
│ rule_type       VARCHAR(16)  │  ← 'list' | 'range' | 'prefix' | 'suffix'

└──────────────────────────────┘
`

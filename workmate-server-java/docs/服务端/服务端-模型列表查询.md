# WorkMate Server — 模型平台代理

编辑时间：2026年6月3日

> **关联文档**：[API与数据库设计](./服务端-API与数据库设计.md) | [客户端-模型列表](./客户端-模型列表.md)

本文档描述 WorkMate Server 如何代理查询大模型平台，获取用户的 API Key 和可用模型列表。

---

## 一、是什么

客户端需要获取当前用户的 API Key 和可用模型列表。由于 API Key 需要安全管控，且各用户在大模型平台的权限不同，不适合由客户端直接访问大模型平台。因此服务端作为代理，根据 EIP 网关注入的工号，向大模型平台查询该用户的信息并返回。

## 二、为什么

| 原因 | 说明 |
|------|------|
| **API Key 安全** | Key 不直接暴露给客户端网络层，由服务端代理获取 |
| **模型权限管控** | 由大模型平台统一管理用户可见的模型，服务端透传，不做本地缓存 |
| **无额外认证** | 客户端已通过 EIP 网关登录，无需再对接大模型平台的认证体系 |

## 三、怎么做

### 3.1 调用流程

`
客户端 → EIP 网关 → GET /workmate/models (Header: X-EIPGW-USERID 由网关注入)
                       ↓
             models.controller.ts
                       ↓
             model-platform.service.ts
                       ↓        ↑
              GET 大模型平台 API ─┘  （实时查询，携带工号）
                       ↓
              JSON 透传 → 客户端
`

### 3.2 API 接口


#### 4.2.1 模型查询模块

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/workmate/models` | 获取用户可用的模型列表 |

**GET `/workmate/models`**

```typescript
// EIP 网关自动注入 X-EIPGW-USERID Header

// Response
{
  "code": 0,
  "message": "ok",
  "data": {
    "models": [
      {
        "id": "claude-sonnet-4-20250514",
        "name": "Claude 4 Sonnet",
        "provider": "anthropic",
        "type": "chat"
      },
      {
        "id": "gpt-5",
        "name": "GPT-5",
        "provider": "openai",
        "type": "chat"
      }
    ]
  }
}
```

**实现方式**：

服务端不存储模型数据。收到请求后，`model-platform.service.ts` 携带工号向大模型平台 API 发起实时查询，将平台返回的模型列表透传给客户端。

```
客户端 → EIP 网关 → GET /workmate/models (Header: X-EIPGW-USERID 由网关注入)
                       ↓
             models.controller.ts
                       ↓
             model-platform.service.ts
                       ↓        ↑
              GET 大模型平台 API ─┘  （实时查询，携带工号）
                       ↓
              JSON 透传 → 客户端
```

配置项（`.env`）：

```bash
MODEL_PLATFORM_API_URL="https://model-platform.example.com/api/v1/models"
MODEL_PLATFORM_API_KEY="your-api-key"
```

## 四、相关数据模型

模型列表**不存储数据库**。服务端收到请求后实时向大模型平台发起查询，model-platform.service.ts 携带工号调用平台 API：
- **入参**：工号 jobId（从 X-EIPGW-USERID 解密）
- **出参**：{ apiKey: string, models: ModelInfo[] }
- 服务端过滤 enabled=false 的模型后透传给客户端

详细客户端侧方案见 [客户端-模型列表](./客户端-模型列表.md)。

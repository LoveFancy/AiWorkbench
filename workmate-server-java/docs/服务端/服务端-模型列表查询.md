# WorkMate Server — 模型平台代理

编辑时间：2026年6月5日

> **关联文档**：[API与数据库设计](./服务端-API与数据库设计.md) | [客户端-模型列表](./客户端-模型列表.md)

本文档描述 WorkMate Server 如何代理查询大模型平台，获取用户的 API Key 和可用模型列表。

---

## 一、是什么

客户端需要获取当前用户的 API Key 和可用模型列表。由于 API Key 需要安全管控，且各用户在大模型平台的权限不同，不适合由客户端直接访问大模型平台。因此服务端作为代理，根据 EIP 网关注入的工号，向大模型平台查询该用户的信息并返回。

## 二、为什么

| 原因 | 说明 |
|------|------|
| **API Key 安全** | Key 不直接暴露给客户端网络层，由服务端代理获取 |
| **模型权限管控** | 由大模型平台统一管理用户可见的模型，服务端透传，不做本地缓存 |
| **无额外认证** | 客户端已通过 EIP 网关登录，无需再对接大模型平台的认证体系 |

## 三、怎么做

### 3.1 调用流程

```
客户端 → EIP 网关 → GET /workmate/models (Header: X-EIPGW-USERID 由网关注入)
                       ↓
             ModelsController
                       ↓
             ModelPlatformService
                       ↓
             ModelPlatformClientManager
                       ↓        ↑
        POST 大模型平台认证接口 ─┘  （实时查询，携带 userId + scene）
                       ↓
              JSON 透传 → 客户端
```

### 3.2 API 接口

#### 3.2.1 模型查询模块（对客户端暴露）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/workmate/models` | 获取用户可用的模型列表 |

**GET `/workmate/models`**

```
// EIP 网关自动注入 X-EIPGW-USERID Header

// Response
{
  "code": 0,
  "message": "ok",
  "data": {
    "apiKey": "sk-xxx",
    "models": [
      {
        "id": "deepseek",
        "name": "DeepSeek",
        "provider": "anthropic",
        "baseUrl": "http://168.63.65.40:8090/llm-service/v1/messages"
      }
    ],
    "total": 1
  }
}
```

#### 3.2.2 大模型平台认证接口（服务端调用）

测试和生产环境使用同一接口：

| 项目 | 值 |
|------|-----|
| **URL** | `http://168.63.65.40:8090/llm-service/apikey/model/auth` |
| **方法** | POST |
| **Content-Type** | application/json |

**请求体**：

```json
{
  "userId": "022480",
  "scene": "workmate"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| userId | String | 工号（从 X-EIPGW-USERID 解密） |
| scene | String | 固定值 `"workmate"` |

**响应体**：

```json
{
  "finalApiKey": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "modeljson": {
    "object": "list",
    "data": [
      {
        "id": "deepseek",
        "object": "model",
        "permission": [],
        "owned_by": "llmpf"
      }
    ]
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| finalApiKey | String | 用户 API Key |
| modeljson | Object | 模型列表对象 |
| modeljson.object | String | 固定值 `"list"` |
| modeljson.data | Array | 模型数组 |
| modeljson.data[].id | String | 模型 ID |
| modeljson.data[].object | String | 固定值 `"model"` |
| modeljson.data[].permission | Array | 权限列表 |
| modeljson.data[].owned_by | String | 所属平台 |

#### 3.2.3 LLM 大模型服务调用地址

| 协议 | URL |
|------|-----|
| OpenAI 兼容 | `http://168.63.65.40:8090/llm-service/v1/chat/completions` |
| Anthropic 兼容 | `http://168.63.65.40:8090/llm-service/v1/messages` |

### 3.3 实现类关系

```
ModelsController
    │
    └── ModelPlatformService.getUserCredentials(jobId)
            │
            ├── [local-dev.enabled=true] → buildLocalDevCredentials()  ← application-localdev.yml
            │
            └── [测试/生产] → ModelPlatformClientManager.fetchUserCredentials(userId)
                    │
                    └── POST http://168.63.65.40:8090/llm-service/apikey/model/auth
                            body: {userId, scene: "workmate"}
                            │
                            ├── finalApiKey → UserCredentials.apiKey
                            └── modeljson.data → 解析为 ModelInfo 列表
                                    ├── id       ← item.id
                                    ├── name     ← model-name-mapping[id] 或 item.id
                                    ├── provider ← model-provider-mapping[匹配] 或 default-provider
                                    └── baseUrl  ← provider-url-mapping[provider] 或 llm-service.base-url
```

## 四、配置

### application.yml（公共配置，测试 & 生产共用）

```yaml
workmate:
  model-platform-timeout-ms: 10000

  model-platform:
    auth-url: "http://168.63.65.40:8090/llm-service/apikey/model/auth"
    default-provider: "openai"
    provider-url-mapping:
      openai: "http://168.63.65.40:8090/llm-service/v1/chat/completions"
      anthropic: "http://168.63.65.40:8090/llm-service/v1/messages"

  llm-service:
    base-url: "http://168.63.65.40:8090/llm-service/v1"
    openai-path: "/v1/chat/completions"
    anthropic-path: "/v1/messages"
```

**配置说明**：
- `default-provider`：所有模型统一使用的协议提供方（`openai` 或 `anthropic`），决定使用哪种协议接口
- `provider-url-mapping`：provider → 模型调用地址映射

> **说明**：所有模型都支持 openai 和 anthropic 两种协议接口，通过 `default-provider` 统一决定使用哪种协议。

### application-localdev.yml（本地开发，不与上述配置冲突）

```yaml
workmate:
  local-dev:
    enabled: true        # 启用后跳过模型平台 API 调用
    api-key: "sk-xxx"
    models:
      - id: "deepseek-v4-pro"
        name: "DeepSeek V4 Pro"
        provider: "anthropic"
        base-url: "https://api.deepseek.com/anthropic"
        max-tokens: 128000
        enabled: true
```

本地开发模式下 `local-dev.enabled=true` 会短路整个流程，不调用大模型平台认证接口，直接使用 YAML 中预定义的模型列表。

## 一、本地开发

### 1.1 前置条件

| 软件 | 最低版本 | 验证命令 |
|------|----------|----------|
| JDK | 17 | `java -version` |
| Maven | 3.9+ | `mvn -version` |
| MySQL | 8.0 | `mysql --version` |
| Node.js | 18.x（推荐 20.x） | `node -v` |
| 包管理器 | npm / bun | `npm -v` 或 `bun -v` |

### 1.2 启动后端

```bash
# 1. 创建数据库
mysql -u root -p <<EOF
CREATE DATABASE workmate_server CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EOF

# 2. 启动（dev profile 自动关闭身份校验，使用预定义模型列表）
#    所有可调参数均在 src/main/resources/application.yml 中，
#    通过 ${ENV_VAR:default} 占位符支持环境变量覆盖
mvn spring-boot:run -Dspring-boot.run.profiles=localdev
```

> 本地常用环境变量（如 `DB_PASSWORD`）可在 `application.yml` 中直接修改默认值，或通过 IDE Run Configuration / 命令行 `-D` 注入。

启动成功日志：

```
Started WorkmateServerApplication in X.XXX seconds
  Tomcat started on port 6173
```

### 1.3 启动前端

```bash
cd admin
npm install         # 或 bun install
npm run dev
```

Vite 启动后访问 `http://localhost:7173`，`/workmate/console/**` 与 `/workmate/health` 自动代理到 `localhost:6173`。

### 1.4 启动后验证

| 验证项 | 命令 | 预期 |
|--------|------|------|
| 健康检查 | `curl http://localhost:6173/workmate/health` | `{"code":0,"message":"ok","data":{"status":"ok",...}}` |
| 模型接口 | `curl http://localhost:6173/workmate/models` | 模型 JSON 列表（dev profile 为预定义列表） |
| 升级检测 | `curl http://localhost:6173/workmate/upgrade/check` | `{"code":0,...}` |
| 管理台 | 浏览器 `http://localhost:7173` | 进入 React + Ant Design 仪表盘 |

### 1.5 常用开发命令

| 操作 | 命令 |
|------|------|
| 启动后端 | `mvn spring-boot:run -Dspring-boot.run.profiles=localdev` |
| 启动前端 | `cd admin && npm run dev` |
| 运行测试 | `mvn test`（使用 H2 内存库，无需外部 MySQL） |
| 单类测试 | `mvn test -Dtest=UpgradeServiceTest` |
| Flyway 状态 | `mvn flyway:info` |
| Flyway 修复 | `mvn flyway:repair` |
| 清理编译产物 | `mvn clean` |

### 1.6 常见启动问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `Communications link failure` | MySQL 未启动 | 启动 MySQL 并检查 `application.yml` 中 `spring.datasource.url` |
| `Unknown database 'workmate_server'` | 未创建库 | 执行 1.2 步骤 1 |
| 启动后立即退出 | `application.yml` 占位符无匹配值 | 在 `application.yml` 中修改默认值，或注入环境变量 / `-D` 参数 |
| 端口 6173 被占用 | 其他服务占用 | 修改 `application.yml` 中 `server.port`（或环境变量 `PORT`） |
| 前端"缺少用户身份信息" | dev profile 未生效 | 确认启动参数带 `-Dspring-boot.run.profiles=localdev` |
| 前端 404 | 反代未配置 | 确认 vite.config.ts 中 proxy 目标端口与后端一致 |

## 二、前端构建（重点）

管理台前端位于 `admin/` 目录，使用 Vite 6 + React 18 + TypeScript + Ant Design 5。

### 2.1 package.json scripts

```json
{
  "dev":      "vite",                       // 本地开发（HMR）
  "build":    "tsc -b && vite build",       // 类型检查 + 生产构建
  "preview":  "vite preview"                // 本地预览构建产物
}
```

### 2.2 构建命令

```bash
cd admin
npm install
npm run build
# 产物：admin/dist/
```

构建过程：

1. `tsc -b` 进行 TypeScript 类型检查（`strict: true`）
2. `vite build` 打包 → 输出到 `admin/dist/`
   - `base: '/workmate/admin/'` —— 资源以 `/workmate/admin/` 为基准路径
   - React + Ant Design 拆 chunk、压缩、生成 sourcemap
3. 产物结构：

```
admin/dist/
├── index.html
├── assets/
│   ├── index-[hash].js
│   ├── index-[hash].css
│   ├── antd-[hash].js        # antd 单独 chunk
│   └── react-vendor-[hash].js # react / react-dom
└── vite.svg
```

### 2.3 关键配置

`admin/vite.config.ts`：

| 配置 | 值 | 作用 |
|------|-----|------|
| `base` | `/workmate/admin/` | 静态资源与路由 basename |
| `server.port` | `7173` | Vite dev 端口 |
| `server.proxy['/workmate/console']` | `http://localhost:6173` | dev 代理后端 |
| `server.proxy['/workmate/health']` | `http://localhost:6173` | dev 代理后端 |
| `resolve.alias['@']` | `src/` | 路径别名 |

`admin/tsconfig.json`：

- `target: ES2020` / `module: ESNext`
- `strict: true`
- `paths: { "@/*": ["./src/*"] }`

### 2.4 前端产物集成到后端

后端 `WebConfig` 同时支持两套静态资源位置（按优先级）：

```java
registry.addResourceHandler("/workmate/admin/**")
    .addResourceLocations(
        "classpath:/static/admin/",   // 1) JAR 内嵌（mvn package 前拷贝到此处）
        "file:./public/admin/"        // 2) 外部目录（运行时挂载/Docker COPY 均可）
    )
    .resourceChain(true)
    .addResolver(SPA 回退到 index.html);
```

两种集成方式：

| 方式 | 适用 | 操作 |
|------|------|------|
| **外部目录**（推荐） | Docker / 解压即用的分发包 | 构建后将 `admin/dist/` 复制为 `public/admin/`（Dockerfile 与启动脚本均按此路径） |
| **JAR 内嵌** | 单 JAR 部署 | `cp -r admin/dist/* src/main/resources/static/admin/` 后 `mvn package` |

> 当前 `Dockerfile` 与 `bin/start.sh` 使用 **外部目录**方式：前端产物放 `./public/admin/`，运行时由 `WebConfig` 自动映射。

### 2.5 路由（basename）

```tsx
<BrowserRouter basename="/workmate/admin">
  <Routes>
    <Route path="/"               element={<DashboardPage />} />
    <Route path="/upgrade"        element={<UpgradeManagementPage />} />
    <Route path="/observability"  element={<ObservabilityViewPage />} />
    <Route path="/settings"       element={<SettingsPage />} />
  </Routes>
</BrowserRouter>
```

所有页面挂在 `/workmate/admin/` 之下，SPA fallback 由后端资源解析器实现，未匹配路径返回 `index.html`。

## 三、后端构建（重点）

### 3.1 Maven 打包形态

`pom.xml` 采用 **瘦 jar + assembly 分发** 的双层结构：

| 步骤 | 插件 | 产出 |
|------|------|------|
| 1. 编译 | `maven-compiler-plugin`（Spring Boot Parent 默认） | `target/classes/` |
| 2. 打包 jar | `maven-jar-plugin`（**排除** yml/xml/properties/db/mapper） | `target/workmate-server-1.0.0.jar`（瘦 jar，无配置无依赖） |
| 3. 分发 zip | `maven-assembly-plugin`（descriptor: `assembly.xml`） | `target/workmate-server-1.0.0.zip` |

> `spring-boot-maven-plugin` 被显式 `skip`，**不生成 Fat JAR**。运行时分发包结构：

```
workmate-server-1.0.0/
├── workmate-server-1.0.0.jar     # 主 jar（编译产物）
├── lib/                           # 所有依赖 jar
│   ├── spring-boot-3.3.6.jar
│   ├── spring-web-6.1.x.jar
│   ├── mybatis-3.5.x.jar
│   ├── mysql-connector-j-8.x.jar
│   ├── flyway-core-10.x.jar
│   ├── bucket4j-core-8.10.1.jar
│   └── ...
├── config/                        # 配置文件（独立于 jar，运行时可改）
│   ├── application.yml
│   ├── application-localdev.yml
│   ├── mapper/
│   └── db/migration/
└── bin/                           # 启动脚本（chmod 755）
    ├── start.sh
    ├── stop.sh
    └── restart.sh
```

### 3.2 打包命令

```bash
# 仅瘦 jar
mvn clean package -DskipTests
# 产物：target/workmate-server-1.0.0.jar

# 完整分发包（推荐）
mvn clean package -DskipTests
# 产物：target/workmate-server-1.0.0.zip
```

### 3.3 启动命令

```bash
# 方式 A：使用 bin/start.sh（自动 source .env、检测端口、nohup 后台）
cd workmate-server-1.0.0
# 编辑 .env，配置数据库等环境变量（Windows 用 notepad，Linux/Mac 用 vi/nano）
bin/start.sh

# 方式 B：手动 java -cp
java -cp "workmate-server-1.0.0.jar:lib/*:config" \
  com.workmate.server.WorkmateServerApplication
```

> 启动时 Flyway 自动执行 `config/db/migration/` 下的 SQL 脚本（`V1__init_schema.sql` 等）。

### 3.4 配置分层

配置文件按"**公共 + 各环境 profile**"严格分层，Spring Boot 按 `application.yml` → `application-<profile>.yaml` 顺序叠加加载。

- **`application.yml` 只放所有环境完全相同的固定值**（无任何 `${}` 占位符）
- **profile 文件显式列出所有差异化项**（数据库、密钥、池大小、日志级别等）

#### 3.4.1 公共配置（`application.yml`）

所有环境完全一致：

| 配置 | 说明 |
|------|------|
| `server.port` / `server.address` / `max-http-request-header-size` | 服务端口与监听 |
| `spring.servlet.multipart` | 上传大小限制 1MB |
| `spring.flyway` | 迁移策略 |
| `spring.datasource.driver-class-name` | MySQL JDBC 驱动 |
| `spring.datasource.hikari` 基础项 | `pool-name`、`connection-timeout`、`idle-timeout`、`max-lifetime` |
| `mybatis.*` | MyBatis 映射配置 |
| `pagehelper.*` | 物理分页方言 |
| `jackson.*` | JSON 序列化 |
| `workmate.model-platform-timeout-ms` | 大模型平台请求超时（10000） |
| `workmate.observability-sample-rate` | 采样率（1.0） |
| `workmate.observability-max-events-per-minute` | 限流（60） |
| `logging.file.name` | 日志文件路径 `logs/server.log` |
| `logging.pattern.file` | 日志格式 |

#### 3.4.2 各环境 profile（差异化项）

profile 文件**显式**配置所有环境差异：

| 文件 | 差异化项 | 启动方式 |
|------|----------|----------|
| `application-prod.yaml` | 数据库 URL/用户名/密码、HikariCP 池大小、密钥、鉴权、模型平台、日志级别、日志滚动 | `--profile prod` |
| `application-uat.yaml` | UAT 数据库、池大小 10/2、密钥、UAT 模型平台、日志级别、日志保留 7 天 | `--profile uat` |
| `application-dev.yaml` | 开发数据库、池大小 5/1、关闭鉴权、debug 日志、开发模型平台 | `--profile dev` |
| `application-localdev.yml` | 本地预定义模型、关闭鉴权、启用 `workmate.local-dev` | `--profile localdev` |
| `config/application-prod.yaml`（分发包） | 部署时运维直接修改，独立于 jar | 替换分发包内同名文件 |

#### 3.4.3 公共 vs 差异化对照表

| 配置项 | 公共 | prod | uat | dev | localdev |
|--------|:----:|:----:|:---:|:---:|:--------:|
| `server.port` | ✓ 6173 | — | — | — | — |
| `spring.datasource.driver-class-name` | ✓ | — | — | — | — |
| `spring.datasource.url` | — | 生产 | UAT | 开发 | 本地 |

| `spring.datasource.password` | — | `${DB_PASSWORD}` | `${DB_PASSWORD}` | `${DB_PASSWORD}` | — |

> 本地开发仍用 `application-localdev.yml`（已存在，引用 `workmate.local-dev.*` 预定义模型）。

### 3.5 配置覆盖规则

完整优先级（从高到低）：

1. **命令行参数**：`--spring.profiles.active=prod`、`-Dproperty=value`
2. **操作系统环境变量**：`export DB_PASSWORD=xxx`
3. **`.env` 文件**（仅 `bin/start.sh` 加载）：`DB_PASSWORD=xxx`
4. **外部 `config/application-<profile>.yaml`**（分发包）：部署时运维修改
5. **JAR 内 `application-<profile>.yaml`**（打包产物）
6. **JAR 内 `application.yml`**（公共固定值，无占位符）

> 由于 `application.yml` 不再含 `${ENV:default}` 占位符，所有"必填敏感项"（密钥、URL、用户名、密码等）必须由 profile 文件或环境变量提供，缺失时启动会失败——这正是生产环境期望的行为。

#### 启动时必填的环境变量（缺失则启动失败）

| 环境变量 | profile 中引用位置 | 说明 |
|----------|-------------------|------|
| `SPRING_PROFILES_ACTIVE` | 启动命令 | 激活 profile（prod / uat / dev / localdev），由 `bin/start.sh --profile <name>` 设置 |
| `DB_HOST` | `application-{prod,uat,dev}.yaml` | 数据库地址 |
| `DB_PORT` | 同上 | 数据库端口（默认 3306） |
| `DB_NAME` | 同上 | 数据库名（默认 workmate_server） |
| `DB_USERNAME` | 同上 | 数据库用户 |
| `DB_PASSWORD` | 同上 | 数据库密码（**必填**） |
| `USER_ID_ENCRYPTION_KEY` | `application-{prod,uat,dev}.yaml` 的 `workmate.user-id-encryption-key` | AES-128-GCM 密钥（32 位 Hex = 16 字节，**必填**） |

### 3.6 关键 Bean / 配置说明

| 类 | 作用 |
|----|------|
| `WorkmateServerApplication` | 启动类，开启 `@EnableScheduling`（定时任务）、`@EnableTransactionManagement`、`@MapperScan` |
| `AppProperties` | `@ConfigurationProperties("workmate")`，绑定自定义配置（含 `local-dev` 嵌套） |
| `WebConfig` | 注册 `RequestLoggingInterceptor` / `UserIdInterceptor` / `AdminWhitelistInterceptor`；托管 `/workmate/admin/**` 静态资源并实现 SPA fallback；配置 CORS |
| `SecurityConfig` | 通过 Filter 设置 `X-Content-Type-Options: nosniff` 等安全头 |
| `RateLimitFilter` | Bucket4j 内存限流：业务接口 200 req/min、admin 100 req/min、observability 300 req/min |
| `JacksonConfig` | JSON 序列化配置（null 字段不输出、日期不写为时间戳） |

### 3.7 数据库迁移

Flyway 在启动时自动执行 `db/migration/` 下的脚本，按 `V<version>__<description>.sql` 顺序执行：

- `V1__init_schema.sql`：建表（`upgrade_releases` / `upgrade_whitelist` / `upgrade_strategies` / `upgrade_strategy_stages` / `upgrade_strategy_stage_rules` / `admin_whitelist` / `observability_events`）
- 后续迭代以 `V2__xxx.sql` 形式追加

```bash
mvn flyway:info     # 查看迁移状态
mvn flyway:repair   # 修复校验失败
```

## 四、生产部署（虚拟机部署）

> 部署目标：CentOS 7+ / Rocky Linux 8+ 等 Linux 虚拟机，无 Docker 依赖。
> 部署产物：`workmate-server-1.0.0.zip` 分发包 + Nginx 反向代理 + systemd 托管。

### 4.1 部署架构

```
                    ┌──────────────────────────────┐
                    │       用户 / EIP 网关          │
                    └──────────────┬───────────────┘
                                   │
                                   ▼
                    ┌──────────────────────────────┐
                    │     Nginx (端口 18080)        │
                    │  反代 + 静态资源托管 + SPA     │
                    │                              │
                    │  /workmate/admin/**  → 静态文件 │
                    │  /workmate/**        → :6173   │
                    │  /                   → 301 重定向│
                    └──────────────┬───────────────┘
                                   │
                    ┌──────────────┴───────────────┐
                    │      Spring Boot (6173)       │
                    │   WorkmateServerApplication   │
                    └──────────────┬───────────────┘
                                   │
                    ┌──────────────┴───────────────┐
                    │       MySQL 8.0 (3306)        │
                    └──────────────────────────────┘
```

### 4.2 准备分发包

在构建机（任意 Linux/Mac/Windows）上产出完整分发包：

```bash
# 1. 构建前端产物
cd admin
npm install
npm run build          # 产物：admin/dist/
cd ..

# 2. 构建后端分发包
mvn clean package -DskipTests
# 产物：target/workmate-server-1.0.0.zip
#   内部结构：workmate-server-1.0.0/
#   ├── workmate-server-1.0.0.jar     # 主 jar
#   ├── lib/                           # 依赖 jar
#   ├── config/                        # application.yml / mapper / db/migration
#   └── bin/                           # start.sh / stop.sh / restart.sh

# 3. 拼接前端产物到分发包
unzip -o target/workmate-server-1.0.0.zip -d dist
cp -r admin/dist/* dist/workmate-server-1.0.0/public/admin/

# 4. 重新打包（可选）
cd dist && zip -r ../workmate-server-deploy.zip workmate-server-1.0.0/ && cd ..
```

> 前端产物放 `./public/admin/`，由 `WebConfig` 通过 `file:./public/admin/` 路径在运行时加载。

### 4.3 上传到目标服务器


### 4.5 启动 / 停止 / 重启

```bash
cd /opt/workmate/workmate-server-1.0.0

# 启动（指定 profile，会自动加载 config/application-<profile>.yml）
bin/start.sh --profile prod
bin/start.sh --profile uat
bin/start.sh --profile local-dev

# 停止 / 重启（restart.sh 透传 --profile 参数）
bin/stop.sh
bin/restart.sh --profile prod
```

`bin/start.sh` 支持的入参（也可通过环境变量配置）：

| 参数 | 环境变量 | 默认值 | 说明 |
|------|----------|--------|------|
| `--profile <name>` | `SPRING_PROFILES_ACTIVE` | （无） | 加载 `config/application-<name>.yml` |
| `--port <port>` | `PORT` | `6173` | 覆盖服务端口 |
| `--java-home <path>` | — | `/app/workmate/jdk17` | 指定 JDK 安装目录，用绝对路径启动 java，**不依赖 JAVA_HOME / PATH** |

> JDK 路径默认 `/app/workmate/jdk17`（部署约定），脚本内 hardcode。若实际路径不同，通过 `--java-home` 覆盖。
> 启动时会检查 `${JAVA_HOME}/bin/java` 是否存在并可执行，缺失立即报错退出。

> 配置优先级：命令行参数 > 操作系统环境变量 > `.env` 文件 > 默认值。
> 例如 `bin/start.sh --profile prod` 会自动启动加载 `config/application-prod.yml` 的 Spring Boot 实例。
> profile 文件不存在时仅打印 WARN，仍会启动（回退到 `application.yml`）。

`bin/start.sh` 关键逻辑：

1. 解析命令行参数 → 写入 `SPRING_PROFILES_ACTIVE` / `PORT`
2. `source .env`（如存在）→ 覆盖默认环境变量
3. 校验 `config/application-<profile>.yml` 存在性
4. `curl` 健康检查 → 已运行则提示并退出
5. 找 `*.jar` 主 jar（取 `ls | head -1`）
6. `mkdir -p logs`
7. `nohup java -cp "${JAR_FILE}:lib/*:config" com.workmate.server.WorkmateServerApplication --spring.profiles.active=<profile> > logs/server.log 2>&1 &`
8. 写入 `logs/server.pid`（供 stop.sh 与 systemd 复用）

`bin/stop.sh` 关键逻辑：

1. 优先从 `logs/server.pid` 读取 PID
2. 兜底：`ps aux | grep` 查找主类进程
3. `kill PID` → 等待 2s → `kill -9` 兜底

`bin/restart.sh` = `stop.sh` + `sleep 2` + `start.sh "$@"`，参数透传。

> 启动时 Flyway 自动执行 `config/db/migration/` 下的 SQL 脚本（`V1__init_schema.sql` 等）。

### 4.6 systemd 托管（推荐）

将 `bin/start.sh` 注册为 systemd 服务，由系统拉起 + 自动重启 + 开机自启。

#### 4.6.1 创建专用账号

```bash
sudo useradd -r -s /sbin/nologin workmate
sudo chown -R workmate:workmate /opt/workmate
```

#### 4.6.2 systemd 单元文件

```bash
sudo tee /etc/systemd/system/workmate-server.service <<'EOF'
[Unit]
Description=WorkMate Server
After=network.target mysqld.service
Wants=mysqld.service

[Service]
Type=forking
User=workmate
Group=workmate
WorkingDirectory=/opt/workmate/workmate-server-1.0.0
# 通过 --profile 指定环境，prod / uat / dev 等
ExecStart=/opt/workmate/workmate-server-1.0.0/bin/start.sh --profile prod
ExecStop=/opt/workmate/workmate-server-1.0.0/bin/stop.sh
PIDFile=/opt/workmate/workmate-server-1.0.0/logs/server.pid
Restart=on-failure
RestartSec=10
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
EOF
```

> 注：`start.sh` 用 `nohup &` 启动子进程，因此 `Type=forking`。若改为前台启动（`Type=simple`），需改写 `start.sh`。
>
> 多环境可使用 `systemctl edit` 创建 override 配置，会自动叠加到主单元：
>
> ```bash
> sudo systemctl edit workmate-server
> ```
>
> ```ini
> # /etc/systemd/system/workmate-server.service.d/override.conf
> [Service]
> Environment="SPRING_PROFILES_ACTIVE=prod"
> Environment="JAVA_TOOL_OPTIONS=-Xms512m -Xmx1024m"
> ExecStart=
> ExecStart=/opt/workmate/workmate-server-1.0.0/bin/start.sh --profile prod
> ```
>
> 保存后 `sudo systemctl daemon-reload` 即可生效。

#### 4.6.3 启用与运维

```bash
sudo systemctl daemon-reload
sudo systemctl enable workmate-server    # 开机自启
sudo systemctl start workmate-server     # 启动
sudo systemctl status workmate-server    # 状态
sudo journalctl -u workmate-server -f    # 实时日志
sudo systemctl restart workmate-server   # 重启
sudo systemctl stop workmate-server      # 停止
```

### 4.7 Nginx 反向代理

仓库自带两段配置文件：

| 文件 | 作用 | 包含块 |
|------|------|--------|
| `src/main/resources/nginx.conf` | **主框架**：main / events / http 公共配置（gzip、client_max_body_size、log_format、mime.types） | `events`、`http`（含 `include`） |
| `src/main/resources/nginx-server.conf` | **独立 server**：listen 18080、各 location 路由、**独立访问/错误日志** | `server`（仅一个） |

**部署时两个文件需放在同一目录**（`/etc/nginx/conf.d/` 或 `/app/workmate/nginx/`），主配置通过 `include nginx-server.conf;` 引入独立 server。

`nginx-server.conf` 要点：

| 配置 | 值 | 说明 |
|------|-----|------|
| `listen` | `18080` | 对外统一端口 |
| `access_log` | `/app/workmate/nginxlog/server-access.log main` | **独立**访问日志（与主配置日志隔离） |
| `error_log` | `/app/workmate/nginxlog/server-error.log warn` | **独立**错误日志 |
| `client_max_body_size` | `1m` | 与 Spring Boot 上传限制一致 |
| `location /workmate/admin/` | `alias .../public/admin/; try_files ... /workmate/admin/index.html;` | 托管管理台 SPA |
| `location /workmate/` | `proxy_pass http://127.0.0.1:6173;` | 反代到 Spring Boot |
| `location = /` | `return 301 /workmate/admin/;` | 根路径跳管理台 |

**关键代理头**（用于 `X-Forwarded-For` 透传、EIP 网关解密 `X-EIPGW-USERID`）：

```
proxy_set_header Host              $host;
proxy_set_header X-Real-IP         $remote_addr;
proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
```

#### 4.7.1 安装 Nginx（Rocky / CentOS）

```bash
sudo dnf install -y nginx      # Rocky 8/9 / CentOS 8+
# 或
sudo yum install -y nginx      # CentOS 7

sudo systemctl enable nginx
```

#### 4.7.2 部署 nginx.conf 与 nginx-server.conf

```bash
# 1. 创建静态资源与日志目录
sudo mkdir -p /app/workmate/nginxlog
sudo mkdir -p /opt/workmate/workmate-server-1.0.0/public/admin

# 2. 复制两个配置文件到同一目录
sudo cp src/main/resources/nginx.conf        /etc/nginx/conf.d/workmate.conf
sudo cp src/main/resources/nginx-server.conf /etc/nginx/conf.d/nginx-server.conf
#   或统一放 /app/workmate/nginx/ 目录下，调整主配置中的 include 路径

# 3. 按实际部署路径修改（搜索替换）
sudo sed -i 's#/app/workmate/nginxlog/#/var/log/workmate/nginx/#g' /etc/nginx/conf.d/workmate.conf /etc/nginx/conf.d/nginx-server.conf
sudo sed -i 's#/app/workmate/web/#/opt/workmate/workmate-server-1.0.0/public/admin/#g' /etc/nginx/conf.d/nginx-server.conf

# 4. 验证与重载
sudo nginx -t
sudo systemctl reload nginx
```

```bash
# CentOS 7（firewalld 旧版）

```

## 七、常见问题

| 问题 | 排查方向 |
|------|----------|
| 启动报 `ClassNotFoundException: com.workmate.server.WorkmateServerApplication` | classpath 缺主 jar 或 lib。检查 `java -cp "${JAR}:lib/*:config"` 写法 |
| `NoClassDefFoundError: org/mybatis/...` | lib/ 目录为空 → 检查 `maven-assembly-plugin` 是否执行（应在 package 阶段） |
| 启动后 `application.yml` 修改不生效 | 分发包 config 目录被覆盖、且优先级低于环境变量；确认改的是 `./config/application.yml` 而非 jar 内 |
| Flyway 报 "Validate failed" | 已有库结构与迁移脚本不一致 → 评估后用 `flyway:repair` 或备份后重置 |
| 静态资源 404 | `public/admin/` 目录为空或 `WebConfig` 路径不匹配；检查 `docker exec ls public/admin` |
| 升级检测返回空 | `upgrade_releases` 表无 `is_active=true` 数据；或 dev profile 未生效 |
| 管理台"用户不在白名单" | `admin_whitelist` 表规则未覆盖当前工号；检查 `rule_type` / `rule_value` |
| Nginx 502 Bad Gateway | Spring Boot 未启动或 6173 端口未监听；`netstat -tlnp | grep 6173` |

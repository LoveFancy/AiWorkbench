## 一、设计思路

客户端登录 EIP 网关后，调用 `GET /workmate/models` 接口获取当前用户的大模型平台 API Key 和可用模型列表。服务端根据 EIP 网关注入的工号，代理查询大模型平台，返回用户的 API Key 和模型信息。

### 为什么走服务端代理

- **API Key 安全**：Key 在客户端落盘时使用 `safeStorage` 加密
- **模型权限管控**：由大模型平台统一管理用户可见的模型范围，服务端透传
- **无额外认证**：客户端已通过 EIP 网关登录，无需再对接大模型平台的认证体系
- **访问控制**：服务端可按需增加额外逻辑（如审计日志、限流等）

### 核心原则：最小侵入

与登录方案一致，所有模型列表相关代码集中在 `apps/electron/src/models/` **独立目录**内，原有文件仅做最少量的**接线式修改**。

```
原有文件修改策略：
  不改逻辑，只加一行 import + 一行调用
```

### 三级缓存

```
┌─────────────────────────────────────────────┐
│  L1 内存缓存（进程内变量，启动时从 L2 加载）  │
├─────────────────────────────────────────────┤
│  L2 磁盘缓存（~/.proma/models-cache.json）   │
├─────────────────────────────────────────────┤
│  L3 服务端（GET /workmate/models）          │
└─────────────────────────────────────────────┘

读取顺序：L1 → L2 → L3
写入顺序：L3 命中 → 同时写 L1 + L2
```

## 三、接口定义

### 3.1 客户端 → 服务端

```
GET /workmate/models
Cookie: EIPGW-TOKEN=<jwt>
```

**说明**：无需传参，工号由 EIP 网关注入 `X-EIPGW-USERID` Header。

### 3.2 服务端 → 大模型平台

```
GET {MODEL_PLATFORM_API_URL}/users/{jobId}/credentials
```

**入参**：`jobId` — 从 `X-EIPGW-USERID` 解密得到的工号

### 3.3 服务端 → 客户端

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "apiKey": "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "models": [
      {
        "id": "gpt-4",
        "name": "GPT-4",
        "description": "OpenAI GPT-4 模型",
        "provider": "openai",
        "maxTokens": 8192,
        "enabled": true
      }
    ],
    "total": 1
  },
  "timestamp": 1717300000000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `apiKey` | string | 用户的大模型平台 API Key |
| `models` | ModelInfo[] | 用户有权限使用的模型列表 |
| `models[].id` | string | 模型唯一标识 |
| `models[].name` | string | 模型显示名称 |
| `models[].provider` | string | 模型提供商 |
| `models[].maxTokens` | number | 最大 Token 数 |
| `models[].enabled` | boolean | 是否启用（服务端仅返回 enabled=true 的模型） |
| `total` | number | 可用模型总数 |

## 四、存储设计

### 4.1 落盘路径

- 路径：`~/.proma/models-cache.json`
- 与 `auth.json` 同级，不侵入 Proma 上游

### 4.2 文件结构

```typescript
interface ModelsCache {
  /** safeStorage 加密后的 API Key（base64） */
  encryptedApiKey?: string
  /** safeStorage 不可用时的明文 Key（不推荐，仅 fallback） */
  apiKey?: string
  /** 可用模型列表（仅保留 enabled=true） */
  models: ModelInfo[]
  /** 模型总数 */
  total: number
  /** 缓存写入时间（Unix 毫秒），用于 TTL 判断 */
  cachedAt: number
  /** 缓存关联的工号，账号切换时缓存失效 */
  jobId?: string
}
```

### 4.3 关键设计点

1. **API Key 必须加密落盘**：使用 `safeStorage`，与 `auth.json` 中 `encryptedToken` 相同模式
2. **缓存与 jobId 绑定**：账号切换后旧缓存自动失效，避免 Key 串号
3. **模型列表明文落盘**：模型元数据（id/name/description）无敏感性，可明文存储
4. **失败降级**：网络/服务端失败时返回旧缓存，确保离线场景下仍能用已缓存的 Key 调用大模型 API

## 五、缓存策略

### 5.1 TTL 判定

```
if (缓存不存在)                     → 未命中
else if (Date.now() - cachedAt > TTL) → 过期失效
else if (缓存.jobId !== 当前 jobId)   → 账号不一致失效
else                                   → 命中
```

### 5.2 TTL 默认值

- 默认 1 小时（3_600_000 ms）
- 可在 `~/.proma/workmate.json` 中通过 `modelsCacheTtlMs` 自定义

### 6.2 `model-service.ts` — 模型服务（主进程）

```
函数列表：
┌── fetchUserModels(forceRefresh?)                 — ★ 总入口：获取 API Key + 模型列表
│   ├── 缓存命中 → 直接返回
│   ├── 缓存过期 → 网络请求，失败降级为旧缓存
│   └── 写入 L1（内存）+ L2（磁盘，apiKey safeStorage 加密）
├── getApiKey(): string | null                     — 读 L1 缓存的 API Key
├── getModels(): ModelInfo[]                       — 读 L1 缓存的模型列表
├── clearCache(): void                             — 清除所有缓存（登出时调用）
├── initModelRefresh(intervalMs?)                  — 启动定期后台刷新
├── shutdownModelRefresh()                         — 停止定期刷新
├── loadCacheFromDisk(jobId?)                      — 从 L2 加载到 L1（启动时调用）
└── buildAuthHeaders (复用 auth 模块的)            — 构建 Cookie 请求头
```

```typescript
// apps/electron/src/models/model-service.ts

import { safeStorage } from 'electron'
import { readFileSync, writeFileSync, existsSync } from 'node:fs'
import { join } from 'node:path'
import { getToken, getJobId } from '../auth'
import type { ModelInfo, ModelListResponse, ModelsCache } from './types'

// ---- 注意：configDir 通过 initModelService() 注入 ----

const CACHE_FILE = 'models-cache.json'
const DEFAULT_TTL_MS = 3_600_000  // 1 小时

let _configDir: string | null = null
let _serverUrl: string | null = null

/** 初始化模型服务（由 main/index.ts 调用） */
export function initModelService(configDir: string, serverUrl?: string): void {
  _configDir = configDir
  _serverUrl = serverUrl ?? null
}

function cachePath(): string {
  if (!_configDir) throw new Error('Model service not initialized: call initModelService() first')
  return join(_configDir, CACHE_FILE)
}

// ===== L1 内存缓存 =====

let l1ApiKey: string | null = null
let l1Models: ModelInfo[] = []
let l1Total: number = 0
let l1CachedAt: number = 0
let l1JobId: string | null = null
let _refreshTimer: ReturnType<typeof setInterval> | null = null

// ===== 总入口：获取 API Key + 模型列表 =====

/**
 * 获取用户 API Key 和模型列表（带缓存）
 *
 * 行为矩阵：
 * | 缓存命中 + 账号一致          → 直接返回缓存（不发请求）
 * | 缓存过期                    → 发起网络请求，失败时降级为旧缓存
 * | 未登录（无 token）          → 返回 { apiKey: '', models: [], total: 0 }
 * | serverUrl 未配置            → 返回 { apiKey: '', models: [], total: 0 }
 * | 网络失败 / 服务端错误       → 降级为旧缓存，无旧缓存则返回空
 * | forceRefresh=true          → 跳过缓存检查直接请求网络
 *
 * 所有路径都不抛异常。
 */
export async function fetchUserModels(
  forceRefresh = false,
): Promise<ModelListResponse> {
  const token = getToken()
  const currentJobId = getJobId()

  // 未登录，返回空（不抛异常）
  if (!token || !currentJobId || !_serverUrl) {
    return { apiKey: '', models: [], total: 0 }
  }

  // 缓存命中（未过期 + 账号一致）
  if (!forceRefresh && isCacheValid(currentJobId)) {
    return { apiKey: l1ApiKey ?? '', models: l1Models, total: l1Total }
  }

  // 发起网络请求
  try {
    const response = await fetch(`${_serverUrl}/workmate/models`, {
      headers: {
        'Cookie': `EIPGW-TOKEN=${token}`,
        'Content-Type': 'application/json',
      },
      signal: AbortSignal.timeout(10_000),
    })

    if (!response.ok) {
      // 降级：返回旧缓存
      return fallbackToOldCache(currentJobId)
    }

    const json = await response.json()
    if (json.code !== 0) {
      return fallbackToOldCache(currentJobId)
    }

    const { apiKey, models, total } = json.data

    // 写入 L1
    l1ApiKey = apiKey
    l1Models = models
    l1Total = total
    l1CachedAt = Date.now()
    l1JobId = currentJobId

    // 写入 L2（磁盘）
    saveCacheToDisk(apiKey, models, total, Date.now(), currentJobId)

    return { apiKey, models, total }
  } catch {
    return fallbackToOldCache(currentJobId)
  }
}

// ===== L1 读取 =====

export function getApiKey(): string | null {
  return l1ApiKey
}

export function getModels(): ModelInfo[] {
  return l1Models
}

// ===== 缓存清除 =====

export function clearCache(): void {
  l1ApiKey = null
  l1Models = []
  l1Total = 0
  l1CachedAt = 0
  l1JobId = null

  // 清除磁盘缓存
  writeFileSync(cachePath(), JSON.stringify({}), 'utf-8')
}

// ===== 定期刷新 =====

export function initModelRefresh(intervalMs?: number): void {
  const ttl = intervalMs ?? DEFAULT_TTL_MS
  _refreshTimer = setInterval(() => {
    fetchUserModels(true).catch(() => {
      // 静默失败
    })
  }, ttl)
}

export function shutdownModelRefresh(): void {
  if (_refreshTimer) {
    clearInterval(_refreshTimer)
    _refreshTimer = null
  }
}

// ===== 启动时从磁盘加载 =====

export function loadCacheFromDisk(jobId?: string): ModelListResponse | null {
  const filePath = cachePath()
  if (!existsSync(filePath)) return null

  try {
    const cache: ModelsCache = JSON.parse(readFileSync(filePath, 'utf-8'))

    if (!cache.models || cache.models.length === 0) return null

    // 账号不一致 → 失效
    if (jobId && cache.jobId && cache.jobId !== jobId) return null

    // 解密 API Key
    let apiKey = ''
    if (cache.encryptedApiKey && safeStorage.isEncryptionAvailable()) {
      apiKey = safeStorage.decryptString(Buffer.from(cache.encryptedApiKey, 'base64'))
    } else if (cache.apiKey) {
      apiKey = cache.apiKey
    }

    // 恢复到 L1
    l1ApiKey = apiKey
    l1Models = cache.models
    l1Total = cache.total
    l1CachedAt = cache.cachedAt
    l1JobId = cache.jobId ?? null

    return { apiKey, models: cache.models, total: cache.total }
  } catch {
    return null
  }
}

// ===== 内部工具 =====

function isCacheValid(jobId: string): boolean {
  if (!l1Models.length) return false
  if (Date.now() - l1CachedAt > DEFAULT_TTL_MS) return false
  if (l1JobId && l1JobId !== jobId) return false
  return true
}

function fallbackToOldCache(jobId: string): ModelListResponse {
  if (l1JobId === jobId && l1Models.length) {
    return { apiKey: l1ApiKey ?? '', models: l1Models, total: l1Total }
  }
  return { apiKey: '', models: [], total: 0 }
}

function saveCacheToDisk(
  apiKey: string, models: ModelInfo[], total: number,
  cachedAt: number, jobId: string,
): void {
  const cache: ModelsCache = { models, total, cachedAt, jobId }

  if (safeStorage.isEncryptionAvailable()) {
    cache.encryptedApiKey = safeStorage.encryptString(apiKey).toString('base64')
  } else {
    cache.apiKey = apiKey
  }

  writeFileSync(cachePath(), JSON.stringify(cache, null, 2), 'utf-8')
}
```

### 6.3 `ipc-handlers.ts` — IPC 处理器注册

不修改原有的 `main/ipc.ts`，导出注册函数。

```typescript
// apps/electron/src/models/ipc-handlers.ts

import { ipcMain } from 'electron'
import { fetchUserModels, getApiKey, getModels, clearCache } from './model-service'

const CHANNELS = {
  FETCH_MODELS: 'models:fetch',
  GET_API_KEY: 'models:get-api-key',
  GET_MODELS: 'models:get-models',
  CLEAR_CACHE: 'models:clear-cache',
} as const

/**
 * 注册 models 相关的 IPC handler。
 *
 * 调用方式（main/index.ts 中加一行）：
 *   import { registerModelsIpcHandlers } from '../models'
 *   registerModelsIpcHandlers()
 */
export function registerModelsIpcHandlers(): void {
  ipcMain.handle(CHANNELS.FETCH_MODELS, async (_event, forceRefresh?: boolean) => {
    return fetchUserModels(forceRefresh)
  })

  ipcMain.handle(CHANNELS.GET_API_KEY, () => {
    return getApiKey()
  })

  ipcMain.handle(CHANNELS.GET_MODELS, () => {
    return getModels()
  })

  ipcMain.handle(CHANNELS.CLEAR_CACHE, () => {
    clearCache()
    return { success: true }
  })
}

export { CHANNELS as MODELS_IPC_CHANNELS }
```

### 6.4 `preload-bridge.ts` — Preload 桥接注册

```typescript
// apps/electron/src/models/preload-bridge.ts

import { contextBridge, ipcRenderer } from 'electron'
import { MODELS_IPC_CHANNELS } from './ipc-handlers'

export interface ModelsElectronAPI {
  fetchModels: (forceRefresh?: boolean) => Promise<{
    apiKey: string; models: unknown[]; total: number
  }>
  getApiKey: () => Promise<string | null>
  getModels: () => Promise<unknown[]>
  clearCache: () => Promise<{ success: boolean }>
}

/**
 * 将 models API 挂载到 window.electronAPI.models 上。
 *
 * 调用方式（preload/index.ts 中加一行）：
 *   import { registerModelsPreloadBridge } from '../models'
 *   registerModelsPreloadBridge()
 *
 * 注意：需要与 auth 等模块协调 contextBridge.exposeInMainWorld 的调用方式。
 * 推荐各模块同时挂载到一个 electronAPI 对象上。
 */
export function registerModelsPreloadBridge(): void {
  contextBridge.exposeInMainWorld('electronAPI', {
    models: {
      fetchModels: (forceRefresh?: boolean) =>
        ipcRenderer.invoke(MODELS_IPC_CHANNELS.FETCH_MODELS, forceRefresh),
      getApiKey: () =>
        ipcRenderer.invoke(MODELS_IPC_CHANNELS.GET_API_KEY),
      getModels: () =>
        ipcRenderer.invoke(MODELS_IPC_CHANNELS.GET_MODELS),
      clearCache: () =>
        ipcRenderer.invoke(MODELS_IPC_CHANNELS.CLEAR_CACHE),
    } satisfies ModelsElectronAPI,
  })
}
```

### 6.5 `atoms.ts` — Jotai 状态

```typescript
// apps/electron/src/models/atoms.ts

import { atom } from 'jotai'
import type { ModelInfo } from './types'

export const modelsAtom = atom<ModelInfo[]>([])
export const apiKeyAtom = atom<string | null>(null)
export const modelsLoadingAtom = atom<boolean>(false)

/** 异步拉取模型列表并更新 atom */
export const fetchModelsAtom = atom(
  null,
  async (_get, set) => {
    set(modelsLoadingAtom, true)
    try {
      // 调用 preload 暴露的 API
      const result = await (window as any).electronAPI?.models?.fetchModels()
      if (result) {
        set(modelsAtom, result.models)
        set(apiKeyAtom, result.apiKey)
      }
    } catch (error) {
      console.error('获取模型列表失败:', error)
    } finally {
      set(modelsLoadingAtom, false)
    }
  },
)
```

```
应用启动 (main/index.ts)
    ↓
1. initModelService(configDir, serverUrl)     — 初始化模型服务
2. loadCacheFromDisk(jobId)                   — 从磁盘恢复缓存到 L1
3. registerModelsIpcHandlers()                — 注册 IPC handler
4. initModelRefresh()                         — 启动定期后台刷新
    ↓
5. 前端调用 window.electronAPI.models.fetchModels()
    ↓
   缓存命中 → 直接返回
   缓存过期 → 网络请求 → 落盘 → 返回
    ↓
6. 登出 → clearCache()                       — 清除 L1 + L2
7. shutdownModelRefresh()                     — 停止定期刷新（退出时）
```

### 刷新触发场景

| 场景 | 触发 |
|------|------|
| 定期刷新 | TTL 到期自动触发（静默后台） |
| 登录成功 | 立即触发一次刷新 |
| 登出 | 清除缓存，**不**触发刷新 |
| 用户手动刷新 | `forceRefresh=true` |

### 9.1 模型平台服务

文件：`src/services/model-platform.service.ts`

```typescript
import { config } from '../config'
import { logger } from '../utils/logger'
import type { ModelInfo } from '../types'

interface ModelPlatformCredentialsResponse {
  apiKey: string
  models: Array<{
    id: string
    name: string
    description?: string
    provider?: string
    maxTokens?: number
    enabled: boolean
  }>
}

export interface UserCredentials {
  apiKey: string
  models: ModelInfo[]
}

export async function getUserCredentials(userId: string): Promise<UserCredentials> {
  const url = `${config.modelPlatformApiUrl}/users/${userId}/credentials`
  logger.info('查询用户凭证和模型列表', { userId, url })

  try {
    const controller = new AbortController()
    const timeoutId = setTimeout(() => controller.abort(), config.modelPlatformTimeoutMs)

    const response = await fetch(url, {
      method: 'GET',
      headers: { 'Content-Type': 'application/json' },
      signal: controller.signal,
    })

    clearTimeout(timeoutId)

    if (!response.ok) {
      logger.error('大模型平台返回错误', { status: response.status, userId })
      return { apiKey: '', models: [] }
    }

    const data: ModelPlatformCredentialsResponse = await response.json()
    return {
      apiKey: data.apiKey,
      models: data.models.filter((m) => m.enabled),
    }
  } catch (error) {
    if (error instanceof Error && error.name === 'AbortError') {
      logger.error('大模型平台请求超时', { userId })
    } else {
      logger.error('查询大模型平台失败', { error, userId })
    }
    return { apiKey: '', models: [] }
  }
}
```

### 9.2 控制器

文件：`src/controllers/models.controller.ts`

```typescript
import { Request, Response, NextFunction } from 'express'
import { getUserCredentials } from '../services/model-platform.service'
import { sendSuccess, sendError } from '../utils/response'
import { logger } from '../utils/logger'

export async function getUserModelsHandler(
  req: Request,
  res: Response,
  next: NextFunction
): Promise<void> {
  try {
    const jobId = req.jobId!
    const { apiKey, models } = await getUserCredentials(jobId)
    sendSuccess(res, { apiKey, models, total: models.length })
  } catch (error) {
    logger.error('获取模型列表失败', { error })
    next(error)
  }
}
```

## 十、安全注意事项

| 项 | 措施 |
|----|------|
| API Key 传输 | 通过 EIP 网关，请求链路全程经企业网关保护 |
| API Key 落盘 | `safeStorage.encryptString()` 加密存储 |
| 模型列表落盘 | 明文存储（无敏感信息） |
| 工号不可伪造 | 工号由 EIP 网关加密注入，服务端解密后作为查询条件 |
| 模型权限 | 由大模型平台统一下发，服务端不做额外过滤（仅过滤 enabled=false） |
| 缓存账号隔离 | 缓存与 jobId 绑定，账号切换后自动失效 |
| 日志脱敏 | 日志中不输出完整的 apiKey 值 |

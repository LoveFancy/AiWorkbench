## 一、设计思路

应用启动后以**随机延迟**首次检测，之后按**随机抖动的周期**重复检测。检测到更新后**自动后台下载安装包**，下载完成后弹窗提示用户安装，用户可选择立即安装或下次安装。安装前校验版本号一致性。

### 核心原则

1. **最小侵入**：所有升级代码集中在 `apps/electron/src/upgrade/` 独立目录
2. **随机分散**：首次检测延迟和周期检测间隔均加入随机因子，避免"早高峰"集中请求
3. **自动下载 + 用户确认安装**：检测到更新后自动后台下载，下载完成弹窗提示，用户确认后执行安装

### 与现有 auto-updater 的互斥

Proma 已有 `auto-updater.ts`（基于 `electron-updater` 检查 GitHub Release）。WorkMate 升级检测走服务端 API。**两套升级系统互斥**，按 `workmate.json` 是否配置 `serverUrl` 决定启用哪一套：

| `workmate.serverUrl` | 启用模块 | 升级源 |
|---------------------|---------|--------|
| 已配置 | `registerUpgradeCheck()` | WorkMate Server |
| 未配置 | 原有 `initAutoUpdater()` | GitHub Release |

## 三、随机检测策略

### 3.1 为什么需要随机分散

如果不加随机，所有用户早上 9:00 打开客户端，都在 9:00:10 发起首次检测，12 小时后又在 21:00:10 同时检测——服务端瞬间承受大量并发。加入随机后，检测时间被分散到不同的时间窗口内。

### 3.2 随机算法

```
首次检测延迟:
  delay = random(10_000, 3_600_000)    // 10秒 ~ 60分钟

周期检测间隔:
  interval = BASE_INTERVAL + random(0, JITTER_RANGE)
  BASE_INTERVAL = 6 * 3600 * 1000      // 6小时
  JITTER_RANGE  = 2 * 3600 * 1000      // ±2小时随机抖动

  即：实际间隔在 6h ~ 8h 之间随机分布
```

**效果**：假设 100 个用户同一时刻启动，首次检测会分散在 10s~60min 内。后续周期检测，每个人的节奏也不同（有人 6.3h，有人 7.7h），最终各用户的检测时间点会完全发散。

### 3.3 行为矩阵

| 场景 | 首次检测延迟 | 后续检测间隔 |
|------|-------------|-------------|
| 正常启动 | 10s ~ 60min 随机 | 6h ~ 8h 随机 |
| 登录成功 | 10s 后立即检查一次 | 不影响周期计时器 |
| 用户手动点击刷新 | 立即检查 | 不影响周期计时器 |
| 登出 | — | 不触发检测 |
| `serverUrl` 未配置 | 不启动检测 | — |

## 四、升级流程

```
1. 后台静默检测 → 发现新版本
          ↓
2. 自动后台下载安装包（downloadUrl）
   - 下载过程中渲染进程展示下载进度
   - 下载失败可重试，不影响正常使用
          ↓
3. 下载完成 → 弹窗提示用户安装
   ┌──────────────────────────────────────┐
   │  更新已就绪                          │
   │                                      │
   │  当前版本：1.0.0                     │
   │  目标版本：1.2.0                     │
   │  更新类型：升级                      │
   │                                      │
   │  更新内容：                          │
   │  - 修复了若干已知问题                │
   │  - 优化了性能                        │
   │                                      │
   │       [下次安装]    [立即安装]        │
   └──────────────────────────────────────┘
          ↓
4a. 用户点击 [立即安装]
    → 安装前校验（见 4.2 安装前版本校验）
    → 校验通过 → 执行安装
    → 校验失败 → 提示异常，不执行安装
          ↓
4b. 用户点击 [下次安装]
    → 关闭弹窗，安装包保留在本地
    → 下次启动时自动提示安装（步骤 3）
```

### 4.1 与现有 auto-updater 流程对比

| 阶段 | 现有 auto-updater | WorkMate 升级检测 |
|------|-------------------|-------------------|
| 检测源 | `electron-updater` → `latest.yml` | WorkMate Server API |
| 灰度控制 | 无 | 服务端白名单 + 阶段推进 |
| 回退支持 | 无 | `releaseType: ROLLBACK` |
| 下载方式 | `electron-updater` 增量下载 | `fetch` 全量下载安装包 |
| 安装方式 | `quitAndInstall()` 自动重启 | `shell.openPath()` 启动安装程序 |
| 安装前校验 | 无 | 版本号一致性校验 |

### 4.2 安装前版本校验

**为什么需要校验**：下载的安装包版本可能与检测结果不一致（服务端策略变更、下载文件被替换等），安装前必须二次确认。

**校验规则**：

```
执行安装前：
1. 读取本地安装包的版本号（Windows: 解析 EXE 文件版本信息；
   macOS: 读取 Info.plist 中的 CFBundleShortVersionString）
2. 对比安装包版本号与 UpgradeCheckResult.latestVersion：
   - 一致 → 执行安装
   - 不一致 → 提示"安装包版本异常，无法安装"，不执行安装
3. 对比安装包版本号与当前运行版本 app.getVersion()：
   - UPGRADE: 安装包版本 > 当前版本 → 执行安装
   - ROLLBACK: 安装包版本 < 当前版本 → 执行安装
   - 方向不符 → 提示"版本方向异常"，不执行安装
```

### 4.3 forceUpdate 的特殊处理

当 `forceUpdate=true` 时：
- 弹窗仅显示 [立即安装] 按钮，无 [下次安装] 选项
- 弹窗不可关闭/忽略
- 安装包下载完成后持续提示，直到用户完成安装

### 4.4 下载中断与重试

| 场景 | 处理方式 |
|------|---------|
| 下载过程中网络断开 | 记录已下载字节数，下次检测时尝试断点续传（如果服务端支持 Range），否则重新下载 |
| 下载过程中应用退出 | 临时文件保留，下次启动时重新下载（覆盖不完整文件） |
| 安装包下载后文件被删除 | 下次检测时重新下载 |
| 安装包校验失败 | 删除本地文件，下次检测时重新下载 |

## 五、接口定义

### 5.1 客户端 → 服务端

```
GET /workmate/upgrade/check?currentVersion=X&platform=Y
Cookie: EIPGW-TOKEN=<jwt>
```

**请求参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `currentVersion` | string | 当前客户端版本号 |
| `platform` | string | 平台：win32 / darwin / linux |

### 5.2 服务端 → 客户端

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "hasUpdate": true,
    "forceUpdate": false,
    "releaseType": "UPGRADE",
    "latestVersion": "1.2.0",
    "downloadUrl": "https://example.com/releases/1.2.0/installer.exe",
    "releaseNotes": "修复了若干已知问题，优化了性能",
    "minVersion": "1.0.0",
    "hint": "有新版本可用，建议更新",
    "publishedAt": "2026-05-20T10:00:00Z"
  },
  "timestamp": 1717300000000
}
```

| 字段 | 说明 |
|------|------|
| `hasUpdate` | 是否有更新（包括升级和回退） |
| `forceUpdate` | 是否强制升级（用户必须升级才能继续使用） |
| `releaseType` | `UPGRADE` 升级 / `ROLLBACK` 回退 |
| `latestVersion` | 目标版本号 |
| `minVersion` | 最低允许的当前版本（低于此版本必须升级） |
| `downloadUrl` | 安装包下载地址 |
| `releaseNotes` | 发布说明 |
| `hint` | 提示文案 |

### 5.3 服务端检测逻辑（参考，不在客户端范围内）

```
GET /workmate/upgrade/check
    ↓
1. 查 ACTIVE 策略 → 无 → hasUpdate=false
                 → 有 → 进入步骤 2
2. 白名单校验（基于策略已执行阶段的规则）：
   - 全量阶段（rules 为空）→ 全员可用
   - 非全量阶段 → 合并已执行阶段规则，匹配用户工号
     → 不匹配 → hasUpdate=false
3. 版本比较：
   - UPGRADE: targetVersion > currentVersion → hasUpdate=true
   - ROLLBACK: targetVersion < currentVersion → hasUpdate=true
4. 返回 { hasUpdate, releaseType, latestVersion, downloadUrl, ... }
```

### 5.4 端侧二次校验

收到 `hasUpdate=true` 后，客户端必须校验版本方向的正确性：

```
releaseType = 'UPGRADE':
  latestVersion > currentVersion → 有效
  latestVersion ≤ currentVersion → 忽略（异常数据）

releaseType = 'ROLLBACK':
  latestVersion < currentVersion → 有效
  latestVersion ≥ currentVersion → 忽略（异常数据）
```

```typescript
// ===== SemVer 比较 =====

export function isVersionNewer(a: string, b: string): boolean {
  const pa = a.split('.').map(n => parseInt(n, 10) || 0)
  const pb = b.split('.').map(n => parseInt(n, 10) || 0)
  for (let i = 0; i < 3; i++) {
    const diff = (pa[i] ?? 0) - (pb[i] ?? 0)
    if (diff !== 0) return diff > 0
  }
  return false
}

// ===== 随机工具 =====

function randomBetween(min: number, max: number): number {
  return min + Math.floor(Math.random() * (max - min))
}

// ===== 核心检测逻辑 =====

export async function checkForUpgrade(): Promise<UpgradeCheckResult> {
  const currentVersion = app.getVersion()
  const empty: UpgradeCheckResult = {
    hasUpdate: false, forceUpdate: false, releaseType: null,
    currentVersion, latestVersion: null, minVersion: null,
    downloadUrl: null, hint: null, releaseNotes: null,
  }

  if (!_serverUrl) return empty

  const token = getToken()
  if (!token) return empty

  setStatus({ status: 'checking' })

  const baseUrl = _serverUrl.replace(/\/+$/, '')
  const platform = process.platform
  const url = `${baseUrl}/workmate/upgrade/check`
    + `?currentVersion=${encodeURIComponent(currentVersion)}`
    + `&platform=${encodeURIComponent(platform)}`

  try {
    const response = await fetch(url, {
      headers: { 'Cookie': `EIPGW-TOKEN=${token}` },
      signal: AbortSignal.timeout(10_000),
    })

    if (!response.ok) { setStatus({ status: 'idle' }); return empty }

    const json = await response.json()
    if (json.code !== 0 || !json.data) { setStatus({ status: 'idle' }); return empty }

    const data = json.data
    const result: UpgradeCheckResult = {
      hasUpdate: Boolean(data.hasUpdate),
      forceUpdate: Boolean(data.forceUpdate),
      releaseType: data.releaseType ?? null,
      currentVersion,
      latestVersion: data.latestVersion ?? null,
      minVersion: data.minVersion ?? null,
      downloadUrl: data.downloadUrl ?? null,
      hint: data.hint ?? null,
      releaseNotes: data.releaseNotes ?? null,
      publishedAt: data.publishedAt,
    }

    // 端侧二次校验
    if (result.hasUpdate && result.latestVersion) {
      const valid = validateVersionDirection(
        result.releaseType, result.latestVersion, currentVersion,
      )
      if (!valid) {
        result.hasUpdate = false
        result.releaseType = null
      }
    }

    _lastResult = result

    // 检测到更新 → 自动启动下载
    if (result.hasUpdate && result.downloadUrl) {
      void downloadUpdate(result)
    } else {
      setStatus({ status: 'idle' })
    }

    return result
  } catch {
    setStatus({ status: 'idle' })
    return empty
  }
}

function validateVersionDirection(
  releaseType: string | null,
  latestVersion: string,
  currentVersion: string,
): boolean {
  if (releaseType === 'UPGRADE') {
    return isVersionNewer(latestVersion, currentVersion)
  }
  if (releaseType === 'ROLLBACK') {
    return !isVersionNewer(latestVersion, currentVersion) && latestVersion !== currentVersion
  }
  return false
}

// ===== 下载逻辑 =====

function getInstallerDir(): string {
  return join(app.getPath('userData'), 'upgrade')
}

function getInstallerPath(result: UpgradeCheckResult): string {
  const ext = process.platform === 'win32' ? '.exe' : '.dmg'
  const filename = `WorkMate-${result.latestVersion}${ext}`
  return join(getInstallerDir(), filename)
}

/** 后台下载安装包 */
export async function downloadUpdate(result: UpgradeCheckResult): Promise<string> {
  const installerPath = getInstallerPath(result)

  // 已下载完成且文件存在 → 直接返回
  if (existsSync(installerPath) && statSync(installerPath).size > 0) {
    setStatus({ status: 'downloaded', result, installerPath })
    return installerPath
  }

  setStatus({ status: 'downloading', result, progress: { transferred: 0, total: -1, bytesPerSecond: 0, percent: 0 } })

  try {
    const response = await fetch(result.downloadUrl!, {
      signal: AbortSignal.timeout(30 * 60 * 1000), // 30 分钟超时
    })

    if (!response.ok) throw new Error(`下载失败: HTTP ${response.status}`)

    const total = parseInt(response.headers.get('content-length') ?? '-1', 10)
    const startTime = Date.now()
    let transferred = 0

    // 确保目录存在
    const { mkdirSync } = await import('node:fs')
    mkdirSync(getInstallerDir(), { recursive: true })

    const fileStream = createWriteStream(installerPath)

    // 逐 chunk 追踪进度
    const reader = response.body!.getReader()
    let lastProgressTime = 0

    while (true) {
      const { done, value } = await reader.read()
      if (done) break

      fileStream.write(value)
      transferred += value.length

      const now = Date.now()
      // 每 500ms 更新一次进度，避免过于频繁
      if (now - lastProgressTime > 500) {
        lastProgressTime = now
        const elapsed = (now - startTime) / 1000
        const progress: DownloadProgress = {
          transferred,
          total,
          bytesPerSecond: elapsed > 0 ? transferred / elapsed : 0,
          percent: total > 0 ? Math.round((transferred / total) * 100) : 0,
        }
        setStatus({ status: 'downloading', result, progress })
      }
    }

    fileStream.end()
    await new Promise<void>((resolve, reject) => {
      fileStream.on('finish', resolve)
      fileStream.on('error', reject)
    })

    setStatus({ status: 'downloaded', result, installerPath })
    return installerPath
  } catch (error) {
    // 下载失败，清理不完整文件
    if (existsSync(installerPath)) {
      try { unlinkSync(installerPath) } catch { /* ignore */ }
    }
    setStatus({ status: 'error', error: error instanceof Error ? error.message : String(error) })
    throw error
  }
}

// ===== 安装逻辑 =====

/**
 * 校验安装包版本号
 *
 * Windows: 解析 PE 文件版本信息（使用 electron 的 app.getVersion 或第三方库）
 * macOS: 读取 .dmg 内 .app 的 Info.plist
 *
 * 简化实现：NSIS 安装包文件名包含版本号，从文件名中提取校验；
 * 生产环境应使用 rcedit / electron-builder 的元数据校验。
 */
export function verifyInstallerVersion(
  installerPath: string,
  expectedVersion: string,
): boolean {
  // 从文件名提取版本号: WorkMate-1.2.0.exe → 1.2.0
  const match = installerPath.match(/WorkMate-(\d+\.\d+\.\d+)/)
  if (!match) return false
  return match[1] === expectedVersion
}

/** 安装前校验 + 执行安装 */
export async function installUpdate(): Promise<boolean> {
  const status = _upgradeStatus
  if (status.status !== 'downloaded') return false

  const { result, installerPath } = status
  if (!result.latestVersion) return false

  // 校验 1: 文件是否存在
  if (!existsSync(installerPath)) {
    setStatus({ status: 'error', error: '安装包文件不存在' })
    return false
  }

  // 校验 2: 安装包版本号与检测结果一致
  if (!verifyInstallerVersion(installerPath, result.latestVersion)) {
    setStatus({ status: 'error', error: `安装包版本异常（期望 ${result.latestVersion}），无法安装` })
    // 删除异常文件，下次重新下载
    try { unlinkSync(installerPath) } catch { /* ignore */ }
    return false
  }

  // 校验 3: 版本方向正确
  const currentVersion = app.getVersion()
  if (result.releaseType === 'UPGRADE' && !isVersionNewer(result.latestVersion, currentVersion)) {
    setStatus({ status: 'error', error: '版本方向异常（升级包版本不高于当前版本），无法安装' })
    return false
  }
  if (result.releaseType === 'ROLLBACK' && isVersionNewer(result.latestVersion, currentVersion)) {
    setStatus({ status: 'error', error: '版本方向异常（回退包版本不低于当前版本），无法安装' })
    return false
  }

  // 执行安装
  await shell.openPath(installerPath)
  return true
}

// ===== 定时检测 =====

export function startPeriodicCheck(): void {
  if (_checkTimer) return
  if (!_serverUrl) return

  const firstDelay = randomBetween(FIRST_CHECK_DELAY_MIN, FIRST_CHECK_DELAY_MAX)
  console.log(`[升级检测] 首次检测将在 ${Math.round(firstDelay / 1000)}s 后执行`)

  setTimeout(() => {
    void checkForUpgrade()
    scheduleNext()
  }, firstDelay)
}

function scheduleNext(): void {
  if (!_serverUrl) return
  const interval = BASE_INTERVAL_MS + randomBetween(0, JITTER_RANGE_MS)
  _checkTimer = setTimeout(() => {
    void checkForUpgrade()
    scheduleNext()
  }, interval)
}

export function stopPeriodicCheck(): void {
  if (_checkTimer) {
    clearTimeout(_checkTimer)
    _checkTimer = null
  }
  _lastResult = null
}

export function getLastResult(): UpgradeCheckResult | null {
  return _lastResult
}
```

### 6.4 `preload-bridge.ts` — Preload 桥接注册

主进程状态变化时通过 `webContents.send` 通知渲染进程；渲染进程也可通过 IPC 主动查询和操作。

```typescript
// apps/electron/src/upgrade/preload-bridge.ts

import { contextBridge, ipcRenderer } from 'electron'
import { UPGRADE_IPC_CHANNELS } from './ipc-handlers'

export interface UpgradeElectronAPI {
  /** 手动检查更新 */
  checkForUpdate: () => Promise<unknown>
  /** 获取当前升级状态 */
  getStatus: () => Promise<{
    status: 'idle' | 'checking' | 'downloading' | 'downloaded' | 'error'
    result?: {
      hasUpdate: boolean; forceUpdate: boolean; releaseType: string | null
      currentVersion: string; latestVersion: string | null
      downloadUrl: string | null; releaseNotes: string | null; hint: string | null
    }
    progress?: { transferred: number; total: number; bytesPerSecond: number; percent: number }
    installerPath?: string
    error?: string
  }>
  /** 监听升级状态变化 */
  onStatusChanged: (callback: (status: unknown) => void) => () => void
  /** 执行安装（安装前自动校验版本号） */
  installUpdate: () => Promise<boolean>
}

/**
 * 调用方式（preload/index.ts 中加一行）：
 *   import { registerUpgradePreloadBridge } from '../upgrade'
 *   registerUpgradePreloadBridge()
 */
export function registerUpgradePreloadBridge(): void {
  contextBridge.exposeInMainWorld('electronAPI', {
    upgrade: {
      checkForUpdate: () =>
        ipcRenderer.invoke(UPGRADE_IPC_CHANNELS.CHECK),
      getStatus: () =>
        ipcRenderer.invoke(UPGRADE_IPC_CHANNELS.GET_STATUS),
      onStatusChanged: (callback: (status: unknown) => void) => {
        const handler = (_event: Electron.IpcRendererEvent, status: unknown) => callback(status)
        ipcRenderer.on(UPGRADE_IPC_CHANNELS.ON_STATUS_CHANGED, handler)
        return () => ipcRenderer.removeListener(UPGRADE_IPC_CHANNELS.ON_STATUS_CHANGED, handler)
      },
      installUpdate: () =>
        ipcRenderer.invoke(UPGRADE_IPC_CHANNELS.INSTALL),
    } satisfies UpgradeElectronAPI,
  })
}
```

### 6.5 `atoms.ts` — Jotai 状态

```typescript
// apps/electron/src/upgrade/atoms.ts

import { atom } from 'jotai'
import type { UpgradeStatus } from './types'

/** 升级状态 atom（与主进程 UpgradeStatus 同步） */
export const upgradeStatusAtom = atom<UpgradeStatus>({ status: 'idle' })

/** 是否有可用更新（下载中或已下载） */
export const hasUpdateAtom = atom((get) => {
  const { status } = get(upgradeStatusAtom)
  return status === 'downloading' || status === 'downloaded'
})

/** 是否可安装（已下载完成） */
export const canInstallAtom = atom((get) => {
  return get(upgradeStatusAtom).status === 'downloaded'
})

/**
 * 初始化升级状态订阅
 *
 * 订阅主进程推送的升级状态变化事件。
 * 返回清理函数。
 */
export function initializeUpgradeStatus(
  setStatus: (status: UpgradeStatus) => void,
): () => void {
  const upgrade = window.electronAPI?.upgrade
  if (!upgrade) return () => {}

  // 获取初始状态
  upgrade.getStatus().then(setStatus).catch(() => {})

  // 订阅状态变化
  const cleanup = upgrade.onStatusChanged(setStatus)
  return cleanup
}
```

## 九、URL 约定

- `serverUrl` **不包含**末尾 `/`，**不包含** `/workmate` 前缀
- 内部拼接：`${serverUrl}/workmate/upgrade/check`，并 `replace(/\/+$/, '')` 兜底
- 示例：
  - 配置：`"serverUrl": "https://workmate-server.example.com"`
  - 实际请求：`https://workmate-server.example.com/workmate/upgrade/check?currentVersion=1.0.0&platform=win32`

## 十、安全注意事项

| 项 | 措施 |
|----|------|
| 版本方向校验 | 端侧二次校验 `releaseType` 与版本号方向一致性 |
| 安装前版本校验 | 安装前校验安装包版本号与 `latestVersion` 一致，方向与 `releaseType` 匹配 |
| 白名单 | 服务端根据策略阶段白名单匹配，不在名单内不推送 |
| 强制升级 | `forceUpdate=true` 时弹窗不可关闭，仅显示 [立即安装] |
| 下载链接 | 服务端统一下发，避免中间人篡改（经 EIP 网关保护） |
| 安装包完整性 | 文件名包含版本号用于校验；生产环境应增加 hash 校验 |

## 一、本地开发

### 1.1 前置条件

| 软件 | 最低版本 | 验证命令 |
|------|----------|----------|
| JDK | 17 | `java -version` |
| Maven | 3.9+ | `mvn -version` |
| MySQL | 8.0 | `mysql --version` |
| Node.js（仅前端） | 18.x | `node -v` |

### 1.2 首次启动步骤

#### 1）创建数据库

```sql
CREATE DATABASE workmate_server CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 2）配置环境变量

复制 `.env.example` 为 `.env`，修改数据库连接信息：

```bash
DATABASE_URL=jdbc:mysql://localhost:3306/workmate_server?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
DB_USERNAME=root
DB_PASSWORD=your_password
REQUIRE_USER_ID=false
DEFAULT_USER_ID=test_user
```

> 环境变量会被 `application.yml` 中的 `${VAR_NAME:default}` 语法读取。也可通过 IDEA Run Configuration 或命令行 `-D` 参数传入。

#### 3）编译并启动

```bash
# 在 workmate-server-java 目录下

# 编译 + 启动（开发模式，支持热重载需配合 spring-boot-devtools）
mvn spring-boot:run

# 或者先打包再运行
mvn package -DskipTests
java -jar target/workmate-server-1.0.0.jar
```

启动成功后终端输出：

```
Started WorkmateServerApplication in X.XXX seconds
```

#### 4）验证

```bash
curl http://localhost:6173/workmate/health
# {"code":0,"message":"ok","data":{"status":"ok","timestamp":1717300000000},"timestamp":1717300000000}
```

### 1.3 IDEA 开发配置

1. 用 IntelliJ IDEA 打开 `workmate-server-java` 目录
2. 等待 Maven 依赖下载完成
3. 创建 Spring Boot Run Configuration：
   - Main class: `com.workmate.server.WorkmateServerApplication`
   - Environment variables: 填入 `.env` 中的变量
4. 点击 Run 启动

### 1.4 数据库迁移

Flyway 会在应用启动时自动执行 `src/main/resources/db/migration/` 下的 SQL 迁移脚本。

```bash
# 手动执行迁移（不启动应用）
mvn flyway:migrate

# 查看迁移状态
mvn flyway:info

# 修复（解决校验失败）
mvn flyway:repair
```

### 1.5 运行测试

```bash
# 运行所有测试
mvn test

# 运行指定测试类
mvn test -Dtest=UpgradeServiceTest
```

测试环境使用 H2 内存数据库（`application-test.yml`），无需外部 MySQL。

## 二、管理台前端

### 2.1 启动前端开发服务器

管理台前端代码在 `workmate-server-java/admin/` 目录（React + Vite + Ant Design）：

```bash
cd admin
npm install
npm run dev
```

前端启动后访问 `http://localhost:5173`，Vite 自动将 `/workmate/console` 请求代理到后端 `localhost:6173`。

### 2.2 构建前端生产包

```bash
cd admin
npm run build
# 产物在 admin/dist/
```

构建后的静态文件可被 Java 后端直接托管（见下方部署章节）。

## 三、打包

### 3.1 Maven 打包

```bash
cd workmate-server-java
mvn clean package -DskipTests
```

产物：`target/workmate-server-1.0.0.jar`（Spring Boot Fat JAR，包含所有依赖）

### 3.2 包含管理台前端的完整包

```bash
# 1. 先构建前端
cd admin
npm run build

# 2. 将前端产物复制到 Java 项目的静态资源目录
cp -r dist/* ../src/main/resources/static/admin/

# 3. 打包 Java
cd ..
mvn clean package -DskipTests
```

这样打包出的 JAR 内嵌了管理台前端，访问 `/workmate/admin/` 即可使用。

### 3.3 运行 JAR

```bash
java -jar target/workmate-server-1.0.0.jar
```

通过环境变量覆盖配置：

```bash
java -DDATABASE_URL="jdbc:mysql://localhost:3306/workmate_server" \
     -DDB_USERNAME=root \
     -DDB_PASSWORD=password \
     -DREQUIRE_USER_ID=true \
     -jar target/workmate-server-1.0.0.jar
```

## 四、Docker 部署

### 4.1 构建镜像

```bash
# 先构建管理台前端（如果需要在镜像中包含）
cd admin
npm install && npm run build
mkdir -p ../admin/dist
cp -r dist/* ../admin/dist/

# 构建 Docker 镜像
cd ..
docker build -t workmate-server:latest .
```

### 4.2 Docker Compose 一键部署

```bash
# 启动 MySQL + 应用
docker compose up -d

# 查看日志
docker compose logs -f app

# 停止
docker compose down
```

### 4.3 单独运行应用容器（已有 MySQL）

```bash
docker run -d \
  --name workmate-server \
  -p 6173:6173 \
  -e DATABASE_URL="jdbc:mysql://host.docker.internal:3306/workmate_server?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai" \
  -e DB_USERNAME=root \
  -e DB_PASSWORD=password \
  -e USER_ID_ENCRYPTION_KEY="0123456789abcdef0123456789abcdef" \
  -e REQUIRE_USER_ID=true \
  workmate-server:latest
```

### 4.4 常用 Docker 命令

```bash
docker ps                          # 查看运行容器
docker logs -f workmate-server     # 查看实时日志
docker restart workmate-server     # 重启容器
docker exec -it workmate-server sh # 进入容器调试
docker system prune -a             # 清理无用镜像和容器
```

## 五、虚拟机部署

### 5.1 环境要求

| 组件 | 版本要求 |
|------|----------|
| 操作系统 | CentOS 7+ / Rocky Linux 8+ |
| JDK | 17+ |
| MySQL | 8.0+ |
| Nginx | 1.20+ |

### 5.2 部署架构

```
用户 / EIP 网关
       │
       ▼
  Nginx (端口 80/443)
  反向代理 + 静态资源托管
       │
       ▼
  Java 应用 (端口 6173)
       │
       ▼
  MySQL 8.0 (端口 3306)
```

### 5.3 安装 JDK 17

```bash
# Rocky Linux 8/9
sudo dnf install -y java-17-openjdk java-17-openjdk-devel

# CentOS 7
sudo yum install -y java-17-openjdk java-17-openjdk-devel

# 验证
java -version
```

### 5.4 部署步骤

```bash
# 1. 上传 JAR 到服务器
scp target/workmate-server-1.0.0.jar user@server:/opt/workmate/

# 2. 创建 systemd 服务
sudo tee /etc/systemd/system/workmate-server.service <<'EOF'
[Unit]
Description=WorkMate Server
After=network.target

[Service]
Type=simple
User=workmate
WorkingDirectory=/opt/workmate
ExecStart=/usr/bin/java -jar /opt/workmate/workmate-server-1.0.0.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 3. 启动服务
sudo systemctl daemon-reload
sudo systemctl enable workmate-server
sudo systemctl start workmate-server

# 4. 查看状态
sudo systemctl status workmate-server
sudo journalctl -u workmate-server -f
```

### 5.5 Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name workmate.example.com;

    # 管理台前端静态文件（如果 JAR 未内嵌）
    location /workmate/admin/ {
        alias /opt/workmate/public/admin/;
        try_files $uri $uri/ /workmate/admin/index.html;
    }

    # 后端 API
    location /workmate/ {
        proxy_pass http://127.0.0.1:6173;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 七、配置项说明

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `DATABASE_URL` | `jdbc:mysql://localhost:3306/workmate_server` | 数据库连接 |
| `DB_USERNAME` | `root` | 数据库用户名 |
| `DB_PASSWORD` | (空) | 数据库密码 |
| `USER_ID_ENCRYPTION_KEY` | (空) | AES-128-GCM 密钥（16字节 Hex） |
| `REQUIRE_USER_ID` | `true` | 是否强制校验用户身份 |
| `DEFAULT_USER_ID` | `test_user` | 开发环境默认用户 |
| `MODEL_PLATFORM_API_URL` | `http://model-platform.htsc.com/api/v1` | 大模型平台地址 |
| `MODEL_PLATFORM_TIMEOUT_MS` | `10000` | 大模型平台请求超时（毫秒） |
| `OBSERVABILITY_SAMPLE_RATE` | `1.0` | 观测数据采样率 |
| `OBSERVABILITY_MAX_EVENTS_PER_MINUTE` | `60` | 每分钟最大事件数 |
| `LOG_LEVEL` | `info` | 日志级别 |

# WorkMate 伴行使用教程

编辑时间：2026年6月14日

## 工具定位

WorkMate 伴行是华泰面向内部研发与办公场景，基于开源 Agent 技术深度定制的智能工作助手，深度整合了多模型对话、任务执行、知识沉淀和远程协同能力。

它提供两种工作方式：

- **Chat 模式**：日常问答、文档总结、文本润色、翻译、信息查询等轻量任务。
- **Agent 模式**：需要多步骤处理、涉及文件操作或需要产出交付物的复杂任务。

如果使用过程中遇到问题，请联系秦晓012950。

## 快速上手

### 第一步：登录认证

WorkMate 伴行需要先登录后才能使用全部功能。

1. 打开 WorkMate 伴行，进入设置页面。
2. 在"泰为平台模型"区域点击登录，输入工号和 OA 密码完成认证。
3. 登录后可获取公司模型平台能力、访问 SkillHub 市场等。

> **Token 有效期**：30 天。有效期内重新打开客户端会自动恢复登录状态。

### 第二步：环境检查

首次使用建议先完成环境检查，确认本机基础环境正常。

1. 进入设置页面，找到"环境检查"入口，点击运行检查。
2. 确认 Git、Node.js 等基础工具已安装并加入 PATH。

### 第三步：配置模型

使用 AI 功能前需要配置至少一个模型渠道。WorkMate 伴行支持两种模型来源：

| 来源 | 说明 | 如何配置 |
|------|------|---------|
| 泰为平台模型 | 登录 OA 后自动拉取 | 设置 → 泰为平台模型 → 勾选启用 |
| 自定义模型 | 手动配置第三方渠道 | 设置 → 模型配置 → 添加渠道 |

如果你有自己的第三方模型 Key（如 DeepSeek、OpenAI 等），可在"模型配置（自定义）"区域添加渠道。

每个模型标注了能力标识：

- **纯文本**：仅支持文字对话。
- **多模态**：支持识别图片、文档中的图片内容。

启用的模型会自动出现在 Agent 供应商列表中，Agent 模式下会同时使用所有已启用的供应商。

### 第四步：开始对话

1. 在左侧边栏点击"新建聊天"。
2. 输入你的问题或任务描述。
3. 如需处理文件，可直接拖拽或粘贴附件。

## Chat 模式

Chat 模式适合一次性、轻量、以文本回复为主的任务。

### 常见用法

- 解释概念、总结文档、翻译和润色文本。
- 上传 PDF、Word、Markdown、图片等附件进行分析和摘要。
- 通过联网搜索获取最新信息。
- 使用记忆能力保持对话连续性。

### 使用建议

- 给模型一个清晰的**角色**和**任务描述**，能得到更精准的回复。
- 处理长文档时，可以先让模型列出要点，再逐步深入。
- 如果对回答不满意，可以补充信息或换个角度重新提问。
- Chat 模式不会修改你本地的文件。如需创建或编辑文件，请使用 Agent 模式。

## Agent 模式

Agent 模式用于处理复杂的多步骤任务，可以自动拆解、逐步执行。

### 适合处理

- 整理材料并生成报告、表格或文档。
- 资料收集与方案对比。
- 批量处理文件或数据。
- 将重复性工作流程沉淀为 Skill，后续一键复用。

### 使用建议

1. 给 Agent 描述清楚**目标**、**输入材料**、**期望输出**和**保存位置**。
2. 对于复杂任务，先让 Agent 制定计划，确认后再执行。
3. 在 Agent 执行涉及文件修改、删除等操作时，注意确认权限提示。
4. 长任务建议要求 Agent 先列出步骤清单。
5. 任务完成后，检查输出的文件路径和内容。

## 工作区

工作区用于隔离不同类型的工作，每个工作区有独立的会话、文件和 Skills。

### 建议的划分方式

- 日常办公工作区：周报、汇报、材料整理。
- 项目工作区：某个具体项目的所有相关工作和产出。
- 文档分析工作区：集中处理文档类任务。
- 数据整理工作区：处理 Excel、CSV 等数据类工作。

划分工作区有助于：
- 避免不同项目的内容互相干扰。
- 将约定和流程沉淀到对应工作区。
- 让 Agent 更精准地理解当前任务的上下文。

## Skills

Skills 是把某类工作的固定流程沉淀下来的方法说明，让 Agent 在需要时自动读取并按流程执行。

### 适合做成 Skill 的场景

- 固定的报告或文档写作格式。
- 重复的数据整理规则。
- 需要多次执行的操作步骤。
- 特定业务材料的处理标准。

### 使用方式

1. 在 Agent 对话中描述你希望沉淀的流程。
2. 让 Agent 帮你生成 Skill 文件。
3. Skill 会保存在当前工作区，后续对话中 Agent 会自动识别并加载。
4. 如果 Skill 不够准确，可以让 Agent 继续优化。

## SkillHub 市场

SkillHub 是公司内部的 Skill 共享平台，可以浏览和安装他人发布的 Skills。

### 使用方法

1. 在 Agent 设置中找到 SkillHub 面板（需先登录 OA）。
2. 浏览可用 Skills 列表，可按名称或分类搜索。
3. 点击某个 Skill 可预览其详细说明。
4. 点击安装即可添加到当前工作区。
5. 安装的 Skill 可通过开关控制是否启用。
6. 有新版本时可一键升级。

### 注意事项

- 从 SkillHub 安装的 Skills 会存放在当前工作区的 `skills/` 目录下。
- 即使 SkillHub 不可用，已有的本地 Skills 仍可正常使用。
- 不再需要的 Skill 可以卸载删除。

## MCP（外部服务连接）

MCP 用于将外部工具或服务接入 Agent，减少手动切换系统和复制粘贴。

### 典型应用

- 连接公司内部文档系统，让 Agent 直接检索和引用。
- 接入数据平台，让 Agent 查询和分析数据。
- 连接邮件或日历服务，管理日程和沟通。

### 使用建议

- 仅启用在当前工作区真正需要的 MCP。
- 涉及账号、凭证等敏感信息时，按公司安全要求处理。
- 对涉及线上数据修改的操作保持确认意识。

## 记忆与联网搜索

### 记忆

让 WorkMate 伴行记住你的偏好、背景和常用约定，在后续对话中自动沿用。

- 重要偏好、项目背景等可以直接告诉模型"请记住"。
- 可随时在设置中查看、编辑或清除已保存的记忆。

### 联网搜索

用于获取模型训练知识范围之外的实时信息。

- 在对话中手动开启联网搜索开关。
- 适合查询最新政策、新闻、版本信息等时效性强的内容。
- 对重要结论，建议要求 Agent 给出来源和验证依据。

## 远程使用

配置飞书、钉钉或微信桥接后，可通过手机或群聊向本机的 Agent 提交任务。

### 典型场景

- 外出时通过手机发送任务。
- 让 Agent 整理材料后返回摘要到聊天。
- 通过群聊提交需求或待办事项。

### 前提条件

- 本机客户端保持在线。
- 网络连接正常。
- 已配置好对应的桥接设置。

## 客户端升级

WorkMate 伴行支持自动检测和安装新版本。

### 升级方式

- **自动检查**：启动后会在一定时间内随机触发首次检查，之后每隔 6 到 8 小时自动检查一次。
- **手动检查**：进入设置 → 关于，点击"检查更新"。

### 升级流程

1. 检测到新版本后，客户端自动下载安装包，可查看下载进度。
2. 下载完成后，弹窗提示安装。
3. 确认后自动安装并重启客户端。

## 常见问题

### 模型相关

**问：模型回复不够准确怎么办？**

可以尝试：补充更多背景信息、调整提问方式、换一个模型渠道、或使用 Agent 模式让模型先分析再回复。

**问：为什么有些模型不可用？**

检查模型是否在设置中已启用、网络是否正常、API Key 是否有效。泰为平台模型需要先登录 OA。

### 环境相关

**问：环境检查显示 Git 或 Node.js 未安装怎么办？**

根据提示安装对应工具。安装时建议使用默认路径，并勾选"加入 PATH"选项。

**问：为什么 Agent 执行命令时失败？**

确认本机已安装所需工具、PATH 配置正确、网络可访问外部资源。

### 工作区相关

**问：我应该创建多少个工作区？**

建议按项目或工作类型划分，3-5 个工作区通常够用。太少会互相干扰，太多管理不便。

**问：工作区之间可以共享 Skills 吗？**

Skills 是绑定到工作区的。如果需要跨工作区使用，可以复制 Skill 文件到另一个工作区的 `skills/` 目录。

### 其他问题

**问：配置失败或功能异常怎么办？**

记录你的操作步骤、错误提示、当前工作区和使用的模型渠道，联系秦晓012950。

## 使用建议

- 提问时尽量说明**背景**、**目标**、**输入材料**、**输出格式**。
- 复杂任务先让 Agent 制定计划，确认后再执行。
- 重要结论要求 Agent 给出依据和验证方式。
- 重复性流程逐步沉淀为 Skills，一次投入、长期复用。
- 长期项目把关键材料放在对应工作区，方便后续继续。
- 善用记忆功能，让工具更了解你的工作习惯。

# WorkMate 伴行使用教程

编辑时间：2026年5月19日

## 工具定位

WorkMate 伴行是华泰面向内部研发与办公场景，基于开源 Agent 技术深度定制的智能工作助手，用于把多模型 Chat、Agent 工作区、Skills、MCP、记忆、联网搜索和远程机器人能力整合到同一个桌面客户端中。

它适合两类工作方式：

- **Chat 模式**：适合日常问答、总结、翻译、润色、附件解析、轻量分析和简单知识查询。
- **Agent 模式**：适合需要读取文件、修改文件、执行命令、拆解多步骤任务、沉淀工作流或生成交付物的复杂任务。

如果使用过程中遇到问题，请联系秦晓012950。

## 首次使用

首次打开 WorkMate 伴行后，建议按下面顺序完成检查和配置：

1. 先完成环境检查，确认本机 Git、Node.js / Bun、Shell 等基础环境可用。
2. 进入设置页，确认 AI 渠道、模型、Agent 工作区已经配置。
3. 如果需要联网搜索、记忆、飞书、钉钉、微信或 MCP 能力，进入设置页对应模块继续配置。
4. 新建一个 Chat 对话测试模型是否可用。
5. 新建一个 Agent 会话，选择合适工作区，再尝试让 Agent 读取或整理一个小文件。

Windows 用户如果需要使用 Agent 模式，建议将 Git、Node.js / Bun 等基础依赖安装到默认系统盘路径，并确保安装时勾选加入 PATH。

## Chat 模式

Chat 模式更适合一次性、轻量、以文本回复为主的问题。

常见场景：

- 解释概念、总结材料、翻译和润色文本。
- 读取 PDF、Word、Markdown、图片等附件内容后做摘要和分析。
- 使用联网搜索能力获取较新的公开信息。
- 使用记忆能力延续个人偏好、项目背景和常用表达方式。
- 在不同模型之间切换，对比回答质量。

注意事项：

- Chat 模式通常不会直接修改本地文件。如果任务需要生成、编辑或批量处理文件，优先使用 Agent 模式。
- 上传文档后，模型读取到的通常是文本内容；如果图片、排版、表格格式很重要，建议使用 Agent 模式处理。
- 实时性或准确性要求高的问题，建议打开联网搜索，并要求模型给出依据或让 Agent 进一步核验。

## Agent 模式

Agent 模式用于执行复杂任务。它可以结合文件系统、Shell、Git、MCP、Skills 等能力完成更长流程的工作。

适合场景：

- 修改、创建、整理本地文件。
- 读取项目代码、分析问题、生成方案、执行构建或测试。
- 将调研结果输出为 Markdown、表格、报告或其他本地文件。
- 处理需要多步骤拆解的工作，例如资料收集、方案对比、文档改写、批量整理。
- 将重复性工作沉淀为 Skills，后续在同一工作区复用。

使用建议：

1. 给 Agent 描述清楚目标、输入材料、期望输出格式和保存位置。
2. 涉及代码或重要文件时，先让 Agent 分析现状并给出计划。
3. 遇到破坏性操作、删除文件、覆盖文件、推送代码等动作时，确认后再执行。
4. 对于长任务，要求 Agent 输出阶段性计划和待办清单。
5. 任务完成后，检查 Agent 给出的文件路径、摘要和验证结果。

## 工作区

工作区用于隔离不同项目或不同类型工作。每个工作区可以拥有独立的会话、文件、Skills 和 MCP 配置。

建议按工作类型创建工作区，例如：

- 日常办公工作区
- 代码项目工作区
- 文档分析工作区
- 数据整理工作区
- 飞书或远程任务工作区

这样可以减少无关上下文干扰，也方便把固定流程沉淀成可复用能力。

## Skills

Skills 是可复用的工作技能说明。它把某类工作的方法、步骤、规则、输入输出格式沉淀下来，让 Agent 在需要时自动读取并执行。

适合沉淀为 Skills 的内容：

- 固定的报告写作格式。
- 固定的需求分析、测试分析、代码审查流程。
- 部门内部常用的数据整理规则。
- 需要多次复用的操作步骤。
- 对某类业务材料的判断标准。

使用建议：

- 如果一项工作会重复做三次以上，就可以考虑沉淀为 Skill。
- Skill 内容应尽量结构化，包含触发场景、输入要求、处理步骤、输出标准。
- 发现 Skill 不准确时，可以让 Agent 帮你继续优化。

## MCP

MCP 用于把外部工具或服务连接到 Agent，例如文档系统、表格系统、日历、邮件、知识库、内部服务等。

配置 MCP 后，Agent 可以在权限允许的范围内调用外部能力，减少人工切换系统和复制粘贴。

使用 MCP 时需要注意：

- 只启用当前工作区真正需要的 MCP。
- 涉及账号、凭证、授权的信息，要按公司安全要求处理。
- 对写入、删除、发送消息、修改线上数据等操作保持确认意识。

## 记忆与联网搜索

记忆能力用于让工具记住长期有效的偏好、背景、项目约定和协作习惯。

联网搜索用于补充模型知识范围之外或时效性较强的信息。

建议：

- 重要偏好可以明确说“请记住”。
- 对公开信息、政策、版本、价格、新闻等内容，优先使用联网搜索确认。
- 对高风险结论，要求 Agent 给出来源、核验过程和不确定性说明。

## 远程使用

如果配置了飞书、钉钉或微信桥接，可以通过移动端或群聊触发本机 Agent 工作流。

典型用途：

- 在外出时发送任务给本机 Agent。
- 让 Agent 在工作区内整理材料并返回摘要。
- 通过群聊提交需求、问题或待办。
- 让 Agent 在完成任务后通知结果。

注意：远程任务依赖本机客户端在线、网络可用、工作区和模型渠道配置正确。

## 常见问题

### 问：我应该用 Chat 还是 Agent？

只需要回答、解释、总结时用 Chat；需要操作文件、执行命令、处理多步骤任务或生成交付物时用 Agent。

### 问：Agent 为什么要选择工作区？

工作区能隔离项目上下文、Skills 和 MCP 配置，避免不同任务互相干扰。

### 问：为什么 Agent 有时会请求权限确认？

涉及命令执行、文件修改、外部工具调用等操作时，权限确认可以降低误操作风险。

### 问：配置失败或功能异常怎么办？

请记录你的操作步骤、错误提示、当前工作区和使用的模型渠道，然后联系秦晓012950。

## 使用建议

- 提问时尽量说明背景、目标、输入材料、输出格式和限制条件。
- 对复杂任务，先让 Agent 制定计划，再执行。
- 对重要结论，要求给出依据和验证方式。
- 对重复流程，逐步沉淀为 Skills。
- 对长期项目，把关键材料放到对应工作区，方便后续复用。

WorkMate 伴行的核心目标是帮助你把日常工作中的问答、研究、文件处理和流程执行整合起来，让 AI 不只停留在聊天，而是能稳定地产出结果。

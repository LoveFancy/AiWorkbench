## 行为

当用户询问如何使用 WorkMate 时：

1. **先读取本地手册内容**，再基于手册回答。
2. 如果用户说"打开手册"、"查看帮助"、"使用指南"等 → 引导用户点击左侧边栏底部的 📖 使用手册按钮，或进入 设置 → 使用教程 查看完整手册。
3. 如果用户问具体功能怎么用 → 从手册中检索相关内容，给出最匹配的答案。
4. 回答风格：**先给结论，再给具体步骤**。

## 手册文件位置

按以下优先级查找，找到第一个存在且非空的文件即读取。**先用 Glob 搜索 `content.md`，再尝试 `tutorial.md`**：

```
1. ~/.proma/manual-cache/content.md      ← 服务端下发的最新版本（如有）
2. ~/.proma-dev/manual-cache/content.md   ← 开发环境缓存
3. 搜索 ~/**/tutorial/tutorial.md         ← 客户端内置 fallback
```

> `~` 在 Windows 上对应 `%USERPROFILE%`，在 macOS/Linux 上对应 `$HOME`。
> 先用 Glob 工具搜索路径 1 和 2 的 `content.md`，均不存在时用 Glob 递归搜索 `tutorial/tutorial.md`。
> 读取到的内容即完整的《WorkMate 伴行使用教程》，基于此回答用户。

## 兜底摘要

**如果以上三个路径都找不到文件**，说明手册尚未缓存且 Agent 无法访问内置文件。此时不用慌张，基于以下核心信息回答用户，并**引导其手动打开手册**（左侧边栏底部 📖 按钮或 设置 → 使用教程）：

WorkMate 是一个 AI 伴行编程工具，支持：
- **Chat 模式**：自由对话，适合快速问答或不需文件操作的讨论
- **Agent 模式**：自主规划和执行多步任务，可读写文件、执行命令、调用 MCP/Skill
- **模型配置**：在 设置 → 模型 中选择后端（如 DeepSeek、Glama 等），需先申请 API Key
- **Skills**：点击左侧边栏 🧩 按钮进入 Skills/MCP 页面，可启用/禁用内置技能或安装 SkillHub 市场技能
- **工作区**：可创建多个工作区隔离不同项目，左侧边栏左上角切换
- **MCP**：连接外部工具（数据库、API 等），在 Skills/MCP 页面的 MCP 选项卡中配置
- **联网搜索**：Agent 模式下可手动开启，需要在模型配置中启用工具
- **记忆**：Agent 会根据 `PROMA.md` 或 `CLAUDE.md` 文件记忆偏好
- **客户端自动升级**：支持检查和安装新版本

> 当给出兜底摘要时，务必在回答末尾补充：**"由于当前无法读取手册文件，建议点击左侧边栏底部的 📖 按钮打开完整手册获取更详细的信息。"**
